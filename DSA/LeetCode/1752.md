# 1752. Check if Array Is Sorted and Rotated 🟢

**Topics:** Array





## Intuition

Imagine a sorted circular belt of numbers.

Example:

`[1,2,3,4,5]`

If we rotate the belt, we might get:

`[3,4,5,1,2]`

Notice something important:

* In a perfectly sorted array, numbers only increase or stay equal.
* After rotation, there can be only one "drop" where a bigger number is followed by a smaller one.

Example:

`3 -> 4 -> 5 -> 1 -> 2`

The only drop is:

`5 -> 1`

If we find more than one drop, then the array cannot come from a sorted rotated array.








## Approach

We will count how many times the order breaks.

For every index `i`:

* Compare `nums[i]` with the next element
* The next element is:

  `nums[(i + 1) % n]`

  This `% n` helps us wrap around to the beginning.

If:

```python
nums[i] > nums[(i + 1) % n]
```

then we found a drop.

Rules:

* `0` drops → already sorted
* `1` drop → sorted and rotated
* More than `1` drop → invalid








## Example Walkthrough

### Input

```python
nums = [3,4,5,1,2]
```

### Step-by-step

| Current | Next | Drop? |
| ------- | ---- | ----- |
| 3       | 4    | No    |
| 4       | 5    | No    |
| 5       | 1    | Yes   |
| 1       | 2    | No    |
| 2       | 3    | No    |

Total drops = `1`

So the answer is:

```python
True
```










## Code

```python
from typing import List

class Solution:
    def check(self, nums: List[int]) -> bool:

        breaks=0

        n = len(nums)
        for i in range(n):
            if nums[i] > nums[(i+1)%n]:
                breaks+= 1
                if breaks>1:
                    return False
        return True
```






## Complexity

* Time: $O(n)$
* Space: $O(1)$







## Edge Cases

1. Already sorted array

```python
[1,2,3,4]
```

Valid because rotation by `0` is allowed.

2. Array with duplicates

```python
[2,2,2,1,2]
```

Still valid.

3. Multiple drops

```python
[2,1,3,4]
```

Invalid because there are multiple ordering breaks.

4. Single element

```python
[5]
```

Always valid.










## Lessons

* Rotated sorted arrays usually have at most one ordering break.
* Circular traversal using `% n` is a very common trick.
* Instead of simulating rotation, look for structural patterns in the array.
