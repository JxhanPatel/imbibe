# 3474. Lexicographically Smallest Generated String 🔴

**Topics:** String Matching, Greedy, KMP / Z-function

## Intuition

Imagine you're typesetting a long banner. Each position on the banner has a rule card: either **"this exact phrase must start here"** (T) or **"this phrase must NOT start here"** (F). Your goal is to print the cheapest-looking banner (the one that comes earliest alphabetically).

The T positions are bossy. They slam down the phrase `str2` and you have no say. The F positions are more subtle. You want to put `'a'` there (to keep it lex-small), but you have to check: did the T positions already force some letters at this spot? And does putting 'a' here accidentally create a forbidden match?

That's the whole puzzle. Respect the T-rules, then sneak around the F-rules as cheaply as possible.


## Approach

We use two passes:

**Pass 1 (T positions):** Every `'T'` at index `i` forces `word[i:i+m] = str2`. We stamp those characters down. If two T positions conflict (they stamp different letters at the same spot), we immediately return `""`.

**Pass 2 (F positions):** Fill remaining spots with `'a'`. Then for each `'F'` at index `i`, check if `word[i:i+m]` accidentally equals `str2`. If it does, we need to change exactly one character to break the match, choosing the rightmost changeable position (to keep it lex-small) and bumping it to `'b'`. But we must not touch a spot that was locked by a T. We use the **KMP failure function** (also called the prefix function) to efficiently figure out the longest prefix of `str2` that also appears as a suffix of `str2`, so we know exactly which character to change.

The key insight for the F-fix: if `word[i:i+m] == str2`, we scan from the right of that window to find the first position not locked by T, and change it to `'b'`. Then we use the KMP prefix array to check if THAT change still accidentally creates a match somewhere else.

## Example Walkthrough

**Input:** `str1 = "TFTF"`, `str2 = "ab"` → expected output: `"ababa"`

Word length = 4 + 2 - 1 = 5. Start with `word = ['?','?','?','?','?']`.

**Pass 1 (T positions):**
- `i=0` is T → stamp `word[0:2] = "ab"` → `['a','b','?','?','?']`
- `i=2` is T → stamp `word[2:4] = "ab"` → `['a','b','a','b','?']`
- No conflicts, so we're good.

**Pass 2 (F positions):**
- Fill remaining `?` with `'a'` → `['a','b','a','b','a']` = `"ababa"`
- `i=1` is F → check `word[1:3]` = `"ba"` ≠ `"ab"`. OK.
- `i=3` is F → check `word[3:5]` = `"ba"` ≠ `"ab"`. OK.
- No F window accidentally matches `str2`. Done!

**Output:** `"ababa"` ✓



## Code

```python
class Solution:
    def generateString(self, str1: str, str2: str) -> str:

        n =  len(str1)
        m = len(str2)

        total = n + m - 1  

        word = [''] * total


        locked = [False] * total


        # pass 1: handle all 'T' cases
        for i in range(n):
            if str1[i] == 'T':

                for j in range(m):
                    ch = str2[j]
                    pos = i + j  
                    

                    if word[pos] == '':
                        word[pos] = ch
                        locked[pos] = True

                    else:
                        if word[pos] != ch:
                            # conflict... yeah this can't work
                            return ""


        # fill remaining spots
        # defaulting to 'a' (could be anything honestly)
        for i in range(total):
            if word[i] == '':
                word[i] = 'a'


        # build KMP prefix (failure) function
        pi = [0] * m
        k = 0


        for i in range(1, m):
            # FALLBACK until we find a match or reach 0
            while k > 0 and str2[i] != str2[k]:
                k = pi[k - 1]


            if str2[i] == str2[k]:
                k += 1

            pi[i] = k

        # pass 2: handle 'F' case
        for i in range(n):
            if str1[i] == 'F':
                window = word[i:i + m]

                # converting every time... not optimal but okay
                if window == list(str2):
                    fixed = False

                    # try to break match from right side
                    for j in range(m - 1, -1, -1):
                        pos = i + j

                        if not locked[pos]:
                            word[pos] = 'b'  
                            fixed = True
                            break

                    if not fixed:
                        # everything is locked... no way to fix this
                        return ""

        # Final check using KMP
        text = word
        matches = set()
        k = 0

        for i in range(total):
            while k > 0 and text[i] != str2[k]:
                k = pi[k - 1]

            if text[i] == str2[k]:
                k += 1

            if k == m:
                start_idx = i - m + 1
                matches.add(start_idx)

                k = pi[k - 1]

        # sanity check
        for i in range(n):
            if str1[i] == 'T':

                if i not in matches:
                    return ""
                    
            elif str1[i] == 'F':
                if i in matches:
                    # still matched where it shouldn't
                    return ""

        # join and return
        result_str = ''.join(word)

        # debug line I used earlier (keeping it here just in case)
        # print("Final:", result_str)

        return result_str
```



## Complexity

**Time:** $O((n + m) \cdot m)$ in the naive window-comparison steps (pass 1 and 2 each scan windows of size $m$), plus $O(n + m)$ for the KMP run. In the worst case this is $O(n \cdot m)$.

**Space:** $O(n + m)$ for the word array and the KMP prefix table.



## Edge Cases

**1. Conflicting T windows**
If two T positions both try to write different characters at the same index, it's immediately impossible. Return `""`.
Example: `str1 = "TT"`, `str2 = "ab"` → index 1 gets stamped `'b'` by i=0 and `'a'` by i=1 → conflict.

**2. F position that can't be broken**
If a F window's every character is locked by surrounding T positions, you can't change anything to avoid the match. Return `""`.

**3. Single character str2**
If `m = 1`, every T forces that one character at that index, and every F must NOT have that character there. This is the tightest overlap scenario since every position is potentially both a T-stamp target and an F-check target.

**4. All F, no T**
The word is all `'a'`s by default. You just need to verify no window of `'a'`s equals `str2`. This only breaks if `str2 == "aaa...a"` (all a's) and any F window exists → change the last `'a'` in each such window to `'b'`.

**5. str2 has a repeated prefix (KMP matters)**
Example: `str2 = "abab"`. When you break a match at an F position, you need the KMP table to know whether your fix accidentally creates another partial match that chains into a full one. This is subtle and exactly why we do the final KMP verification pass.



## Lessons

**1. Two-pass greedy is your friend here.** Separate "what must be true" (T pass) from "what should be true by default" (F fill). Don't try to handle both simultaneously.

**2. KMP failure function is for more than just searching.** It also tells you about the self-overlapping structure of a string, which is essential when you're trying to break a match without accidentally creating another.

**3. Lock tracking is critical.** Whenever greedy fills might conflict with forced constraints, keep a boolean mask of "who put this character here" so you never overwrite a hard requirement.

**4. When in doubt, verify at the end.** A full KMP scan at the end as a sanity check is cheap compared to debugging a subtle off-by-one. It also catches edge cases where your greedy fix accidentally broke a T window.

**5. Lex-smallest means rightmost fix.** When you have to change something to break a forbidden match, change the rightmost changeable position. This preserves as many early small characters as possible.
