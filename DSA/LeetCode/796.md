# 796. Rotate String 🟢

**Topics:** String, Simulation






## Intuition

Think of the string like a circular necklace of beads. Instead of physically rotating it again and again, imagine joining the string to itself.

For example:
`s = "abcde"` → `"abcdeabcde"`

Now any rotation of `s` will appear somewhere inside this doubled string. So instead of simulating shifts, we just check if `goal` exists inside `s + s`.






## Approach

We use a simple trick:

1. If lengths of `s` and `goal` are different → return `False`.
2. Concatenate the string with itself → `s + s`.
3. Check if `goal` is a substring of `s + s`.

Why it works:
All possible rotations of `s` are contained within `s + s`.







## Example Walkthrough

**Input:**
`s = "abcde"`
`goal = "cdeab"`

Step 1:
Check lengths → both are 5, continue

Step 2:
Concatenate → `"abcdeabcde"`

Step 3:
Check substring → `"cdeab"` is inside `"abcdeabcde"`

**Output:** `True`







## Code

```python
class Solution:
    def rotateString(self, s: str, goal: str) -> bool:

        if len(s) != len(goal):
            return False

        return goal in s+s
```







## Complexity

Time Complexity:
$O(n)$

Space Complexity:
$O(n)$




## Edge Cases

1. `s = "a", goal = "a"` → True
2. `s = "abc", goal = "cab"` → True
3. `s = "abc", goal = "acb"` → False
4. `s = "", goal = ""` → True (though constraints avoid this)





## Lessons

* Rotations often hint at circular behavior
* Doubling a string is a powerful trick for rotation problems
* Always check length first to avoid unnecessary work
