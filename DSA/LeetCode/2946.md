# 2946. Matrix Similarity After Cyclic Shifts 🟢
Topics: Array, Matrix, Simulation

## Intuition

Think of each row in the matrix like a **circular conveyor belt**. Every round, items on even-indexed belts slide one spot to the left, and items on odd-indexed belts slide one spot to the right. After some number of rounds (k), you want to know: does every belt look exactly the same as it did at the start?

Here's the key insight: a belt with `n` items naturally completes a full cycle after exactly `n` shifts. After `n` moves, every item is right back where it started. So the only thing you actually need to check is whether `k` is a multiple of `n`. If `k % n == 0` for a given row, that row is guaranteed to be identical to the original, no need to even look at it. If not, you simulate the shift and compare.

## Approach

Instead of actually shifting the matrix `k` times (which could be slow), we shift each row just once by the effective amount (`k % n`), then compare to the original. Even-indexed rows shift left, odd-indexed rows shift right. If every row after this effective shift matches the original, return `True`.

For a row of length `n`:
- Left shift by `k % n` positions: `row[k%n:] + row[:k%n]`
- Right shift by `k % n` positions: `row[-(k%n):] + row[:-(k%n)]`

This is O(m * n) instead of O(m * n * k), which is the win here.

## Example Walkthrough

Using `mat = [[1,2,3],[4,5,6],[7,8,9]], k = 4`, `n = 3`:

Effective shift = `4 % 3 = 1` (shift by 1 position)

- Row 0 (even, left shift by 1): `[1,2,3]` → `[2,3,1]`. Compare to `[1,2,3]` → not equal.
- We can already stop and return `False`!

Using `mat = [[1,2,1,2],[5,5,5,5],[6,3,6,3]], k = 2`, `n = 4`:

Effective shift = `2 % 4 = 2`

- Row 0 (even, left shift by 2): `[1,2,1,2]` → `[1,2,1,2]`. Same!
- Row 1 (odd, right shift by 2): `[5,5,5,5]` → `[5,5,5,5]`. Same!
- Row 2 (even, left shift by 2): `[6,3,6,3]` → `[6,3,6,3]`. Same!
- All rows match → return `True`.

## Code

```python
class Solution:
    def areSimilar(self, mat: List[List[int]], k: int) -> bool:

        m = len(mat)
        n = len(mat[0])

        for i in range(m):

            row = mat[i]
            shift = k % n

            if shift == 0:
                continue



            if i % 2 == 0:
                shifted = row[shift:] + row[:shift]

            else:
                shifted = row[-shift:] + row[:-shift]



            if shifted != row:
                return False

        return True
```

## Complexity

**Time:** $O(m \times n)$ where $m$ is the number of rows and $n$ is the number of columns. We visit each cell at most once.

**Space:** $O(n)$ for the temporary `shifted` row. We don't store the whole matrix again.

## Edge Cases

1. **All rows are identical values (like `[[2,2],[2,2]]`):** Any shift of a uniform row gives the same row back, so this always returns `True` no matter what `k` is.

2. **k is an exact multiple of n:** `k % n == 0`, so every row is trivially back to its original state. Return `True` immediately without simulating anything.

3. **Single column matrix (`n = 1`):** Any shift of a 1-element row does nothing. Always returns `True`.

4. **k = 1 with a non-symmetric row:** The very first row that isn't a rotation of itself will fail immediately. Good to check your direction logic (left vs right) here.

## Lessons

- **Cyclic shifts have a natural period.** A row of length `n` returns to itself every `n` shifts. This means you never need to simulate more than `n` shifts total. Always ask: "what's the effective change?" before simulating naively.

- **Early return saves time.** The moment one row doesn't match, you're done. No need to check the rest.

- **Slicing is your friend for rotations.** The pattern `arr[k:] + arr[:k]` for left rotation and `arr[-k:] + arr[:-k]` for right rotation is a clean, reusable trick you'll see in many rotation problems.

- **Handle the `shift == 0` case explicitly.** Slicing with `row[0:]` or `row[-0:]` can behave unexpectedly in Python (`row[-0:]` returns the whole list, not an empty slice), so checking for zero shift upfront keeps your code clean and correct.
