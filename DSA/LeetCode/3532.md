# 3532. Path Existence Queries in a Graph I 🟡

**Topics:** Graph, Union Find, Disjoint Set Union (DSU), Array

## Intuition

Imagine a line of train stations arranged in increasing order of their distance from a city center.

Two neighboring stations can be connected directly only if the distance between them is at most `maxDiff`.

Now suppose you want to know whether two stations can reach each other. Instead of exploring the graph for every query, we can first group together all stations that are connected.

The key observation is that **`nums` is already sorted**.

If two consecutive values differ by more than `maxDiff`, then every value after that will be even farther away, so that gap can never be crossed. Therefore, we only need to connect adjacent elements whose difference is within `maxDiff`.

After these groups are formed, answering each query becomes easy: if both nodes belong to the same group, a path exists.


## Approach

We use the **Disjoint Set Union (Union Find)** data structure.

1. Initialize every node as its own parent.
2. Traverse the sorted array once.
3. For every adjacent pair:

   * If their difference is at most `maxDiff`, merge their sets.
4. Use the `find()` function with **path compression** to efficiently locate the representative of each connected component.
5. For every query:

   * Find the root of both nodes.
   * If both roots are the same, they are connected.
   * Otherwise, they belong to different components.

Since the graph's connectivity is completely determined by adjacent values, we never need to explicitly build all possible edges.


## Example Walkthrough

**Input**

```text
nums = [2,5,6,8]
maxDiff = 2

queries = [[0,1],[0,2],[1,3],[2,3]]
```

### Step 1: Process adjacent pairs

```text
|2 - 5| = 3 > 2
No union

|5 - 6| = 1 <= 2
Union(1, 2)

|6 - 8| = 2 <= 2
Union(2, 3)
```

The connected components become:

```text
{0}
{1,2,3}
```

### Step 2: Answer queries

| Query | Same Component? | Answer |
| ----- | --------------- | ------ |
| [0,1] | No              | False  |
| [0,2] | No              | False  |
| [1,3] | Yes             | True   |
| [2,3] | Yes             | True   |

Output

```text
[False, False, True, True]
```


## Code

```python
from typing import List

class Solution:
    def pathExistenceQueries(self, n: int, nums: List[int], maxDiff: int, queries: List[List[int]]) -> List[bool]:
        
        parent = list(range(n))

        def find(x):
            root=x

            while parent[root] != root:
                root = parent[root]

            while parent[x] != root:
                parent[x], x = root, parent[x]

            return root

        for i in range(n-1):
            if nums[i+1] - nums[i] <= maxDiff:
                a= find(i)
                b= find(i+1)

                if a != b:
                    parent[a] = b

        res=[]
        
        for u, v in queries:
            res.append(find(u) == find(v))
        return res
```


## Complexity

**Time Complexity**

Building the connected components:

$$
O(n \alpha(n))
$$

Answering all queries:

$$
O(q \alpha(n))
$$

Overall:

$$
O((n + q)\alpha(n))
$$

where $\alpha(n)$ is the inverse Ackermann function, which grows so slowly that it is effectively constant.

**Space Complexity**

$$
O(n)
$$

for the parent array.


## Edge Cases

### 1. Single node

```text
nums = [7]
queries = [[0,0]]
```

The node is always connected to itself.

Output:

```text
[True]
```


### 2. No adjacent values can be connected

```text
nums = [1,5,9]
maxDiff = 2
```

Every node remains in its own component, so only queries where `u == v` return `True`.


### 3. Every node belongs to one component

```text
nums = [2,3,4,5]
maxDiff = 3
```

Every adjacent pair is merged, so all nodes become connected.


### 4. Duplicate values

```text
nums = [4,4,4,4]
maxDiff = 0
```

Every adjacent difference is `0`, so all nodes belong to the same connected component.


## Lessons

* A sorted array often lets you reduce graph construction to only adjacent elements.
* Union Find is one of the fastest ways to answer multiple connectivity queries.
* Path compression makes repeated `find()` operations nearly constant time.
* When many connectivity queries are asked on a static graph, preprocessing connected components is much more efficient than running a graph traversal for every query.
