# 3753. Total Waviness of Numbers in Range II 🔴

**Topics:** Digit DP, Dynamic Programming, Memoization

## Intuition

Think of building a number one digit at a time.

A digit becomes a peak or valley only when we know its left and right neighbors.

For example, while writing:

```text
4 8 4
```

we cannot decide whether `8` is a peak until the third digit `4` appears.

This suggests a digit DP approach where we process digits from left to right and keep track of the last two digits. Whenever we add a new digit, we can determine whether the middle digit of those three forms a peak or valley.

Instead of checking every number individually (which is impossible up to `10^15`), we count the total waviness of all numbers `≤ x` and then compute:

```text
answer = F(num2) - F(num1 - 1)
```

where `F(x)` returns the total waviness of all numbers from `0` to `x`.



## Approach

We use **Digit DP**.

For every position, store:

* `pos` → current digit index
* `tight` → whether we are still matching the upper bound
* `started` → whether the number has started (ignoring leading zeros)
* `prev1` → previous digit
* `prev2` → digit before previous digit
* `length` → digits chosen so far
* return:

  * total waviness accumulated
  * number of valid completions

### Key Observation

When adding a new digit:

```text
prev2 prev1 digit
```

we can immediately determine whether `prev1` is:

* a peak:

```text
prev1 > prev2 and prev1 > digit
```

* a valley:

```text
prev1 < prev2 and prev1 < digit
```

If yes, add `1` to the waviness count.

The DP returns both:

* how many numbers can be formed
* total waviness contributed by those numbers

This lets us aggregate counts efficiently.



## Example Walkthrough

Suppose:

```text
num = 4848
```

Digits are built as:

```text
4 → 8 → 4 → 8
```

### Add third digit

```text
4 8 4
```

Check middle digit `8`:

```text
8 > 4 and 8 > 4
```

Peak found.

Waviness = 1

### Add fourth digit

```text
8 4 8
```

Check middle digit `4`:

```text
4 < 8 and 4 < 8
```

Valley found.

Waviness = 2

Final waviness:

```text
2
```



## Code

```python
from functools import lru_cache
class Solution:
    def totalWaviness(self, num1: int, num2: int) -> int:
        
        def count_wavy_up_to(limit: int) -> int:
            if limit<0:
                return 0
                
            s = str(limit)
            n=len(s)

            @lru_cache(maxsize=None)
            def memo(idx, is_restricted, has_started, p2, p1, chain_len):
                if idx==n:
                    return 1, 0

                
                limit_digit = int(s[idx]) if is_restricted else 9
                total_ways, total_waviness = 0, 0


                for digit in range(limit_digit+1):
                    next_restricted = is_restricted and (digit==limit_digit)

                    if not has_started:
                        if digit==0:
                            ways, waviness = memo(idx + 1, next_restricted, False, -1, -1, 0)
                        else:
                            ways, waviness = memo(idx + 1, next_restricted, True, -1, digit, 1)
                        total_ways+= ways
                        total_waviness+= waviness




                    else:
                        score=0

                        if chain_len >= 2:
                            if (p1 > p2 and p1 > digit) or (p1 < p2 and p1 < digit):
                                score=1

                        ways, waviness = memo(idx + 1, next_restricted, True, p1, digit, chain_len + 1)
                        total_ways+= ways
                        total_waviness+= waviness + (score*ways)

                return total_ways, total_waviness

            return memo(0, True, False, -1, -1, 0)[1]

        return count_wavy_up_to(num2) - count_wavy_up_to(num1-1)
```



## Complexity

There are only:

* Positions: at most `16`
* Previous digits: `10 × 10`
* Tight states: `2`
* Started states: `2`

So the number of states is bounded by a small constant.

Time complexity:

$$
O(16 \times 10 \times 10 \times 2 \times 2 \times 10)
$$

$$
O(1)
$$

More precisely:

$$
O(\text{digits} \times 10^3)
$$

Space complexity:

$$
O(\text{digits} \times 10^2)
$$



## Edge Cases

### 1. Numbers with fewer than 3 digits

```text
num1 = 1
num2 = 99
```

Every number has waviness `0`.

Answer:

```text
0
```

### 2. Repeated digits

```text
111
```

Middle digit is neither strictly greater nor strictly smaller.

Waviness:

```text
0
```

### 3. Single number range

```text
4848
```

Contains one peak and one valley.

Waviness:

```text
2
```

### 4. Large bounds

```text
1 ≤ num ≤ 10^15
```

Brute force is impossible, but Digit DP handles it efficiently.



## Lessons

* When a property depends on neighboring digits, Digit DP is often the right tool.
* If a condition involves three consecutive digits, keep the last two digits in the DP state.
* Sometimes DP should return multiple values, such as:

  * count of valid numbers
  * total contribution across those numbers
* For range queries, a common pattern is:

$$
\text{Answer} = F(R) - F(L-1)
$$

where `F(x)` computes the result for all numbers from `0` to `x`.
