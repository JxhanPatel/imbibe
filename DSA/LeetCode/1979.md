# 1979. Find Greatest Common Divisor of Array 🟢

**Topics:** Array, Math, Number Theory

## Intuition

Imagine you have several ropes of different lengths, but you only care about the **shortest** and the **longest** one.

The problem asks for the largest length that can divide **both** of these ropes into equal pieces without leaving any remainder. There is no need to compare every number in the array because the answer depends only on the minimum and maximum values.

To efficiently find this greatest common divisor (GCD), we use the **Euclidean Algorithm**, one of the fastest algorithms for computing the GCD of two integers.

## Approach

The solution consists of two simple steps:

1. Find the smallest and largest numbers in the array.
2. Use the **Euclidean Algorithm** to compute their GCD.

The Euclidean Algorithm repeatedly replaces the larger number with its remainder when divided by the smaller number.

For example:

```text
gcd(24, 18)

24 % 18 = 6
18 % 6 = 0

Answer = 6
```

Once the remainder becomes `0`, the other number is the greatest common divisor.

## Example Walkthrough

**Input**

```text
nums = [2,5,6,9,10]
```

### Step 1: Find the smallest and largest numbers

```text
lo = 2
hi = 10
```

### Step 2: Apply the Euclidean Algorithm

First iteration:

```text
hi = 2
lo = 10 % 2 = 0
```

Since `lo` is now `0`, the loop stops.

### Step 3: Return the answer

```text
hi = 2
```

**Output**

```text
2
```

## Code

```python
class Solution:
    def findGCD(self, nums: List[int]) -> int:

        lo = min(nums)
        hi = max(nums)

        while lo!=0:
            hi, lo = lo, hi % lo

        return hi
```

## Complexity

**Time Complexity**

Finding the minimum and maximum requires:

$$
O(n)
$$

The Euclidean Algorithm runs in:

$$
O(\log(\min(\text{lo}, \text{hi})))
$$

Overall:

$$
O(n)
$$

**Space Complexity**

$$
O(1)
$$

Only a constant amount of extra space is used.

## Edge Cases

### 1. Both numbers are the same

```text
nums = [3,3]
```

Smallest = Largest = `3`

Answer:

```text
3
```

### 2. GCD is 1

```text
nums = [7,5,6,8,3]
```

Smallest = `3`

Largest = `8`

Answer:

```text
1
```

### 3. Array contains only two elements

```text
nums = [12,18]
```

The algorithm directly computes:

```text
gcd(12,18) = 6
```

Answer:

```text
6
```

## Lessons

* Read the problem carefully. It only asks for the GCD of the **smallest** and **largest** elements, not every element in the array.
* The **Euclidean Algorithm** is the standard and most efficient method for finding the GCD of two numbers.
* Using `min()` and `max()` keeps the solution clean and easy to understand.
* Many array problems depend only on extreme values, so identifying the minimum and maximum first can simplify the solution significantly.
