
# **1415. The k-th Lexicographical String of All Happy Strings of Length n** 🟡
**Topics:** String, Backtracking, Math



## Intuition

Imagine you're at a fancy ice cream shop that only serves flavors 'a', 'b', and 'c'. But there's a catch: you can't have the same flavor twice in a row (that would be boring!). You want to find the k-th combination when all valid combinations are listed alphabetically.

The key insight is that this forms a **perfect tree** where each node has exactly 2 children (you can pick any of the 3 letters except the one you just used). This predictable structure lets us use math to jump directly to our answer instead of generating all strings!


## Approach:

Instead of generating all happy strings and picking the k-th one, we'll **build the answer character by character** using the power of counting.

Here's the clever observation: once you pick the first character, there are exactly `2^(n-1)` happy strings that start with it. Why? Because each subsequent position has exactly 2 choices (anything except the previous character).

So we can:
1. Calculate how many strings start with 'a', 'b', or 'c' (each has `2^(n-1)` strings)
2. Determine which "block" our k-th string falls into
3. Repeat this logic for each subsequent position, adjusting k as we go

This is like using a "map" to navigate directly to our destination instead of walking down every single path!



## Example Walkthrough

**Input:** n = 3, k = 9

Total happy strings of length 3: 3 × 2 × 2 = 12

**Step 1: First character**
- Strings starting with 'a': positions 1-4 (indices 0-3, 4 strings)
- Strings starting with 'b': positions 5-8 (indices 4-7, 4 strings)  
- Strings starting with 'c': positions 9-12 (indices 8-11, 4 strings)

k = 9 falls in the 'c' block (positions 9-12). So first char = 'c'.
Adjust k: k = 9 - 8 = 1 (we're the 1st string in the 'c' block)

**Step 2: Second character** (can't use 'c')
Available: 'a', 'b'
- Strings starting with 'ca': positions 1-2 in this block
- Strings starting with 'cb': positions 3-4 in this block

k = 1 falls in 'ca'. So second char = 'a'.
k stays 1.

**Step 3: Third character** (can't use 'a')
Available: 'b', 'c'
- 'cab' is position 1
- 'cac' is position 2

k = 1, so we pick 'b'.

**Result:** "cab" ✓



## Code

```python
class Solution:
    def getHappyString(self, n: int, k: int) -> str:

        total_strings = 3 * (2 ** (n - 1))
        
        # If k is beyond total possible strings, return empty
        if k > total_strings:
            return ""
        
        k -= 1
        result = []
        chars = ['a', 'b', 'c']
        

        # Determine first character
        block_size = 2 ** (n - 1)
        first_char_idx = k // block_size
        result.append(chars[first_char_idx])
        
        # Update k to be relative to our chosen block
        k %= block_size
        


        # Building remaining characters
        for pos in range(1, n):
            # Block size halves each level (2 choices instead of 3)
            block_size //= 2
            
            prev_char = result[-1]
            
            # We have 2 valid options: filter out prev_char
            candidates = [c for c in chars if c != prev_char]
            
            # Pick based on which half k falls into
            next_char_idx = k // block_size
            result.append(candidates[next_char_idx])
            
            k %= block_size
        
        return "".join(result)
```



## Complexity

- **Time Complexity:** $O(n)$ - We build the string one character at a time, doing constant work per character
- **Space Complexity:** $O(n)$ - To store the result string (or $O(1)$ extra if we don't count output)

Compare this to backtracking which would be $O(3 \cdot 2^{n-1})$ time to generate all strings!



## Edge Cases

| Scenario | Input | Output | Why |
|----------|-------|--------|-----|
| k too large | n=1, k=4 | "" | Only 3 strings exist ("a", "b", "c") |
| Smallest n | n=1, k=1 | "a" | Just picking from single characters |
| Last valid string | n=2, k=6 | "cb" | Total 6 strings, last one starts with 'c' |
| k at boundary | n=2, k=4 | "ba" | First string of second block |



## Lessons

1. **Count before you construct:** When dealing with combinatorial objects, calculate how many exist in each "branch" before diving in. This often lets you skip massive amounts of work.

2. **Tree navigation with math:** Perfect trees (where each node has the same number of children) are begging for direct index calculation. Think: "Which subtree contains my answer?"

3. **0-indexed vs 1-indexed:** The problem uses 1-indexed k, but math is cleaner with 0-indexed. Convert early, work with 0-indexed, then convert back if needed.

4. **Block size tracking:** Notice how `block_size` starts at `2^(n-1)` and halves each iteration? This pattern appears whenever you have a branching factor that changes by level.
