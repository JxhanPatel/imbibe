# 2075. Decode the Slanted Ciphertext 🟡 

**Topics:** String, Matrix Simulation



## Intuition

Imagine you're writing a diagonal message on graph paper, going top-left to bottom-right, then someone takes a photo of each row and stitches them together into one long string. Your job is to reverse that process and read the diagonal again.

That's exactly what's happening here. The original text was written diagonally across a matrix. The encoded text is just that matrix read row by row. To decode it, we need to figure out the matrix dimensions, then read the diagonals back.



## Approach

First, we figure out how many columns the matrix had. Since we know the total number of characters (`len(encodedText)`) and the number of rows, the number of columns is just `len(encodedText) // rows`.

Then, we "lay" the encoded text back into a 2D matrix and read each diagonal starting from the top row, going down-right. Each diagonal starting at column `col` gives us characters at positions `(0, col), (1, col+1), (2, col+2)...`, which in the flattened string are `col`, `col + 1*(cols+1)`, `col + 2*(cols+1)`, and so on. This is the key insight: diagonals in the matrix correspond to striding through the flat encoded string with a step of `cols + 1`.

We just strip trailing spaces at the end since the problem guarantees the original text had none.Step through the widget above to see each diagonal being read and the original text rebuilding character by character.



## Example Walkthrough

**Input:** `encodedText = "ch   ie   pr"`, `rows = 3`

Step 1: Compute `cols = 12 // 3 = 4`. Lay it into a 3x4 matrix:

```
c h _ _
i e _ _
p r _ _
```

Step 2: Read diagonals starting from each column in row 0:
- Col 0: `(0,0)=c`, `(1,1)=e`... wait, more precisely: col 0 gives `c, e, p` (going down-right), col 1 gives `h, i, r`, col 2 gives ` `, col 3 gives ` ` (spaces).

Step 3: Strip trailing spaces. Result: `"cipher"`.



## Code

```python
class Solution:
    def decodeCiphertext(self, encodedText: str, rows: int) -> str:
        n = len(encodedText)
        
        if rows == 1:
            return encodedText
        
        cols = n // rows  
        
        result = [] 
        

        # from top row, moving across columns
        for start_col in range(cols):
            row = 0
            col = start_col
            
            # walking diagonally (down-right)
            while True:
                if row >= rows or col >= cols:
                    break   
                

                # convert 2D position back into index
                idx = row * cols + col   # could inline this, but keeping it separate for clarity
                result.append(encodedText[idx])
                
                # move diagonally
                row += 1
                col += 1
        
        decoded = "".join(result)
        
        return decoded.rstrip()   # note: only removing right-side spaces, not touching left ones
```



## Complexity

**Time:** $O(n)$ where $n$ is the length of `encodedText`. We visit every character exactly once.

**Space:** $O(n)$ for the result list we build up.



## Edge Cases

- **`rows = 1`:** The matrix is just one row, so the encoding is a no-op. `encodedText` is already the original text. Return it directly.
- **Very long encoded text with lots of trailing spaces:** The spaces pad the last diagonal. `rstrip()` handles this cleanly without any special logic.
- **`encodedText = ""`:** Length is 0, so `cols = 0 // rows = 0`. The `range(0)` loop never runs and we return `""`. Works fine.
- **`rows = encodedText length`:** The matrix is a single column, so every character is its own "diagonal". You're just reading the column back out, which is the same string. The logic handles this naturally.



## Lessons

**1. Flattened matrix indexing is your friend.** You don't need to actually build a 2D list. `encodedText[row * cols + col]` lets you index into the flat string as if it were a matrix. This is a pattern worth burning into memory.

**2. Recognize the diagonal stride pattern.** Moving diagonally down-right in a matrix corresponds to stepping by `cols + 1` in the flat array. Once you see this, you can skip the 2D reconstruction entirely if you prefer:

```python
# alternative one-liner style inner loop
for step in range(rows):
    if start_col + step < cols:
        result.append(encodedText[start_col + step + step * cols])
```

**3. Always handle the single-row edge case first.** It's the simplest and catches a divide-by-zero-style logic error before it bites you.

**4. `rstrip()` is clean.** Rather than tracking where trailing spaces begin, just build the full result and strip. The problem guarantees the original has no trailing spaces, so you'll never accidentally strip something real.
