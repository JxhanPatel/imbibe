# 1846. Maximum Element After Decreasing and Rearranging 🟡

**Topics:** Greedy, Sorting, Array

## Intuition

Imagine you are building a staircase using blocks.

The first step must always have a height of `1`. Every next step can be at most **one unit taller** than the previous step. If a block is too tall, you can cut it down, but you can never make a block taller.

To build the tallest possible staircase, it makes sense to arrange the blocks from **smallest to largest**. Then, for each block, use the largest height it is allowed to have.

This way, every block contributes as much as possible to increasing the maximum value.


## Approach

We use a **greedy strategy**.

1. Sort the array in ascending order.
2. Force the first element to become `1`.
3. Traverse the remaining elements.
4. For each element:

   * If it is larger than `previous + 1`, decrease it to `previous + 1`.
   * Otherwise, keep it as it is.
5. The last element after processing is the maximum possible value.

Why does this work?

* Sorting ensures we always process the smallest available numbers first.
* Since we can only decrease values, choosing the largest valid value for every position is always optimal.
* This greedily maximizes every element while keeping the adjacent difference at most `1`.



## Example Walkthrough

**Input**

```text
arr = [100,1,1000]
```

### Step 1: Sort

```text
[1,100,1000]
```

### Step 2: Process

| Previous | Current | New Value |
| -------- | ------- | --------- |
| 1        | 100     | 2         |
| 2        | 1000    | 3         |

Final array:

```text
[1,2,3]
```

The maximum element is:

```text
3
```


## Code

```python
class Solution:
    def maximumElementAfterDecrementingAndRearranging(self, arr: List[int]) -> int:
        arr.sort()

        arr[0]=1

        for i in range(1, len(arr)):
            arr[i] = min(arr[i], arr[i - 1] +1)
            
        return arr[-1]
```


## Complexity

**Time Complexity**

Sorting dominates the running time.

$$
O(n \log n)
$$

**Space Complexity**

Ignoring the space used by Python's sorting algorithm,

$$
O(1)
$$


## Edge Cases

1. **Single element**

```text
arr = [10]
```

It must become:

```text
[1]
```

Answer:

```text
1
```


2. **Already valid array**

```text
arr = [1,2,3,4,5]
```

No changes are needed.

Answer:

```text
5
```


3. **All elements are very large**

```text
arr = [100,100,100]
```

After processing:

```text
[1,2,3]
```

Answer:

```text
3
```


## Lessons

* When you are allowed to reorder elements, sorting is often the first step to simplify the problem.
* Greedy algorithms work well when making the locally optimal choice never hurts future decisions.
* When values can only be decreased, always keep each value as large as the constraints allow.
* Building the answer incrementally from left to right is a common pattern for array construction problems.
