# 1009. Complement of Base 10 Integer 🟢
**Topics:** Bit Manipulation, Math



## Intuition

Imagine you have a light switch panel with several switches, all currently in some combination of ON (1) and OFF (0) positions. The complement is like creating the exact opposite pattern: every ON becomes OFF, and every OFF becomes ON.

But here is the catch: you only care about the switches that are actually being used. If your number only needs 3 bits to represent (like 5 = "101"), you do not flip an entire 32-bit panel. You only flip those 3 active switches. This is crucial because flipping unused leading bits would give you a huge wrong answer!

So the trick becomes: **create a mask of all 1s that is exactly the same length as your number's binary representation, then XOR (or subtract) to flip only those bits.**



## Approach

We will use the **bit mask** approach:

1. **Find the bit length** of `n` (how many bits are needed to represent it)
2. **Create a mask** of all 1s with that exact length. For example, if `n` needs 3 bits, the mask is `111` in binary (which is 7 in decimal, or `2³ - 1`)
3. **XOR `n` with the mask** (or subtract: `mask - n`). XOR is perfect here because `1^0=1` and `1^1=0`, which is exactly the flip we want

Why XOR with all 1s? Think of the mask as a stencil. Wherever the mask has a 1, we flip the corresponding bit in `n`. Wherever it has a 0 (beyond our number's length), we leave it alone.



## Example Walkthrough

Let us trace `n = 5`:

| Step | Action | Result |
|------|--------|--------|
| 1 | Find bit length of 5 | `5` in binary is `"101"`, so we need **3 bits** |
| 2 | Create mask of 3 ones | `111` in binary = `7` in decimal (which is `2³ - 1`) |
| 3 | XOR: `5 ^ 7` | `101 ^ 111 = 010` = **2** |

Another quick one: `n = 10` ("1010", 4 bits)
- Mask = `1111` = 15
- `10 ^ 15 = 5` ✓



## Code

```python
class Solution:
    def bitwiseComplement(self, n: int) -> int:
        # edge case:
        if n == 0:
            return 1
            
        # Note:- bit_length() returns the number of bits required to represent an integer in binary 
        num_bits = n.bit_length()
        
        mask = (1 << num_bits) - 1
        
        # XOR flips all the bits where mask has 1s
        return n ^ mask
```




## Complexity

- **Time Complexity:** $O(1)$ -- bit operations and `bit_length()` run in constant time for fixed-size integers
- **Space Complexity:** $O(1)$ -- we only use a few variables regardless of input size



## Edge Cases

| Case | Input | Output | Why |
|------|-------|--------|-----|
| Zero | `n = 0` | `1` | 0 is "0" in binary (1 bit), complement is "1" |
| All 1s | `n = 7` | `0` | 7 is "111", complement is "000" |
| Power of 2 | `n = 8` | `7` | 8 is "1000", complement is "0111" = 7 |
| Large number | `n = 10^9` | computed | Still fits in 32-bit, bit_length handles it |



## Lessons

1. **Bit masking is about focus:** The mask lets us target exactly the bits we care about and ignore the rest. This pattern appears everywhere in low-level programming, networking (subnet masks), and graphics.

2. **XOR with 1s flips, XOR with 0s preserves:** Remember this golden rule. `x ^ 1` flips `x`, `x ^ 0` keeps `x` unchanged. This is why creating the right mask is half the battle.

3. **Know your bit utilities:** Python's `int.bit_length()` is incredibly handy. Before knowing this, many people manually loop to find the highest set bit. Use the standard library!

4. **Watch out for zero:** Zero has a bit length of 0, which would make our mask calculation fail (`(1 << 0) - 1 = 0`). Always consider if your bit manipulation logic holds for `n = 0`.
