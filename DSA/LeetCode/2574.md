# 2574. Left and Right Sum Differences 🟢

**Topics:** Array, Prefix Sum

## Intuition

Imagine you are standing at each position in a line of numbers.

For every position, you want to know:

* How much weight is on your **left side**
* How much weight is on your **right side**

Then you compare the two sides and record the absolute difference.

Instead of recalculating the left and right sums for every index, we can keep track of them efficiently while moving through the array.

## Approach

We use a **prefix sum idea**.

1. Compute the total sum of the array.
2. Maintain a running `left_sum`.
3. For each index:

   * Remove the current number from the remaining total to get `right_sum`.
   * Compute `abs(left_sum - right_sum)`.
   * Add the current number to `left_sum` for the next iteration.
4. Store all differences in the answer array.

This avoids creating separate `leftSum` and `rightSum` arrays and gives a clean one-pass solution.

## Example Walkthrough

**Input**

```text
nums = [10,4,8,3]
```

### Initial Values

```text
total_sum = 25
left_sum = 0
```

### Iterate

| Index | Num | Left Sum | Right Sum | Difference |
| ----- | --- | -------- | --------- | ---------- |
| 0     | 10  | 0        | 15        | 15         |
| 1     | 4   | 10       | 11        | 1          |
| 2     | 8   | 14       | 3         | 11         |
| 3     | 3   | 22       | 0         | 22         |

**Output**

```text
[15,1,11,22]
```

## Code

```python
class Solution:
    def leftRigthDifference(self, nums: List[int]) -> List[int]:
        total_sum = sum(nums)
        left_sum = 0
        answer = []

        for num in nums:
            # Current number should not be in right sum
            total_sum -= num
            right_sum = total_sum

            # Store absolute difference
            answer.append(abs(left_sum - right_sum))

            # Update left sum for next position
            left_sum += num

        return answer
```

## Complexity

Time complexity:

$$
O(n)
$$

Space complexity:

$$
O(1)
$$

Ignoring the output array.

## Edge Cases

1. **Single element**

```text
nums = [1]
```

Output:

```text
[0]
```

2. **All elements are the same**

```text
nums = [5,5,5,5]
```

The logic remains unchanged and produces correct differences.

3. **Minimum array size**

```text
nums = [100000]
```

Left and right sums are both `0`, so the answer is `[0]`.

## Lessons

* Prefix sums help avoid repeatedly calculating sums of subarrays.
* When both left and right information are needed, maintaining a running sum is often more efficient than building multiple arrays.
* A useful trick is to start with the total sum and gradually remove elements to obtain the right-side sum.
* Many array problems can be optimized from $O(n^2)$ to $O(n)$ using running totals.
