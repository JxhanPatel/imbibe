# 2196. Create Binary Tree From Descriptions 🟡

**Topics:** Hash Table, Binary Tree, Tree

## Intuition

Think of the descriptions as a list of family relationships.

Each description tells us:

* Who the parent is
* Who the child is
* Whether the child sits on the left or right side

As we read these relationships, we can gradually build every person (node) and connect them together.

The only missing piece is finding the **root**.

A root is simply the one node that **never appears as a child**. Every other node has exactly one parent, but the root has none.

So our plan is:

1. Create nodes whenever we see a value for the first time.
2. Connect parent and child nodes.
3. Track every value that appears as a child.
4. The node that never appears in the child set is the root.



## Approach

We use two hash-based structures:

* `nodes` → maps a value to its `TreeNode`.
* `children` → stores all values that appear as children.

For each description `[parent, child, isLeft]`:

1. Create the parent node if it does not exist.
2. Create the child node if it does not exist.
3. Attach the child to the left or right of the parent.
4. Add the child value to the `children` set.

After processing all descriptions:

* Iterate through all created nodes.
* The node whose value is **not** in `children` must be the root.
* Return that node.



## Example Walkthrough

### Input

```text
descriptions = [
    [20,15,1],
    [20,17,0],
    [50,20,1],
    [50,80,0],
    [80,19,1]
]
```

### Step 1: Create Connections

```text
20
├── 15
└── 17
```

```text
50
├── 20
└── 80
```

```text
80
└── 19
```

### Step 2: Track Children

```text
children = {15, 17, 20, 80, 19}
```

### Step 3: Find Root

All node values:

```text
{50, 20, 15, 17, 80, 19}
```

Only `50` never appears in `children`.

So the root is:

```text
50
```

### Final Tree

```text
        50
       /  \
     20    80
    / \    /
  15  17  19
```



## Code

```python
# Definition for a binary tree node.
# class TreeNode:
#     def __init__(self, val=0, left=None, right=None):
#         self.val = val
#         self.left = left
#         self.right = right
class Solution:
    def createBinaryTree(self, descriptions: List[List[int]]) -> Optional[TreeNode]:

        nodes={}
        has_parent = set()


        for parent_val, child_val, is_left in descriptions:
            if parent_val not in nodes:
                nodes[parent_val] = TreeNode(parent_val)

            if child_val not in nodes:
                nodes[child_val] = TreeNode(child_val)

            node= nodes[parent_val]

            if is_left:
                node.left= nodes[child_val]

            else:
                node.right= nodes[child_val]

            has_parent.add(child_val)

        return next(nodes[v] for v in nodes if v not in has_parent)
```



## Complexity

Building the tree:

$$
O(n)

$$

Overall:

$$
O(n)
$$

$$
O(n)
$$

for the hash map and child set.



## Edge Cases

### 1. Single Parent-Child Pair

```text
[[1,2,1]]
```

Tree:

```text
1
/
2
```

Root is `1`.



### 2. Completely Skewed Tree

```text
[[1,2,1],[2,3,1],[3,4,1]]
```

Tree becomes:

```text
1
/
2
/
3
/
4
```

Root is still the only node that never appears as a child.



### 3. Root Appears Late in Descriptions

```text
[[2,3,1],[1,2,1]]
```

Even though `1` appears later, it never appears as a child, so it is correctly identified as the root.



## Lessons

* When relationships are given as parent-child pairs, a hash map is often the easiest way to build the tree.
* Tracking all children is a common trick for finding the root of a tree.
* If every node except one has a parent, the node without a parent must be the root.
* Hash maps allow us to create and retrieve tree nodes in constant time while building the structure.
