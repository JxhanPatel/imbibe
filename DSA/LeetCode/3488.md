# 3488. Closest Equal Element Queries 🟡

**Topics:** Hashing, Binary Search, Circular Array






## Intuition

Think of each number as belonging to a "group" of identical values.

Imagine people standing in a circle, and each person has a label (number). For a given person, you want to find the closest *other* person with the same label, either by walking clockwise or counterclockwise.

Instead of searching the whole circle every time, it’s smarter to first group all people with the same label together. Then, for each person, you only need to check their nearest neighbors within that group.







## Approach

We will use **hashing + binary search**:

1. **Group indices by value**
   Use a dictionary where:

   ```
   value -> sorted list of indices where it appears
   ```

2. **For each query index `i`:**

   * Get the value `nums[i]`
   * Look at the list of indices where this value appears
   * If only one occurrence → answer is `-1`

3. **Find closest index using binary search**

   * Locate position of `i` in the sorted list
   * Check:

     * previous index
     * next index
   * Also handle circular wrap:

     * first and last indices in the list

4. **Compute circular distance**

   ```
   dist = min(abs(i - j), n - abs(i - j))
   ```








## Example Walkthrough

```
nums = [1,3,1,4,1,3,2]
queries = [0,3,5]
```

### Step 1: Build map

```
1 -> [0,2,4]
3 -> [1,5]
4 -> [3]
2 -> [6]
```





### Query 0 → index 0, value = 1

Indices = [0,2,4]

Closest:

* next = 2 → distance = 2
* prev (circular) = 4 → distance = min(4, 7-4=3) = 3

Answer = 2




### Query 3 → value = 4

Only one occurrence → -1





### Query 5 → value = 3

Indices = [1,5]

Closest:

* prev = 1 → distance = min(4, 7-4=3) = 3

Answer = 3







## Code

```python
from collections import defaultdict
import bisect

class Solution:
    def solveQueries(self, nums: List[int], queries: List[int]) -> List[int]:

        total = len(nums)

        occurrence_map = defaultdict(list)


        for idx, val in enumerate(nums):
            occurrence_map[val].append(idx)


        answers = []


        for q in queries:
            same_val_positions = occurrence_map[nums[q]]


            if len(same_val_positions) == 1:
                answers.append(-1)
                continue



            pos_in_list = bisect.bisect_left(same_val_positions, q)

            sz = len(same_val_positions)


            left_neighbor = same_val_positions[(pos_in_list - 1) % sz]
            right_neighbor = same_val_positions[(pos_in_list + 1) % sz]


            raw_left = abs(q - left_neighbor)
            raw_right = abs(q - right_neighbor)


            circ_left = min(raw_left, total - raw_left)
            circ_right = min(raw_right, total - raw_right)


            answers.append(min(circ_left, circ_right))


        return answers
```






## Complexity

* Building map: $O(n)$
* Each query: $O(\log k)$ where k = frequency of element
* Total: $O(n + q \log n)$






## Edge Cases

1. All elements unique
   → every query returns `-1`

2. All elements same
   → nearest is always 1 (or circular equivalent)

3. Query at boundary (index 0 or n-1)
   → circular distance must be handled properly





## Lessons

* Grouping similar values avoids repeated work
* Circular arrays often require `min(d, n - d)` trick
* Binary search helps find nearest neighbors efficiently
* Always consider wrap-around cases explicitly
