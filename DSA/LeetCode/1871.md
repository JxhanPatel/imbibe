# 1871. Jump Game VII 🟡

**Topics:** String, Dynamic Programming, Sliding Window, Breadth-First Search

## Intuition

Imagine you are standing on stones across a river.

* `'0'` means the stone is safe to stand on.
* `'1'` means the stone is broken.

From any stone, you can jump within a range:

* at least `minJump`
* at most `maxJump`

The main challenge is efficiency.

A brute force solution would try every possible jump from every index, but that becomes too slow for strings of length up to (10^5).

Instead, we keep track of how many reachable positions exist inside the current jump window. If at least one reachable index can jump to the current position, then the current position is also reachable.

This turns the problem into a smart sliding window problem.



## Approach

We use dynamic programming.

Let:

* `dp[i] = True` if index `i` is reachable
* otherwise `False`

### Key Observation

To reach index `i`, we need some reachable index `j` such that:

$$
i - maxJump \le j \le i - minJump
$$

and:

* `dp[j] == True`
* `s[i] == '0'`

Instead of checking all such `j` every time, we maintain a sliding window count of reachable indices.

### Sliding Window Logic

As we move from left to right:

* Add positions entering the valid jump range
* Remove positions leaving the range

If the count of reachable indices inside the window is greater than `0`, then current index is reachable.



## Example Walkthrough

### Input

```text
s = "011010"
minJump = 2
maxJump = 3
```

### Step-by-step

Initial:

```text
dp = [True, False, False, False, False, False]
```

We start at index `0`.



### i = 2

Valid previous range:

```text
[2 - 3, 2 - 2] = [-1, 0]
```

Index `0` is reachable.

But `s[2] = '1'`, so cannot stand here.



### i = 3

Previous range:

```text
[0,1]
```

Index `0` is reachable and `s[3] = '0'`.

So:

```text
dp[3] = True
```



### i = 5

Previous range:

```text
[2,3]
```

Index `3` is reachable and `s[5] = '0'`.

So:

```text
dp[5] = True
```

We reached the last index.

Answer is `True`.



## Code

```python
class Solution:
    def canReach(self, s: str, minJump: int, maxJump: int) -> bool:

        n=len(s)
        reachable = [False]*n
        reachable[0] = True
        window=0

        for i in range(1, n):
            if i-minJump >= 0 and reachable[i-minJump]:
                window+= 1

            if i-maxJump-1 >= 0 and reachable[i-maxJump-1]:
                window-= 1

            if window>0 and s[i] == '0':
                reachable[i] = True

        return reachable[-1]
```



## Complexity

### Time Complexity

$$
O(n)
$$

Each index is processed once.

### Space Complexity

$$
O(n)
$$

For the `dp` array.



## Edge Cases

### 1. Last index is blocked

```python
s = "0101"
```

Cannot finish because last index is `'1'`.



### 2. Only one possible path

```python
s = "00000"
minJump = maxJump = 2
```

Must follow exact jumps.



### 3. Large continuous blocked section

```python
s = "0001111000"
```

May become impossible to cross.



## Lessons

* Sliding window can optimize repeated range checks.
* Dynamic programming works well for reachability problems.
* Instead of checking every previous position, maintain useful information incrementally.
* Many jump problems become easier when viewed as range-based transitions.
