# 3558. Number of Ways to Assign Edge Weights I 🟡

**Topics:** Tree, DFS, Math, Combinatorics


## Intuition

Imagine walking from the root of the tree (node `1`) to the deepest node.

Along this path, every edge can receive either:

* Weight `1`
* Weight `2`

The only thing that matters is whether the total path cost is **odd** or **even**.

Notice an important observation:

* Weight `2` contributes an even value.
* Weight `1` contributes an odd value.

Therefore, the total path cost is odd when the number of edges assigned weight `1` is odd.

Now suppose the path contains `d` edges.

Each edge has 2 choices, so there are:

$$
2^d
$$

possible assignments.

For any binary choice problem like this, exactly half of the assignments produce an odd sum and half produce an even sum.

So the number of valid assignments is:

$$
\frac{2^d}{2}=2^{d-1}
$$

The only remaining task is finding the maximum depth of the tree.

## Approach

We use DFS to compute the maximum depth starting from node `1`.

### Step 1: Build the tree

Since the input is an undirected tree, we create an adjacency list.

### Step 2: Find the maximum depth

Starting from node `1`, recursively explore every child.

For each node:

* Compute the depth of every subtree.
* Take the maximum among them.
* Add `1` for the current edge.

This gives the depth of the deepest descendant.

### Step 3: Count valid assignments

If the maximum depth is `d`, then the path from the root to a deepest node contains exactly `d` edges.

The answer becomes:

$$
2^{d-1}
$$

computed modulo (10^9 + 7).

## Example Walkthrough

### Input

```text
edges = [[1,2],[1,3],[3,4],[3,5]]
```

### Tree Structure

```text
      1
     / \
    2   3
       / \
      4   5
```

### DFS Depth Calculation

Starting from node `1`:

* Depth of node `2` = 0
* Depth of node `4` = 0
* Depth of node `5` = 0

For node `3`:

```text
max(0, 0) + 1 = 1
```

For node `1`:

```text
max(0, 1) + 1 = 2
```

Maximum depth:

```text
d = 2
```

### Count Assignments

There are two edges on the chosen path.

Total assignments:

$$
2^2 = 4
$$

Half produce an odd sum:

$$
2^{2-1}=2
$$

### Output

```text
2
```

## Code

```python
from collections import deque, defaultdict

class Solution:
    def assignEdgeWeights(self, edges: List[List[int]]) -> int:

        MOD = 10**9 + 7

        adj = defaultdict(list)

        for u, v in edges:
            adj[u].append(v)
            adj[v].append(u)

        def depth(node, parent):
            best=0


            for nb in adj[node]:

                if nb != parent:
                    best = max(best, depth(nb, node)+1)
            return best

        return pow(2, depth(1, -1)-1, MOD)
```

## Complexity

Let (n) be the number of nodes.

DFS visits each node exactly once:

$$
O(n)
$$

Space complexity:

$$
O(n)
$$

for the adjacency list and recursion stack in the worst case.

## Edge Cases

### 1. Single Edge

```text
edges = [[1,2]]
```

Maximum depth:

```text
1
```

Answer:

$$
2^{1-1}=1
$$

### 2. Chain Tree

```text
1 - 2 - 3 - 4 - 5
```

Maximum depth:

```text
4
```

Answer:

$$
2^{3}=8
$$

### 3. Star Tree

```text
    2
    |
3 - 1 - 4
    |
    5
```

Maximum depth:

```text
1
```

Answer:

$$
2^{0}=1
$$

## Lessons

* Tree problems often become much simpler once you identify the single property that matters, here it is the **maximum depth**.
* When a problem asks about odd and even sums, think in terms of **parity** rather than actual values.
* For binary choices, assignments frequently split evenly between odd and even outcomes.
* DFS is a natural way to compute the height or maximum depth of a tree.
* Always look for mathematical simplifications before trying to enumerate all possibilities.
