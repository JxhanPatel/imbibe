# 1861. Rotating the Box 🟡

**Topics:** Arrays, Matrix, Simulation, Two Pointers








## Intuition

Think of each row of the box like a shelf with stones `#`, obstacles `*`, and empty spaces `.`.

Now imagine rotating the box 90° clockwise. After rotation, gravity pulls stones downward.

But here is the key insight:
Instead of actually rotating first and then simulating gravity, we can simulate gravity row by row before rotation.

Why this works:

* When rotated, rows become columns
* Gravity will act downward in the rotated view
* That is equivalent to pushing stones to the right in the original row

So the problem reduces to:
**For each row, move stones `#` as far right as possible until blocked by `*` or another `#`.**









## Approach

1. Traverse each row independently
2. Use a pointer `empty` to track the rightmost position where a stone can fall
3. Traverse the row from right to left:

   * If you see `*`, reset `empty` to just left of it
   * If you see `#`, move it to `empty` and decrement `empty`
4. After processing all rows, rotate the matrix 90° clockwise

Rotation rule:

```
rotated[j][m - 1 - i] = box[i][j]
```










## Example Walkthrough

Input:

```
[["#", ".", "*", "."]]
```

Step 1: Apply gravity (push right)

```
Original row:   # . * .
After gravity:  . # * .
```

Step 2: Rotate

Result:

```
["."]
["#"]
["*"]
["."]
```









## Code

```python
class Solution:
    def rotateTheBox(self, boxGrid: List[List[str]]) -> List[List[str]]:

        rows=len(boxGrid)
        cols=len(boxGrid[0])


        for row in boxGrid:
            drop = cols-1


            for j in range(cols-1, -1, -1):

                if row[j] == '*':
                    drop = j - 1

                elif row[j] == '#':
                    row[j], row[drop] = '.', '#'
                    drop -=1

        rotated = [[None] * rows for _ in range(cols)]

        for i in range(rows):
            for j in range(cols):
                rotated[j][rows-1-i] = boxGrid[i][j]

        return rotated
```








## Complexity

* Time: $O(m \times n)$
* Space: $O(m \times n)$ for the rotated matrix







## Edge Cases

1. Single row or single column
2. All obstacles `*`
3. All empty cells `.`
4. Stones already packed to the right
5. Maximum size grid (500 x 500)








## Lessons

* Simulating physics problems often becomes easier by reversing perspective
* Instead of rotating first, simulate gravity in the original orientation
* Two pointer technique helps efficiently "drop" stones
* Matrix rotation is a common transformation pattern
