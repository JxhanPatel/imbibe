# 3513. Number of Unique XOR Triplets I 🟡

**Topics:** Bit Manipulation, Math

## Intuition

Think of XOR as a special kind of light switch.

* Pressing the same switch twice cancels its effect.
* Pressing it three times is the same as pressing it once.

Because the array is a **permutation of `1` to `n`**, every number appears exactly once. Also, the problem allows choosing the **same index multiple times** (`i <= j <= k`), which means:

* `x ^ x ^ x = x`
* `x ^ x ^ y = y`

So every individual number is always obtainable.

The interesting case is when we choose **three distinct numbers**. Their XOR can create entirely new values that may not exist in the original array.

The key observation is that once `n >= 3`, every distinct triple of values is available because every value appears exactly once.

Even better, it turns out:

* If `n < 3`, only the original numbers can be produced.
* If `n >= 3`, every possible XOR value representable with the required number of bits can be produced.

So instead of enumerating triplets, we only need to determine how many different bit patterns are possible.


## Approach

There are two cases.

### Case 1: `n < 3`

There are fewer than three distinct values.

Since repeating elements only gives back one of the existing values:

```text
x ^ x ^ x = x
x ^ x ^ y = y
```

the answer is simply:

```text
n
```


### Case 2: `n >= 3`

Let:

```text
k = number of bits needed to represent n
```

which is:

```python
k = n.bit_length()
```

Any XOR result must fit within these `k` bits.

The total number of distinct bit patterns is:

```text
2^k
```

or in Python:

```python
1 << k
```

This gives the answer directly.

## Code

```python
class Solution:
    def uniqueXorTriplets(self, nums: List[int]) -> int:

        n = len(nums)
        if n<3:
            return n

        bits = n.bit_length()
        return 1<<bits
```

## Lessons

* Always look for mathematical properties before considering brute force.
* XOR has useful cancellation rules:

  * `x ^ x = 0`
  * `x ^ 0 = x`
  * `x ^ x ^ x = x`
* When the input is a permutation, the actual order often becomes irrelevant.
* `bit_length()` is a handy way to determine how many bits are needed to represent a number.
* Sometimes a problem that appears to require enumerating combinations has a direct mathematical formula.
