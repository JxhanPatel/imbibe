# 2126. Destroying Asteroids 🟡

**Topics:** Greedy, Sorting, Array




## Intuition

Imagine your planet is a snowball rolling downhill.

A small snowball can easily absorb tiny snow chunks, making it larger. As it grows, it can absorb bigger and bigger chunks. The key idea is to absorb the smallest chunks first so the snowball grows steadily before facing the larger ones.

Similarly, if we destroy the smallest asteroids first, the planet gains mass as quickly as possible, giving it the best chance to destroy larger asteroids later.




## Approach

We use a **greedy strategy**:

1. Sort all asteroids in ascending order.
2. Process them from smallest to largest.
3. If the current planet mass is smaller than an asteroid's mass, we immediately return `False`.
4. Otherwise, destroy the asteroid and add its mass to the planet.
5. If all asteroids are processed successfully, return `True`.

Why does sorting work?

* Destroying a smaller asteroid first can only help because it increases the planet's mass.
* There is never a benefit to destroying a larger asteroid before a smaller one.
* Therefore, the optimal strategy is always to process asteroids in increasing order.





## Example Walkthrough

**Input**

```text
mass = 10
asteroids = [3,9,19,5,21]
```

### Step 1: Sort

```text
[3,5,9,19,21]
```

### Step 2: Process

| Planet Mass | Asteroid | Result              |
| ----------- | -------- | ------------------- |
| 10          | 3        | Destroy → mass = 13 |
| 13          | 5        | Destroy → mass = 18 |
| 18          | 9        | Destroy → mass = 27 |
| 27          | 19       | Destroy → mass = 46 |
| 46          | 21       | Destroy → mass = 67 |

All asteroids are destroyed.

**Output**

```text
true
```

## Code

```python
class Solution:
    def asteroidsDestroyed(self, mass: int, asteroids: List[int]) -> bool:

        asteroids.sort()
        
        for a in asteroids:
            if mass<a:
                return False

            mass+= a
            
        return True
```




## Complexity

* Sorting: $O(n \log n)$
* Traversal: $O(n)$

Overall:

$$
O(n \log n)
$$

Space complexity:

$$
O(1)
$$

Ignoring the space used by Python's sorting algorithm.





## Edge Cases

1. **Single asteroid**

```text
mass = 5
asteroids = [5]
```

Answer: `True`

2. **Planet too small initially**

```text
mass = 1
asteroids = [2]
```

Answer: `False`

3. **Large asteroid at the end**

```text
mass = 5
asteroids = [4,9,23,4]
```

After destroying all smaller asteroids, mass becomes `22`, which is still less than `23`.

Answer: `False`





## Lessons

* When gaining resources makes future tasks easier, consider a greedy strategy.
* Sorting often helps when smaller elements can be used to build up strength for larger ones.
* If handling items in increasing order always leaves you in the best possible state, a greedy proof is usually nearby.
* Watch for problems where completing easy tasks first increases your ability to complete harder tasks later.
