# 33. Search in Rotated Sorted Array 🟡

**Topics:** Array, Binary Search






## Intuition

Imagine a dictionary where someone tore it into two parts and swapped them.

Normally, words are fully sorted:

```text
apple banana cat dog elephant
```

After rotation:

```text
dog elephant apple banana cat
```

The array is still partially sorted. At least one half of the array will always remain sorted.

That is the key observation.

Using binary search, we can:

1. Find the middle element
2. Determine which half is sorted
3. Check whether the target belongs in that sorted half
4. Ignore the other half

This allows us to keep reducing the search space in logarithmic time.








## Approach

We use a modified binary search.

At every step:

* Compute `mid`
* If `nums[mid] == target`, return `mid`
* Check which side is sorted:

  * Left half sorted
  * Right half sorted
* Decide whether the target lies inside the sorted half
* Move pointers accordingly

### Key Observation

For any index `mid`:

* If `nums[left] <= nums[mid]`

  * Left side is sorted
* Else

  * Right side is sorted






## Example Walkthrough

### Input

```python
nums = [4,5,6,7,0,1,2]
target = 0
```

### Step 1

```text
left = 0
right = 6
mid = 3
nums[mid] = 7
```

Left half `[4,5,6,7]` is sorted.

Target `0` is not inside this range.

So search right side.

```text
left = 4
```



### Step 2

```text
left = 4
right = 6
mid = 5
nums[mid] = 1
```

Left half `[0,1]` is sorted.

Target `0` lies inside this range.

Move right pointer.

```text
right = 4
```



### Step 3

```text
left = 4
right = 4
mid = 4
nums[mid] = 0
```

Found target.

Return `4`.








## Code

```python
class Solution:
    def search(self, nums: List[int], target: int) -> int:

        lo=0
        hi = len(nums)-1

        while lo <= hi:
            mid = (lo+hi)//2

            if nums[mid]==target:
                return mid

            if nums[lo] <= nums[mid]:
                if nums[lo] <= target < nums[mid]:
                    hi = mid-1

                else:
                    lo = mid+1

            else:
                if nums[mid] < target <= nums[hi]:
                    lo = mid+1
                    
                else:
                    hi = mid-1

        return -1
```







## Complexity

Time Complexity:

$$
O(\log n)
$$

Space Complexity:

$$
O(1)
$$








## Edge Cases

### 1. Single element array

```python
nums = [1]
target = 0
```

Return `-1`.




### 2. Target is the first element

```python
nums = [6,7,1,2,3,4,5]
target = 6
```

Return `0`.




### 3. Array is not rotated

```python
nums = [1,2,3,4,5]
target = 4
```

Normal binary search behavior.




### 4. Target does not exist

```python
nums = [4,5,6,7,0,1,2]
target = 10
```

Return `-1`.








## Lessons

* A rotated sorted array still contains one sorted half.
* Binary search is not only for fully sorted arrays.
* Carefully identifying the sorted side is the main trick.
* Always shrink the search space based on where the target can logically exist.
* Modified binary search patterns appear frequently in interview problems.
