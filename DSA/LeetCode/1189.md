# 1189. Maximum Number of Balloons 🟢

**Topics:** String, Hash Table, Counting

## Intuition

Imagine you are assembling toy balloons in a factory.

Each toy requires the exact letters:

```text
balloon
```

But not every letter is needed equally.

| Letter | Needed |
| ------ | -----: |
| b      |      1 |
| a      |      1 |
| l      |      2 |
| o      |      2 |
| n      |      1 |

Suppose your warehouse contains:

```text
b -> 5
a -> 3
l -> 7
o -> 4
n -> 2
```

Even though you have plenty of `b` and `l`, you can only make:

* `5` balloons based on `b`
* `3` balloons based on `a`
* `7 // 2 = 3` balloons based on `l`
* `4 // 2 = 2` balloons based on `o`
* `2` balloons based on `n`

The answer is the smallest among them:

```text
min(5, 3, 3, 2, 2) = 2
```

This is exactly what the code does.


## Approach

We use a frequency counting approach.

### Step 1

Count how many times each character appears in `text`.

```python
freq = Counter(text)
```

For example:

```text
text = "loonbalxballpoon"

freq =
{
    'l': 4,
    'o': 4,
    'b': 2,
    'a': 2,
    'n': 2,
    ...
}
```


### Step 2

Store the required frequency of each character in `"balloon"`.

```python
needed = {
    'b': 1,
    'a': 1,
    'l': 2,
    'o': 2,
    'n': 1
}
```

This tells us:

* One `b` is needed.
* One `a` is needed.
* Two `l`s are needed.
* Two `o`s are needed.
* One `n` is needed.


### Step 3

For every required character:

```python
freq[ch] // cnt
```

This calculates:

> How many complete balloons can this character alone support?

Examples:

```text
freq['b'] = 2

2 // 1 = 2
```

```text
freq['l'] = 4

4 // 2 = 2
```

```text
freq['o'] = 5

5 // 2 = 2
```

Finally:

```python
min(freq[ch] // cnt for ch, cnt in needed.items())
```

The smallest value is the answer because that character runs out first.


## Example Walkthrough

**Input**

```text
text = "loonbalxballpoon"
```

### Step 1: Count characters

```text
b -> 2
a -> 2
l -> 4
o -> 4
n -> 2
```

### Step 2: Check how many balloons each character can support

```text
b : 2 // 1 = 2

a : 2 // 1 = 2

l : 4 // 2 = 2

o : 4 // 2 = 2

n : 2 // 1 = 2
```

### Step 3: Take the minimum

```text
min(2, 2, 2, 2, 2) = 2
```

**Output**

```text
2
```


## Code

```python
from collections import Counter

class Solution:
    def maxNumberOfBalloons(self, text: str) -> int:
        
        freq = Counter(text)
        needed = {'b': 1, 'a': 1, 'l': 2, 'o': 2, 'n': 1}
        
        return min(freq[ch] // cnt for ch, cnt in needed.items())
```


## Complexity

**Time Complexity**

$$
O(n)
$$

where $n$ is the length of `text`.

* Building `Counter` takes $O(n)$.
* Iterating through `needed` takes $O(1)$ because it always contains only 5 characters.

Overall:

$$
O(n)
$$


**Space Complexity**

$$
O(1)
$$

At most 26 lowercase English letters are stored in the hash table.


## Edge Cases

1. **Missing a required character**

```text
text = "leetcode"
```

There is no `'b'`.

```text
freq['b'] = 0

0 // 1 = 0
```

Answer:

```text
0
```


2. **Exactly one balloon**

```text
text = "balloon"
```

All required characters are present exactly once.

Answer:

```text
1
```


3. **Extra unrelated characters**

```text
text = "balloonxyzxyz"
```

The characters `x`, `y`, and `z` do not affect the answer.

Answer:

```text
1
```


4. **Not enough repeated letters**

```text
text = "balon"
```

Required:

```text
l -> 2
o -> 2
```

Available:

```text
l -> 1
o -> 1
```

Answer:

```text
0
```


## Lessons

* Frequency counting is one of the most common techniques for string problems.
* If a word requires repeated characters, divide the available count by the required count.
* The answer is often determined by the **bottleneck**, which is the resource available in the smallest amount.
* A dictionary like `needed` makes the solution generic and easy to adapt for similar problems such as building any target string from another string.
* Generator expressions inside `min()` are a neat Python trick for computing answers without creating an extra list.
