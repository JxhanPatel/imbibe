# 2033. Minimum Operations to Make a Uni-Value Grid 🟡

**Topics:** Array, Math, Sorting





## Intuition

Think of each number in the grid as a height. You are allowed to raise or lower any height, but only in fixed steps of size `x`.

Now imagine trying to make all heights equal. This is only possible if all numbers are aligned in the same "step pattern". In simpler terms, every value must leave the same remainder when divided by `x`. If not, you can never land them on the same value.

Once that condition is satisfied, the problem becomes:
What target value should we choose so that the total number of moves is minimum?

This is very similar to minimizing distance in 1D, where the best choice is the **median**.






## Approach

1. Flatten the grid into a 1D array.
2. Check feasibility:

   * For all values, `(value % x)` must be the same.
   * If not, return `-1`.
3. Sort the array.
4. Choose the median as the target value.
5. Count operations:

   * For each element, operations = `abs(value - median) // x`
6. Return the total operations.

Why median?
Because it minimizes the sum of absolute differences.








## Example Walkthrough

**Input:**
`grid = [[2,4],[6,8]], x = 2`

Step 1: Flatten
`[2, 4, 6, 8]`

Step 2: Check mod
All values % 2 = 0 → valid

Step 3: Sort
`[2, 4, 6, 8]`

Step 4: Median
Median = `6` or `4` both work, choose `4`

Step 5: Count operations

* |2 - 4| / 2 = 1
* |4 - 4| / 2 = 0
* |6 - 4| / 2 = 1
* |8 - 4| / 2 = 2

Total = `4`







## Code

```python
class Solution:
    def minOperations(self, grid: List[List[int]], x: int) -> int:

        nums = sorted(n for row in grid for n in row)
        
        if any(n % x != nums[0] % x for n in nums):
            return -1
        
        median = nums[len(nums) // 2]

        return sum(abs(n - median) // x for n in nums)
```






## Complexity

Time complexity:
$$O(n \log n)$$
Sorting dominates

Space complexity:
$$O(n)$$
For storing flattened array







## Edge Cases

* Grid with one element → already uniform, return 0
* Values not aligned with `x` → return -1
* Large grid with same values → return 0
* Even number of elements → either median works






## Lessons

* Converting 2D problems into 1D often simplifies thinking
* Always check feasibility before optimization
* Median is optimal for minimizing absolute differences
* Watch for step constraints like modulo conditions
