# 2492. Minimum Score of a Path Between Two Cities 🟡

**Topics:** Graph, Depth-First Search (DFS), Breadth-First Search (BFS), Union Find

## Intuition

Imagine a group of cities connected by roads, where every road has a quality rating.

The **score** of a journey is determined by the **worst road** you travel on. If even one road is in poor condition, the entire trip gets that score.

Now comes the interesting part. The problem allows us to **revisit cities and roads as many times as we want**.

This means we are not forced to take a simple path from city `1` to city `n`. If there is any road with a smaller distance anywhere inside the connected component containing city `1`, we can always detour, pass through that road, and then continue to city `n`.

So instead of searching for the best path, we only need to answer:

> What is the smallest edge in the connected component that contains city `1`?

That smallest edge is the answer.


## Approach

We perform a graph traversal (DFS or BFS) starting from city `1`.

1. Build an adjacency list.
2. Start DFS from city `1`.
3. Visit every city reachable from `1`.
4. For every road encountered, update the answer with its distance:

   ```python
   answer = min(answer, distance)
   ```
5. Since city `n` is guaranteed to be in the same connected component as city `1`, the minimum edge seen during the traversal is the minimum possible score.

The key observation is that because revisiting roads is allowed, every edge inside the connected component is potentially usable in a valid path from `1` to `n`.


## Example Walkthrough

**Input**

```text
n = 4
roads = [[1,2,9],[2,3,6],[2,4,5],[1,4,7]]
```

### Step 1: Build the graph

```text
1 --9-- 2
|       | \
7       6  5
|       |   \
4 ------+----3
```

### Step 2: Start DFS from city `1`

Visit cities:

```text
1 → 2 → 3 → 4
```

Road distances seen:

```text
9, 7, 6, 5
```

Minimum distance:

```text
min(9,7,6,5) = 5
```

**Output**

```text
5
```


## Code

```python
from collections import defaultdict
from typing import List

class Solution:
    def minScore(self, n: int, roads: List[List[int]]) -> int:
        
        adj = defaultdict(list)

        for u, v, d in roads:
            adj[u].append((v, d))
            adj[v].append((u, d))

        seen={1}
        stack=[1]
        best=float("inf")

        while stack:
            node = stack.pop()
            
            for nxt, dist in adj[node]:
                if dist<best:
                    best=dist

                if nxt not in seen:
                    seen.add(nxt)
                    stack.append(nxt)

        return best
```


## Complexity

**Time Complexity**

Building the graph takes:

$$
O(m)
$$

DFS visits every city and road once:

$$
O(n + m)
$$

Overall:

$$
O(n + m)
$$

where `m` is the number of roads.

**Space Complexity**

Adjacency list:

$$
O(n + m)
$$

Visited set and recursion stack:

$$
O(n)
$$

Overall:

$$
O(n + m)
$$


## Edge Cases

1. **Only two cities**

```text
n = 2
roads = [[1,2,10]]
```

Answer:

```text
10
```

2. **Graph contains cycles**

```text
1 --5-- 2
|       |
8       3
|       |
4 ------+
```

DFS handles cycles using the `visited` set.

3. **Extra disconnected component**

```text
1 --4-- 2 --6-- 3

5 --1-- 6
```

The second component is ignored because it cannot be reached from city `1`.


## Lessons

* Always pay attention to special conditions in the problem statement. Here, being allowed to revisit roads completely changes the solution.
* When any path inside a connected component is possible, think about properties of the entire component instead of individual paths.
* A DFS or BFS is often enough to gather information about an entire connected component.
* Sometimes the optimal answer depends on a global property of a component rather than the shortest or simplest path.
