# 3660. Jump Game IX 🟡

**Topics:** Array, Greedy, Dynamic Programming




## Intuition

Instead of exploring every possible jump from every index, this solution groups indices into segments that can all eventually reach the same maximum value.

The key observation is:

* If the current segment still has a smaller value somewhere ahead, then movement can continue.
* As long as we can continue moving through valid jumps, all those positions belong to the same reachable region.
* Every index inside that region can eventually reach the largest value found in that region.

To detect whether movement can continue efficiently, we use a `suffix_min` array.









## Core Idea

We build:

```python id="8qngjv"
suffix_min[i]
```

which stores the minimum value from index `i` to the end.

This helps answer:

> "Is there a smaller number ahead?"

Because:

```python id="1ohw6s"
running_max > suffix_min[j + 1]
```

means there exists a future value smaller than the current maximum, so jumps can continue.










## Approach

### Step 1: Build suffix minimum array

```python id="1k9lf1"
suffix_min[i] = minimum value from i → n-1
```

Example:

```python id="ql0tqv"
nums = [2,3,1]
```

Suffix minimum becomes:

```python id="z9aq4l"
[1,1,1,inf]
```

---

### Step 2: Expand reachable segment

We maintain:

* `running_max` → largest value seen in current segment
* `j` → current segment end

As long as:

```python id="0g56hx"
running_max > suffix_min[j + 1]
```

there exists a smaller future value, meaning jumps are still possible.

So we expand the segment.

---

### Step 3: Fill answers

Once expansion stops:

* every index in that segment can reach the same maximum
* fill the entire segment with `running_max`










# Code

```python id="l5y0u0"
from typing import List

class Solution:
    def maxValue(self, nums: List[int]) -> List[int]:
        n = len(nums)

        suffix_min = [float("inf")] * (n +1)

        for i in range(n-1, -1, -1):
            suffix_min[i] = min(nums[i], suffix_min[i+ 1])

        result = [0]*n
        i=0

        while i < n:
            j=i
            running_max = nums[i]


            while j + 1 < n and running_max > suffix_min[j + 1]:
                j+= 1

                if nums[j] > running_max:
                    running_max=nums[j]

            result[i:j+1] = [running_max] * (j- i+1)

            i = j+1

        return result
```

## Why This Works

The condition:

```python id="mq5j7w"
running_max > suffix_min[j + 1]
```

guarantees there exists at least one smaller value ahead.

That smaller value creates a valid forward jump, which helps maintain connectivity inside the current segment.

As long as such a smaller value exists, the segment can continue expanding.

Once no smaller future value exists, the segment becomes closed.








## Lessons

1. Suffix arrays are extremely useful for future-state queries.

2. Many jump problems can be reduced to interval expansion.

3. Greedy segment grouping can avoid expensive graph traversal.

4. Tracking only the minimum future value is enough to determine whether movement can continue.

5. Always search for monotonic properties before using BFS or DFS.
