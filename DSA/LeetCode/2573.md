# 2573. Find the String with LCP 🔴

**Topics:** String, Dynamic Programming, Greedy, Union-Find

## Intuition

Imagine you have a grid of clues that describe a hidden word. Each clue `lcp[i][j]` tells you: "the suffix starting at position `i` and the suffix starting at position `j` share exactly this many characters at the front." Your job is to reverse-engineer the shortest, alphabetically smallest word that fits all those clues.

Think of it like solving a crossword where you don't see the actual letters, only relationships between positions. If `lcp[0][2] = 2`, then positions 0 and 2 must start identically for 2 characters. If `lcp[0][1] = 0`, positions 0 and 1 must differ right away.

The trick is: you don't need to guess. You can *reconstruct* the word greedily, and then *verify* it by rebuilding the LCP matrix from scratch and comparing.

## Approach

There are two clean phases here:

**Phase 1 (Greedy Assignment):** Go through each position left to right. If a position has no assignment yet, give it the next available letter (starting from `'a'`). Then, for every other position `j`, if `lcp[i][j] > 0` it means positions `i` and `j` must have the same character, so copy the letter over. This gives us the lexicographically smallest possibility because we always try `'a'` first, then `'b'`, and so on.

**Phase 2 (DP Verification):** Now we have a candidate string. We verify it by computing what its LCP matrix *should* look like using a classic DP, and check that it matches the input exactly. The recurrence is elegant: `dp[i][j] = dp[i+1][j+1] + 1` if `word[i] == word[j]`, else `0`. We traverse from the bottom-right to top-left so each cell can use the one diagonally below-right it depends on. If any cell mismatches, the input LCP was invalid and we return `""`.

## Example Walkthrough

Input: `lcp = [[4,0,2,0],[0,3,0,1],[2,0,2,0],[0,1,0,1]]`, n = 4

**Phase 1:**
- `i=0`: unassigned, give it `'a'`. `lcp[0][2]=2 > 0` so position 2 also gets `'a'`
- `i=1`: unassigned, give it `'b'`. `lcp[1][3]=1 > 0` so position 3 also gets `'b'`
- Candidate string: `"abab"`

**Phase 2 (DP verify, bottom-right to top-left):**
- `dp[3][3]` = `n - 3` = `1` (diagonal). `lcp[3][3]=1`. Match.
- `dp[2][2]` = 2, `lcp[2][2]=2`. Match.
- `dp[0][2]`: `word[0]='a'==word[2]='a'` so `dp[0][2] = dp[1][3]+1`. `dp[1][3]`: `word[1]='b'==word[3]='b'` so `dp[1][3] = dp[2][4]+1 = 0+1 = 1`. So `dp[0][2] = 2`. `lcp[0][2]=2`. Match.
- All cells match. Return `"abab"`.

## Code

```python
class Solution:
    def findTheString(self, lcp: List[List[int]]) -> str:
        n = len(lcp)
        word = [''] * n

        # phase 1:- greedy character assignment
        char = 'a'

        for i in range(n):


            if word[i]:
                continue  # already assigned from a prev match

            if char > 'z':
                return ""  # ran out of letterss


            word[i] = char


            # any jth that shares a prefix with ith must have the same char.
            for j in range(i + 1, n):

                if lcp[i][j] > 0:
                    word[j] = char


            char = chr(ord(char) + 1) 


        # phase 2:- verify by reconstructing the lcp with dp
        dp = [[0] * (n + 1) for _ in range(n + 1)]


        # fill bottom-right to top-left
        for i in range(n - 1, -1, -1):

            for j in range(n - 1, -1, -1):

                if word[i] == word[j]:
                    dp[i][j] = dp[i + 1][j + 1] + 1

                else:
                    dp[i][j] = 0


                # our reconstructed value must match the input exactly
                if dp[i][j] != lcp[i][j]:
                    return ""

        return ''.join(word)
            
```

## Complexity

**Time:** $O(n^2)$ — We do two passes over the $n \times n$ matrix: one for greedy assignment and one for DP verification.

**Space:** $O(n^2)$ — The DP table is $n \times n$. The word array is $O(n)$, which is dominated by the table.

## Edge Cases

**1. Diagonal values are wrong.** `lcp[i][i]` must always equal `n - i` (the full suffix length from `i`). If it's anything else, the DP catch will return `""`. See Example 3: `lcp[3][3] = 3` but it should be `1`.

**2. Matrix is not symmetric.** If `lcp[i][j] != lcp[j][i]`, the DP check will catch it. A valid LCP matrix is always symmetric since common prefix length is the same both ways.

**3. All positions share the same character.** Like `"aaaa"` — the greedy phase assigns `'a'` to everything (since `lcp[0][j] > 0` for all `j`), and the DP verifies the strictly decreasing diagonal pattern.

**4. n = 1.** Only one cell: `lcp[0][0]` must be `1`. If it is, return `"a"`. The DP handles this naturally since `dp[0][0] = 1` and `word = ['a']`.

**5. More than 26 distinct groups needed.** If positions never share prefixes and you need more than 26 distinct letters, the greedy phase runs out of alphabet and returns `""`.

## Lessons

The big insight here is the **build then verify** pattern. Rather than trying to construct the answer and validate simultaneously (which gets tangled), you split the problem:

- Greedily build the *cheapest* candidate (alphabetically smallest)
- Independently verify using a DP that checks what the LCP *would* be for your candidate

The **LCP suffix DP recurrence** (`dp[i][j] = dp[i+1][j+1] + 1 if chars match, else 0`) is a classic pattern worth memorizing. It shows up in longest common substring and string comparison problems too.

Finally, "alphabetically smallest" often means **greedy from `'a'`**: assign the earliest possible letter and only introduce new letters when forced. That instinct will serve you on many string construction problems.
