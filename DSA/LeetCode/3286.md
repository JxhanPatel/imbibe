# 3286. Find a Safe Walk Through a Grid 🟡

**Topics:** Graph, Breadth-First Search (BFS), Matrix, 0-1 BFS

## Intuition

Imagine you are trying to reach your destination while walking through a city.

* Some roads are perfectly safe (`0`), so using them costs nothing.
* Other roads are dangerous (`1`), and every time you step on one, you lose one unit of health.

Naturally, you want to choose a route that minimizes the total damage instead of simply minimizing the number of steps.

This is exactly what **0-1 BFS** is designed for. It finds the minimum cost path when every move has a cost of either `0` or `1`.

* Moving onto a safe cell has cost `0`.
* Moving onto an unsafe cell has cost `1`.

Once we know the minimum damage required to reach the destination, we simply check whether our health remains positive.


## Approach

We use **0-1 BFS** with a deque.

1. Create a `damage` matrix where `damage[i][j]` stores the minimum health lost to reach each cell.
2. Initialize every value to infinity.
3. The starting cell contributes `grid[0][0]` damage because we begin on that cell.
4. Push the starting cell into a deque.
5. For every neighboring cell:

   * Calculate the new damage required to reach it.
   * If this path is better than the previously known one, update the damage.
   * If the neighbor is safe (`0`), push it to the **front** of the deque.
   * Otherwise, push it to the **back**.
6. Continue until all possible better paths have been explored.
7. Finally, compare the minimum damage at the destination with the available health.

If the minimum damage is strictly less than the initial health, the walk is possible.


## Example Walkthrough

**Input**

```text
grid =
[
 [0,1,0],
 [0,0,0],
 [1,1,0]
]

health = 1
```

Initially,

```text
damage =
[
 [0,∞,∞],
 [∞,∞,∞],
 [∞,∞,∞]
]
```

Start from `(0,0)`.

### Visit neighbors

Move to `(1,0)`:

* Cell value = `0`
* New damage = `0`

Since the move costs nothing, insert it at the **front** of the deque.

Move to `(0,1)`:

* Cell value = `1`
* New damage = `1`

Since this move costs one health, insert it at the **back**.

The deque always prioritizes safer paths, allowing us to compute the minimum damage efficiently.

Eventually, we reach the destination with

```text
damage = 0
```

Since

```text
0 < 1
```

the answer is

```text
true
```


## Code

```python
from collections import deque
class Solution:
    def findSafeWalk(self, grid: List[List[int]], health: int) -> bool:
        
        rows=len(grid)
        cols=len(grid[0])

        damage = [[float("inf")] * cols for _ in range(rows)]
        damage[0][0] = grid[0][0]


        dq = deque([(0, 0)])
        dirs = [(0,1), (0,-1), (1,0), (-1,0)]


        while dq:
            r, c = dq.popleft()
            cur = damage[r][c]


            for dr, dc in dirs:
                nr= r+dr
                nc= c+dc

                if nr<0 or nr>=rows or nc<0 or nc>=cols:
                    continue

                cost = cur+grid[nr][nc]

                if cost >= damage[nr][nc]:
                    continue

                damage[nr][nc] = cost

                if grid[nr][nc]:
                    dq.append((nr, nc))
                else:
                    dq.appendleft((nr, nc))

        return damage[rows-1][cols-1] < health
```


## Complexity

**Time Complexity**

$$
O(m \times n)
$$

Each cell is processed a constant number of times because every edge has a weight of either `0` or `1`.

**Space Complexity**

$$
O(m \times n)
$$

The `damage` matrix stores the minimum damage for every cell, and the deque may contain up to all cells.


## Edge Cases

### 1. Starting cell is unsafe

```text
grid = [[1]]
health = 1
```

You immediately lose one health, leaving your health at `0`.

Answer:

```text
false
```


### 2. Grid with only safe cells

```text
grid =
[
 [0,0],
 [0,0]
]

health = 1
```

No health is lost throughout the journey.

Answer:

```text
true
```


### 3. Every path is dangerous

```text
grid =
[
 [1,1],
 [1,1]
]

health = 2
```

Every possible route requires too much damage before reaching the destination.

Answer:

```text
false
```


## Lessons

* When edge weights are only `0` or `1`, **0-1 BFS** is more efficient than Dijkstra's algorithm.
* A deque allows zero-cost moves to be processed before one-cost moves.
* Store the best cost seen so far for every cell to avoid revisiting worse paths.
* Many grid problems can be converted into shortest path problems by treating penalties as edge weights.
