# 2144. Minimum Cost of Buying Candies With Discount 🟢

**Topics:** Greedy, Sorting, Array

## Intuition

Imagine you're shopping for candies and the store has a special offer:

> Buy any 2 candies, get 1 candy free.

To maximize your savings, you want the **most expensive possible candy to be free** each time.

A good strategy is to group the candies from **most expensive to least expensive**. In every group of three candies:

* Pay for the two most expensive candies.
* Get the third candy (the cheapest in that group) for free.

This gives the largest possible discount because the free candy is as expensive as possible while still satisfying the rules.

## Approach

We use a greedy strategy:

1. Sort the candies in **descending order**.
2. Process candies in groups of three.
3. In every group:

   * Add the cost of the first candy.
   * Add the cost of the second candy.
   * Skip the third candy because it is free.
4. Continue until all candies are processed.

Why does this work?

* The free candy must be less than or equal to the cheaper of the two purchased candies.
* By sorting in descending order, every third candy naturally satisfies this condition.
* Making every third-largest candy free maximizes the total discount.

## Example Walkthrough

**Input**

```text
cost = [6,5,7,9,2,2]
```

### Step 1: Sort in descending order

```text
[9,7,6,5,2,2]
```

### Step 2: Process groups of three

| Group   | Pay   | Free |
| ------- | ----- | ---- |
| [9,7,6] | 9 + 7 | 6    |
| [5,2,2] | 5 + 2 | 2    |

Total cost:

```text
9 + 7 + 5 + 2 = 23
```

**Output**

```text
23
```

## Code

```python
class Solution:
    def minimumCost(self, cost: List[int]) -> int:

        prices = sorted(cost, reverse=True)

        total_spent= 0

        for idx, price in enumerate(prices):
            if (idx+1)%3 == 0:
                continue

            total_spent+= price

        return total_spent
```

## Complexity

* Sorting: $O(n \log n)$
* Traversal: $O(n)$

Overall:

$$
O(n \log n)
$$

Space complexity:

$$
O(1)
$$

Ignoring the space used by Python's sorting algorithm.

## Edge Cases

1. **Only one candy**

```text
cost = [5]
```

Answer: `5`

2. **Only two candies**

```text
cost = [5, 5]
```

Answer: `10`

3. **Exactly three candies**

```text
cost = [1, 2, 3]
```

Sorted:

```text
[3, 2, 1]
```

Pay for `3` and `2`, get `1` free.

Answer: `5`

4. **All candies have the same cost**

```text
cost = [4,4,4,4,4,4]
```

Two candies become free.

Answer: `16`

## Lessons

* When trying to maximize discounts, sorting is often the first thing to consider.
* Greedy strategies work well when a locally optimal choice leads to a globally optimal result.
* Processing items from largest to smallest is a common trick when selecting elements to skip or discount.
* In grouping problems, sorting can make the optimal grouping obvious and easy to implement.
