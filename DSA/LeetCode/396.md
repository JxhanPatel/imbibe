# 396. Rotate Function 🟡

**Topics:** Array, Math, Dynamic Programming








## Intuition

Think of this problem like a circular conveyor belt with weights placed on it.

Each position has a multiplier (its index), and rotating the belt shifts all weights, changing how much each contributes to the total score.

If you recompute the score from scratch after every rotation, it will be too slow.
Instead, we should notice how the score changes when we rotate once.

The key idea is:
Instead of recalculating everything, we reuse the previous result and adjust it smartly.









## Approach

Let:

* `sum_nums = sum(nums)`
* `F(0)` = initial rotation function

Now observe what happens when we go from `F(k)` to `F(k+1)`:

After rotation, the last element moves to the front, and everything else shifts right.

We derive the relation:

[
F(k+1) = F(k) + sum_nums - n \cdot nums[n - 1 - k]
]

So the plan is:

1. Compute `sum_nums`
2. Compute `F(0)`
3. Iteratively compute next values using the formula
4. Track the maximum

This reduces time complexity from (O(n^2)) to (O(n))








## Example Walkthrough

Input: `nums = [4, 3, 2, 6]`

Step 1:
[
sum = 4 + 3 + 2 + 6 = 15
]

Step 2: Compute `F(0)`
[
F(0) = 0*4 + 1*3 + 2*2 + 3*6 = 25
]

Step 3: Use formula

* (F(1) = 25 + 15 - 4*6 = 16)
* (F(2) = 16 + 15 - 4*2 = 23)
* (F(3) = 23 + 15 - 4*3 = 26)

Max = **26**









## Code

```python
class Solution:
    def maxRotateFunction(self, nums: List[int]) -> int:
        n = len(nums)

        total = sum(nums)

        f = sum(i*v for i, v in enumerate(nums))
        best = f

        for k in range(1, n):
            f+= total - n* nums[-k]

            if f > best:
                best=f

        return best
```









## Complexity

* Time: (O(n))
* Space: (O(1))








## Edge Cases

1. Single element array

   * Example: `[100]` → always returns 0

2. All elements same

   * Rotations do not change value

3. Negative numbers

   * Formula still works correctly







## Lessons

* Look for patterns between consecutive states instead of recomputing
* Rotation problems often have hidden recurrence relations
* Precomputing sum helps reduce repeated work
* Turning (O(n^2)) into (O(n)) is often about reusing previous results
