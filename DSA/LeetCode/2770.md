# 2770. Maximum Number of Jumps to Reach the Last Index 🟡

**Topics:** Dynamic Programming, Array





## Intuition

Think of each index as a stone in a river.

You start on the first stone and want to reach the last one. But you are only allowed to jump to another stone if the height difference between the two stones is within a certain range (`target`).

Now here is the interesting part:

Normally, in jump problems, we try to minimize jumps.
But here, we want the **maximum** number of jumps.

So instead of taking large jumps, we try to take as many valid smaller jumps as possible while still eventually reaching the end.

This naturally suggests trying all possible previous positions and building the best answer step by step.









## Approach

We use Dynamic Programming.

Let:

$$
dp[i]
$$

represent the maximum number of jumps needed to reach index `i`.

### Initialization

* Set every value to `-1` initially because some indices may not be reachable.
* `dp[0] = 0` because we start at index `0` with zero jumps.

### Transition

For every index `i`, check all previous indices `j`:

If:

$$
-target \le nums[i] - nums[j] \le target
$$

and `j` is reachable, then we can jump from `j` to `i`.

So we update:

$$
dp[i] = \max(dp[i], dp[j] + 1)
$$

At the end:

* `dp[n-1]` is our answer.
* If it is still `-1`, then the last index cannot be reached.














## Example Walkthrough

### Input

```python
nums = [1,3,6,4,1,2]
target = 2
```

### Step-by-step

Start:

```python
dp = [0, -1, -1, -1, -1, -1]
```

### i = 1

Check index `0`:

$$
3 - 1 = 2
$$

Valid jump.

```python
dp[1] = max(-1, 0 + 1) = 1
```

Now:

```python
dp = [0, 1, -1, -1, -1, -1]
```

---

### i = 2

Check previous indices:

* From `0`:

$$
6 - 1 = 5
$$

Invalid.

* From `1`:

$$
6 - 3 = 3
$$

Invalid.

So:

```python
dp[2] = -1
```

---

### i = 3

Check:

* From `1`:

$$
4 - 3 = 1
$$

Valid.

```python
dp[3] = dp[1] + 1 = 2
```

Now:

```python
dp = [0, 1, -1, 2, -1, -1]
```

---

### i = 4

Check:

* From `3`:

$$
1 - 4 = -3
$$

Invalid.

* From `1`:

$$
1 - 3 = -2
$$

Valid.

```python
dp[4] = 2
```

---

### i = 5

Check:

* From `3`:

$$
2 - 4 = -2
$$

Valid.

```python
dp[5] = dp[3] + 1 = 3
```

Final:

```python
dp = [0, 1, -1, 2, 2, 3]
```

Answer:

```python
3
```










## Code

```python
class Solution:
    def maximumJumps(self, nums: List[int], target: int) -> int:

        n = len(nums)

        dp = [-1]*n
        dp[0] = 0

        for i in range(1, n):

            for j in range(i):

                if dp[j] == -1:
                    continue

                diff = nums[i] - nums[j]
                
                if -target<=diff<=target:
                    dp[i] = max(dp[i], dp[j]+1)

        return dp[-1]
```








## Complexity

### Time Complexity

$$
O(n^2)
$$

We check every pair `(j, i)`.

### Space Complexity

$$
O(n)
$$

For the DP array.







## Edge Cases

### 1. No valid path

```python
nums = [1, 5, 10]
target = 1
```

No jump is possible.

Answer:

```python
-1
```

---

### 2. Every jump is valid

```python
nums = [1,2,3,4]
target = 100
```

We can jump one step at a time and maximize jumps.

Answer:

```python
3
```

---

### 3. Minimum size array

```python
nums = [1,1]
target = 0
```

Direct jump is valid.

Answer:

```python
1
```









## Lessons

* Dynamic Programming works well when a problem asks for the "best" way to reach positions step by step.
* Define your DP state clearly:

$$
dp[i] = \text{best answer to reach index } i
$$

* Always think carefully about:

  * transition condition
  * unreachable states
  * initialization
* This problem is similar to longest path problems in a DAG because indices only move forward.
