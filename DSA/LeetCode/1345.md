# 1345. Jump Game IV 🔴

**Topics:** Array, Hash Table, Breadth-First Search (BFS)



# Intuition

Imagine each index in the array is a city.

From any city, you can:

* move one step left
* move one step right
* instantly teleport to any city with the same value

Since every move costs exactly **1 step**, this becomes a shortest path problem.

Whenever we need the shortest number of moves in an unweighted graph, **BFS** is usually the best choice.

The tricky part is handling teleportation efficiently.

If we keep checking all equal values again and again, the solution becomes too slow. So after using all indices for a value once, we remove that value from our map.

That single optimization is what makes the solution pass.





# Approach

We use **Breadth-First Search** starting from index `0`.

## Step 1: Build a map

Store all indices for each value.

Example:

```python
{
    100: [0, 4],
    -23: [1, 2],
    404: [3, 9]
}
```

This helps us instantly jump to all matching values.



## Step 2: BFS traversal

For every index:

* try `i - 1`
* try `i + 1`
* try every index with the same value

We keep a `visited` set so we do not revisit nodes.



## Step 3: Optimization

After processing all indices for a value:

```python
graph[arr[i]].clear()
```

This prevents repeated work later.

Without this optimization, the solution can become (O(n^2)).

# Code

```python
from collections import defaultdict, deque

class Solution:
    def minJumps(self, arr: List[int]) -> int:

        n = len(arr)

        if n==1:
            return 0

        value_map = defaultdict(list)

        for i, v in enumerate(arr):
            value_map[v].append(i)

        visited = {0}
        queue = deque([(0, 0)])


        while queue:
            idx, steps = queue.popleft()

            if idx == n-1:
                return steps

            candidates = value_map.pop(arr[idx], [])


            if idx -1 >=0:
                candidates.append(idx-1)

            if idx+1 < n:
                candidates.append(idx +1)


            for nxt in candidates:
                if nxt not in visited:
                    visited.add(nxt)
                    queue.append((nxt, steps+1))



```



# Complexity

## Time Complexity

$$
O(n)
$$

Each index is processed at most once.



## Space Complexity

$$
O(n)
$$


Used for:

* hashmap
* queue
* visited set



# Edge Cases

## 1. Single element array

```python
arr = [7]
```

Answer:

```python
0
```

Already at the destination.



## 2. All elements same

```python
arr = [5,5,5,5]
```

Can jump directly to the end in one move.



## 3. No duplicate values

```python
arr = [1,2,3,4,5]
```

Only left and right movement works.



# Lessons

* BFS is perfect for shortest path problems in unweighted graphs.
* Array indices can often be treated as graph nodes.
* Hash maps can help create teleport-like connections efficiently.
* Clearing processed adjacency lists is a powerful optimization trick.
* Always think about repeated work in graph problems.
