# 3838. Weighted Word Mapping 🟢

**Topics:** Array, String, Simulation


## Intuition

Think of each word as a scorecard.

Every letter contributes some points according to the `weights` array. Once we calculate the total score of a word, we only care about its remainder when divided by `26`.

Now imagine the alphabet written in reverse:

```text
z y x w ... c b a
0 1 2 3 ... 23 24 25
```

If the remainder is:

* `0`, we pick `'z'`
* `1`, we pick `'y'`
* `25`, we pick `'a'`

So the task becomes:

1. Calculate the total weight of each word.
2. Take `total % 26`.
3. Convert that remainder to the corresponding reversed alphabet character.
4. Append it to the answer.

Since every word can be processed independently, a simple simulation is enough.





## Approach

We maintain a string `out` to store our answer.

For every word:

1. Initialize `score = 0`.
2. Traverse each character `c` in the word.
3. Convert the character to its alphabet index:

```python
ord(c) - ord('a')
```

4. Add its corresponding weight to `score`.
5. Compute:

```python
score % 26
```

6. Since the mapping is reversed, we can directly get the character using:

```python
chr(ord('z') - score % 26)
```

7. Append this character to `out`.

After processing all words, return `out`.

## Code

```python
class Solution:
    def mapWordWeights(self, words: List[str], weights: List[int]) -> str:

        out = ""

        for w in words:
            score=0

            for c in w:
                score += weights[ord(c)-ord('a')]

            out += chr(ord('z') - score%26)
            
        return out
```








## Complexity

Let:

* `n` = number of words
* `m` = total number of characters across all words

Time complexity:

$$
O(m)
$$

We visit every character exactly once.

Space complexity:

$$
O(n)
$$

The output string stores one character for each word.







## Edge Cases

1. **Single word**

```text
words = ["abcd"]
```

We calculate its weight once and return a single character.

2. **All weights are the same**

```text
words = ["a", "b", "c"]
weights = [1,1,1,...]
```

Every word gets the same score, so all mapped characters are identical.

3. **Modulo becomes 0**

```text
score = 52

52 % 26 = 0
```

This maps to `'z'`.

4. **Modulo becomes 25**

```text
score = 25

25 % 26 = 25
```

This maps to `'a'`.







## Lessons

* Character to index conversion using `ord(c) - ord('a')` is a very common pattern for lowercase English letters.
* Reverse mappings often do not require an extra array. A simple arithmetic expression with ASCII values can do the job.
* When every element can be processed independently and there are no interactions between them, simulation is usually the cleanest solution.
* Modulo operations are frequently used to wrap values into a fixed range such as `0` to `25`.
