# 3620. Network Recovery Pathways 🔴

**Topics:** Graph, Dynamic Programming, Binary Search, Topological Sort

## Intuition

Imagine you're planning a rescue route through a network of cities.

Every road has two values:

* A **travel cost**, which contributes to your limited budget.
* A **quality score**, where larger values mean safer roads.

Your goal is not simply to minimize the total travel cost. Instead, you want the path whose **worst road is as good as possible**, while keeping the total travel cost within the budget `k`.

For example, consider two valid paths:

```text
Path A: 8 → 7 → 6
Score = 6
```

```text
Path B: 9 → 5 → 9
Score = 5
```

Although Path B contains two excellent roads, its weakest road is only `5`, making Path A the better choice.

The graph is a **Directed Acyclic Graph (DAG)**, which lets us efficiently compute the minimum travel cost using dynamic programming over a topological ordering.


## Approach

The solution combines **binary search** with **dynamic programming on a DAG**.

### Step 1: Build the graph

Create:

* An adjacency list.
* An indegree array for topological sorting.
* A set of all unique edge weights.

The unique edge weights are the only possible answers.


### Step 2: Compute a topological order

Since the graph is a DAG, we can process nodes in topological order.

This guarantees that when processing a node, every path leading to it has already been considered.


### Step 3: Binary search the answer

Suppose we guess that the minimum edge weight should be at least `X`.

For this guess:

* Ignore every edge whose weight is smaller than `X`.
* Ignore every offline intermediate node.
* Compute the minimum total cost from node `0` to node `n-1`.

If this minimum cost is within budget `k`, then `X` is feasible.

Since every smaller threshold would also be feasible, binary search can find the largest possible value.


### Step 4: Feasibility check

Maintain

```text
dist[node] = minimum cost needed to reach this node
```

Initially,

```text
dist[0] = 0
```

All other nodes start as infinity.

Traverse nodes in topological order.

For every valid outgoing edge:

* Skip edges whose weight is smaller than the current threshold.
* Skip offline intermediate nodes.
* Relax the edge.

At the end,

```text
dist[n-1] <= k
```

means the current threshold works.


## Example Walkthrough

**Input**

```text
edges = [
    [0,1,7],
    [1,4,5],
    [0,2,6],
    [2,3,6],
    [3,4,2],
    [2,4,6]
]

online = [True, True, True, False, True]

k = 12
```

Unique edge weights:

```text
[2,5,6,7]
```

### Try threshold = 6

Keep only edges with weight at least `6`.

Remaining graph:

```text
0 → 1 (7)

0 → 2 (6)

2 → 3 (6)

2 → 4 (6)
```

Node `3` is offline, so it cannot be used.

The only valid route is

```text
0 → 2 → 4
```

Total cost

```text
6 + 6 = 12
```

Within budget.

So threshold `6` is feasible.


### Try threshold = 7

Remaining graph:

```text
0 → 1 (7)
```

Node `4` is unreachable.

Threshold `7` is not feasible.

Therefore the answer is

```text
6
```


## Code

```python
from collections import deque
from typing import List

class Solution:
    def findMaxPathScore(self, edges: List[List[int]], online: List[bool], k: int) -> int:


        n=len(online)

        adj = [[] for _ in range(n)]
        indeg = [0]*n
        weights = set()

        for u, v, w in edges:
            adj[u].append((v, w))
            indeg[v] += 1
            weights.add(w)

        order=[]

        q = deque(i for i in range(n) if indeg[i] == 0)

        while q:
            u = q.popleft()
            order.append(u)

            for v, _ in adj[u]:
                indeg[v] -= 1
                if indeg[v] == 0:
                    q.append(v)

        sorted_weights = sorted(weights)
        last = n-1

        def feasible(min_w):
            dist = [float("inf")]*n
            dist[0]=0

            for u in order:
                if dist[u] == float("inf"):
                    continue

                if u not in (0, last) and not online[u]:
                    continue

                for v, w in adj[u]:
                    if w < min_w:
                        continue

                    if v != last and not online[v]:
                        continue

                    new_dist = dist[u] + w
                    if new_dist < dist[v]:
                        dist[v] = new_dist

            return dist[last] <= k

        lo=0
        hi = len(sorted_weights)-1
        best = -1

        while lo <= hi:
            mid = (lo + hi) // 2

            if feasible(sorted_weights[mid]):
                best = sorted_weights[mid]
                lo = mid+1

            else:
                hi = mid-1

        return best
```


## Complexity

Let:

* (n) = number of nodes
* (m) = number of edges
* (U) = number of unique edge weights

**Time Complexity**

Building the graph and topological ordering:

$$
O(n+m)
$$

Each feasibility check processes every node and edge once:

$$
O(n+m)
$$

Binary search performs:

$$
O(\log U)
$$

checks.

Overall:

$$
O((n+m)\log U)
$$


**Space Complexity**

The adjacency list, topological order, indegree array, and distance array require

$$
O(n+m)
$$

space.


## Edge Cases

### 1. No path exists

```text
edges = []

online = [True, True]
```

There is no route from the source to the destination.

Answer:

```text
-1
```


### 2. Offline intermediate node

```text
0 → 1 → 2

online = [True, False, True]
```

Since node `1` is offline, the path is invalid.

Answer:

```text
-1
```


### 3. Budget too small

```text
0 → 1 (5)

1 → 2 (6)

k = 8
```

The only path costs `11`, exceeding the budget.

Answer:

```text
-1
```


### 4. Multiple valid paths

```text
0 → 1 → 3

0 → 2 → 3
```

The algorithm correctly selects the path with the highest possible minimum edge weight while still satisfying the budget.


## Lessons

* Binary search is useful when the answer has a monotonic property.
* Topological ordering allows shortest path computation in a DAG without using Dijkstra's algorithm.
* Converting an optimization problem into repeated feasibility checks is a common competitive programming technique.
* Dynamic programming on DAGs often replaces more expensive shortest path algorithms when cycles are absent.
