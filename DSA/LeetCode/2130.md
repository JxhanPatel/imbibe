# 2130. Maximum Twin Sum of a Linked List 🟡

**Topics:** Linked List, Array, Two Pointers

## Intuition

Imagine you have a line of people standing in a row.

The first person is paired with the last person, the second with the second last, and so on.

If you could instantly access anyone in the line by their position, finding these pairs would be easy. But a linked list does not allow random access. You can only move forward one node at a time.

A simple trick is to first copy all node values into an array. Once the values are stored, accessing the first and last elements becomes straightforward, allowing us to calculate all twin sums efficiently.






## Approach

We use an auxiliary array to store all node values.

### Step 1: Convert linked list to array

Traverse the linked list and append each node's value to `nodes`.

For example:

```text
5 -> 4 -> 2 -> 1

nodes = [5,4,2,1]
```

Now we can access values using indices.



### Step 2: Iterate through the first half

If the list length is `n`, then:

* Node `i`
* Node `n-1-i`

form a twin pair.

We only need to process the first half because every pair is counted once.

For each index:

```text
pair_sum = nodes[i] + nodes[n-1-i]
```

Update the maximum twin sum found so far.



### Why does this work?

The array preserves the order of the linked list.

So:

* `nodes[0]` corresponds to node `0`
* `nodes[n-1]` corresponds to node `n-1`
* `nodes[1]` corresponds to node `1`
* `nodes[n-2]` corresponds to node `n-2`

which are exactly the twin pairs defined in the problem.







## Example Walkthrough

**Input**

```text
head = [5,4,2,1]
```

### Step 1: Build the array

Traverse the linked list:

| Current Node |     Array |
| -----------: | --------: |
|            5 |       [5] |
|            4 |     [5,4] |
|            2 |   [5,4,2] |
|            1 | [5,4,2,1] |




### Step 2: Compute twin sums

Length:

```text
n = 4
```

Twin pairs:

|  i | Twin Index |  Pair | Sum |
| -: | ---------: | ----: | --: |
|  0 |          3 | 5 + 1 |   6 |
|  1 |          2 | 4 + 2 |   6 |

Maximum:

```text
6
```




## Code

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def pairSum(self, head: Optional[ListNode]) -> int:

        nodes=[]
        curr=head

        while curr:
            nodes.append(curr.val)
            curr = curr.next

        max_pair_sum = 0
        n = len(nodes)
        
        for i in range(n//2):
            pair_sum = nodes[i] + nodes[n-1-i]
            
            if pair_sum > max_pair_sum:
                max_pair_sum = pair_sum

        return max_pair_sum
```






## Complexity

Traversing the linked list:

$$
O(n)
$$

Computing twin sums:

$$
O(n)
$$

Overall:

$$
O(n)
$$

Space complexity:

$$
O(n)
$$

because we store all node values in an array.





## Edge Cases

### 1. Minimum size linked list

```text
head = [1,100000]
```

Twin pair:

```text
1 + 100000 = 100001
```

Answer:

```text
100001
```






### 2. All nodes have the same value

```text
head = [7,7,7,7]
```

Twin sums:

```text
7 + 7 = 14
7 + 7 = 14
```

Answer:

```text
14
```




### 3. Maximum twin sum is not the first pair

```text
head = [4,2,2,3]
```

Twin sums:

```text
4 + 3 = 7
2 + 2 = 4
```

Maximum:

```text
7
```




## Lessons

* Converting a linked list to an array is often a good idea when random access is needed.
* The twin of index `i` can be found using the formula:

$$
\text{twin index} = n - 1 - i
$$

* When elements are paired symmetrically from both ends, iterating only until `n // 2` avoids duplicate work.
* Sometimes using extra space leads to a much simpler and more readable solution.
