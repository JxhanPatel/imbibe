# 2906. Construct Product Matrix 🟡
Topics: Array, Matrix, Prefix Sum

## Intuition

Imagine you're at a bake sale and you want to know, for each item on the table, "what's the product of the prices of *all other items*?" You could grab every item, compute the product, then put that item back and repeat, but that's exhausting. A smarter baker does it in two passes: first, they walk left to right keeping a running tally of everything *before* each item. Then they walk right to left keeping a tally of everything *after* each item. Multiply those two tallies together at each spot and you're done. That's exactly what we're doing here, but for a 2D grid we treat as a flat list.

## Approach: Flatten + Prefix-Suffix Product

The key insight is that a 2D grid is just a 1D array in disguise. Once we flatten it (or treat it as one), we apply the classic "product of all elements except self" trick:

1. **Flatten** the `n x m` grid into a 1D view (logically, no need to create a copy).
2. **Left-to-right pass**: for each position `i`, store the product of everything to its *left* into the output array `p[i]`. At the start the running product is `1`.
3. **Right-to-left pass**: maintain a running product of everything to the *right*. At each position, multiply `p[i]` by this suffix product. Now `p[i]` holds (left product) x (right product) = product of everything except itself.
4. Take `% 12345` at every multiplication step so numbers never blow up.

The reason we apply mod at *every step* (not just at the end) is that grid values can be up to $10^9$ and there can be up to $10^5$ elements. Without modding early, intermediate products would require arbitrary precision integers and slow everything down.

## Example Walkthrough

Input: `grid = [[1,2],[3,4]]`, flattened to `flat = [1, 2, 3, 4]`

**Prefix pass** (running product starts at 1):
```
i=0: p[0] = 1          (nothing to the left)
i=1: p[1] = 1*1 = 1    (only grid[0] to the left)
i=2: p[2] = 1*2 = 2    (grid[0]*grid[1])
i=3: p[3] = 2*3 = 6    (grid[0]*grid[1]*grid[2])
p = [1, 1, 2, 6]
```

**Suffix pass** (running suffix starts at 1):
```
i=3: p[3] = 6 * 1 = 6,   suffix = 1*4 = 4
i=2: p[2] = 2 * 4 = 8,   suffix = 4*3 = 12
i=1: p[1] = 1 * 12 = 12, suffix = 12*2 = 24
i=0: p[0] = 1 * 24 = 24, suffix = 24*1 = 24
p = [24, 12, 8, 6]
```

Reshape back to 2x2: `[[24,12],[8,6]]`. Matches expected output!

## Code

```python
class Solution:
    def constructProductMatrix(self, grid: List[List[int]]) -> List[List[int]]:
        MOD = 12345
        n, m = len(grid), len(grid[0])
        total = n * m

        # we'll work with a flat output array, then reshape at the end
        p = [0] * total


        # <--- left-to-right prefix pass <---
        prefix = 1
        for i in range(total):
            row, col = divmod(i, m)          # convert flat index to 2d coords
            p[i] = prefix
            prefix = (prefix * grid[row][col]) % MOD   



        # ---> right-to-left suffix pass <---
        suffix = 1
        for i in range(total - 1, -1, -1):
            row, col = divmod(i, m)
            p[i] = (p[i] * suffix) % MOD  
            suffix = (suffix * grid[row][col]) % MOD 


        # reshape flat array back into n x m grid
        return [p[i*m:(i+1)*m] for i in range(n)]
            
```

## Complexity

**Time:** $O(n \times m)$ — we make exactly two linear passes over all elements.

**Space:** $O(n \times m)$ — for the output matrix `p`. No extra arrays beyond the answer itself, which is required by the problem.

## Edge Cases

**1. Element equals 12345 (or a multiple of it)**
Any element that is a multiple of 12345 will make its contribution to the product `0 mod 12345`. So if *any* element is divisible by 12345, every other position's result will be 0, except the position of that element itself (which will contain the product of everything else). Example 2 in the problem shows this with the value `12345`.

**2. All elements are 1**
Every product would be `1 % 12345 = 1`. The output should be an all-ones matrix. This is a great sanity check for your implementation.

**3. Single row or single column (degenerate 2D grid)**
Since the constraint says `n * m >= 2`, a grid like `[[5, 7]]` (1x2) or `[[3],[9]]` (2x1) is valid. The flattening approach handles these naturally since everything is treated as one long sequence.

**4. Large values near $10^9$**
Without applying `% MOD` at each step, the prefix product could balloon astronomically before we ever finish the pass. Always mod *during* the accumulation, not just at assignment.

## Lessons

**The prefix-suffix pattern is a workhorse.** Whenever a problem asks for "everything except me," think two-pass: left gives you the left context, right gives you the right context. Their product at each index is your answer. This is the same technique as LeetCode 238 (Product of Array Except Self), just extended to 2D by flattening.

**2D grids are often just disguised 1D arrays.** Don't let the matrix structure intimidate you. A simple `divmod(i, m)` converts any flat index back to `(row, col)` and lets you process the grid linearly.

**Mod early, mod often.** When the problem says "take mod K", that's not just for the final answer. It's your hint to mod at *every multiplication*, otherwise intermediate values overflow or slow down.

**Don't create extra arrays when you don't need to.** We reused the output array `p` to store prefix products, then multiplied in the suffix in place. That keeps space to the minimum required.
