# 3629. Minimum Jumps to Reach End via Prime Teleportation 🟡

**Topics:** Breadth-First Search (BFS), Graph, Hash Map, Number Theory, Sieve






# Intuition

Think of the array like a city map.

Normally, you can walk only to neighboring buildings using adjacent moves. But some buildings contain a *prime-powered portal*. If you stand on a prime number `p`, you instantly gain access to every building whose value is divisible by `p`.

The challenge is to reach the last index using the fewest moves possible.

Since every move costs exactly `1`, this becomes a shortest path problem. Whenever we need the minimum number of moves in an unweighted graph, **BFS** is usually the best tool.

The tricky part is efficiently finding all indices divisible by a prime `p`. Scanning the whole array every time would be too slow. So we preprocess:

* Factor every number into primes
* Store which indices are divisible by each prime

This allows teleportation in near constant lookup time.






# Approach

We use **Breadth-First Search** starting from index `0`.

## Step 1: Build prime-to-indices mapping

For every number:

* Find its prime factors
* For each prime factor `p`, store the index in:

```python
prime_to_indices[p]
```

Example:

```python
nums = [2, 4, 6]
```

* `2` contributes to prime `2`
* `4` contributes to prime `2`
* `6` contributes to primes `2` and `3`

So:

```python
2 -> [0,1,2]
3 -> [2]
```

---

## Step 2: BFS traversal

From each index:

### Adjacent moves

We can go to:

* `i - 1`
* `i + 1`

if inside bounds.

### Prime teleportation

If `nums[i]` itself is prime:

* teleport to all indices divisible by that prime

To avoid repeating expensive teleport searches:

* once we use a prime `p`,
* clear its list or mark it as used.

This prevents revisiting the same teleport edges again and again.








# Example Walkthrough

```python
nums = [1,2,4,6]
```

### Prime mapping

```python
2 -> [1,2,3]
3 -> [3]
```

---

### BFS

Start:

```python
queue = [0]
steps = 0
```

---

### From index 0

Adjacent:

* go to index `1`

Queue:

```python
[1]
```

Steps = 1

---

### From index 1

`nums[1] = 2` which is prime.

Teleport using prime `2`:

* indices `[1,2,3]`

We reach index `3`.

Steps = 2

Answer:

```python
2
```









# Code

```python
from collections import defaultdict, deque
from math import isqrt


class Solution:
    def minJumps(self, nums: List[int]) -> int:

        n =len(nums)
        if n==1:
            return 0

        def get_prime_factors(x):
            factors = set()
            d=2

            while d*d <= x:
                if x%d == 0:
                    factors.add(d)
                    while x%d ==0:
                        x//= d
                d+= 1


            if x>1:
                factors.add(x)
            return factors


        def is_prime(x):
            if x < 2:
                return False

            if x == 2:
                return True

            if x%2 == 0:
                return False

            for d in range(3, isqrt(x)+1, 2):
                if x % d == 0:
                    return False

            return True

        prime_map = defaultdict(list)

        for idx, val in enumerate(nums):

            for p in get_prime_factors(val):
                prime_map[p].append(idx)

        visited = {0}
        exhausted_primes = set()
        q=deque([0])
        steps=0



        while q:
            for _ in range(len(q)):
                cur = q.popleft()


                if cur == n-1:
                    return steps


                for nb in [cur-1, cur+1]:

                    if 0 <= nb < n and nb not in visited:
                        visited.add(nb)
                        q.append(nb)

                val = nums[cur]
                if not is_prime(val) or val in exhausted_primes:
                    continue

                exhausted_primes.add(val)
                
                for nb in prime_map[val]:

                    if nb not in visited:
                        visited.add(nb)
                        q.append(nb)

            steps+= 1

        return -1
```








# Complexity

## Time Complexity

Factorization for all numbers:

[
O(n \sqrt{m})
]

where:

* (n) = array length
* (m) = maximum value in `nums`

BFS traversal is linear overall.


## Space Complexity

[
O(n)
]

for BFS queue, visited set, and mappings.









# Edge Cases

### 1. Single element array

```python
nums = [7]
```

Already at destination.

Answer:

```python
0
```



### 2. No prime numbers

```python
nums = [4,6,8,10]
```

No teleportation possible.

Only adjacent movement works.



### 3. Large repeated prime teleport

```python
nums = [2,4,6,8,10]
```

Using prime `2` should happen only once.

Otherwise BFS becomes inefficient.

---

### 4. Prime exists but no useful teleport

```python
nums = [5,7,11]
```

Each prime only connects to itself.

Adjacent moves still solve the problem.



# Lessons

### 1. BFS is perfect for minimum moves

Whenever every move costs the same, think about BFS first.



### 2. Preprocessing saves time

Instead of scanning the whole array during teleportation:

* preprocess divisibility relationships once.



### 3. Remove repeated work

The `used_prime` trick is very important.

Without it, the same teleport edges would be explored many times.



### 4. Graph problems are often hidden inside arrays

Even though the input is an array, each index behaves like a graph node with edges:

* adjacent edges
* teleport edges

Recognizing this transformation is the key insight.
