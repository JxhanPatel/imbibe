# 3756. Concatenate Non-Zero Digits and Multiply by Sum II 🟡

**Topics:** String, Prefix Sum, Binary Search

## Intuition

Imagine you have a very long string of digits, and many people keep asking questions about different sections of it.

For each section, you need to:

1. Ignore every `0`.
2. Join the remaining digits together to form a new number.
3. Find the sum of those digits.
4. Multiply the formed number by the digit sum.

If we scan every substring from scratch, each query could take linear time, making the overall solution far too slow for up to `10^5` queries.

Instead, we preprocess all **non-zero digits** once. This lets us answer every query quickly using binary search and prefix arrays.

The key observation is that zeros never contribute to either the concatenated number or the digit sum, so we only need to store the non-zero digits.


## Approach

We first collect every non-zero digit along with its original position.

For example,

```text
s = "10203004"

Positions: 0 2 4 7
Digits:    1 2 3 4
```

Next, we build three prefix arrays.

### Prefix digit sum

Stores the cumulative sum of non-zero digits.

```text
Digits:      1 2 3 4
Prefix Sum:  1 3 6 10
```

This allows us to compute the digit sum of any query in constant time.


### Prefix concatenated number

We gradually build the concatenated number modulo `10^9 + 7`.

```
1
12
123
1234
```

using

```python
num_pref[i + 1] = (num_pref[i] * 10 + digit) % MOD
```


### Powers of 10

We precompute

```text
10^0, 10^1, 10^2, ...
```

so we can efficiently extract any portion of the concatenated number.


### Processing a query

For each query `[l, r]`:

1. Use binary search to locate the first and last non-zero digits inside the range.
2. If no non-zero digits exist, return `0`.
3. Compute the digit sum using the prefix sums.
4. Extract the required concatenated number using the prefix numbers and powers of `10`.
5. Return

```text
(number × digitSum) mod MOD
```

Since binary search is logarithmic and all remaining operations are constant time, each query is processed efficiently.

## Code

```python
from bisect import bisect_left, bisect_right
from typing import List
class Solution:
    def sumAndMultiply(self, s: str, queries: List[List[int]]) -> List[int]:
        
        MOD = 1000000007

        idxs=[]
        digs=[]

        for i, ch in enumerate(s):
            if ch != '0':
                idxs.append(i)
                digs.append(int(ch))

        n=len(digs)
        
        if not n:
            return [0] * len(queries)

        sum_pref = [0] * (n+1)
        num_pref = [0] * (n+1)
        pow10 = [1]*(n + 1)

        for i, d in enumerate(digs):
            sum_pref[i+1] = sum_pref[i] + d
            num_pref[i+1] = (num_pref[i] * 10+d)%MOD
            pow10[i+1] = pow10[i]*10 % MOD


        res=[]

        for l, r in queries:
            lo = bisect_left(idxs, l)
            hi = bisect_right(idxs, r)
            
            if lo==hi:
                res.append(0)
                continue

            cnt = hi-lo
            dsum = sum_pref[hi] - sum_pref[lo]
            num = (num_pref[hi] - num_pref[lo] * pow10[cnt]) % MOD
            res.append(num * dsum % MOD)

        return res
```


## Complexity

**Preprocessing**

* Collect non-zero digits:

$$
O(n)
$$

* Build prefix arrays:

$$
O(n)
$$

where `n` is the number of non-zero digits.


**Each Query**

* Two binary searches:

$$
O(\log n)
$$

* Remaining computations:

$$
O(1)
$$

Overall complexity:

$$
O(m + q \log n)
$$

where:

* `m` is the length of the original string.
* `n` is the number of non-zero digits (`n ≤ m`).
* `q` is the number of queries.


**Space Complexity**

$$
O(n)
$$

We store the non-zero digits, their positions, and three prefix arrays.


## Edge Cases

### 1. Entire string contains only zeros

```text
s = "000000"

queries = [[0,5]]
```

No non-zero digits exist.

Answer:

```text
0
```


### 2. Query contains only zeros

```text
s = "102030"

query = [1,1]
```

Substring:

```text
0
```

No digits remain after removing zeros.

Answer:

```text
0
```


### 3. Only one non-zero digit

```text
s = "0005000"

query = [0,6]
```

Concatenated number:

```text
5
```

Digit sum:

```text
5
```

Answer:

```text
25
```


## Lessons

* Ignore unnecessary data early to simplify later computations.
* Prefix sums are useful for answering repeated range-sum queries efficiently.
* Prefix numbers combined with powers of `10` allow fast extraction of concatenated values.
* Binary search is an excellent tool when repeatedly querying ranges over sorted positions.
* Preprocessing often transforms a slow per-query solution into an efficient one suitable for large constraints.
