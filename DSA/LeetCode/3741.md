
# 3741. Minimum Distance Between Three Equal Elements II 🟡
**Topics:** Array, Hash Map, Greedy





## Intuition

Imagine you're looking at a row of city billboards, and you need to find three billboards with the same company logo. The city charges you based on total "travel distance" between those three billboards. Your job: find the cheapest combo.

Now here's the neat insight. For three indices `i < j < k`, the distance formula simplifies beautifully:

> `abs(i-j) + abs(j-k) + abs(k-i)` = `2 * (k - i)`

Because the middle element cancels out! The total distance is just **twice the span from the first to the last occurrence**. So we don't need to worry about `j` at all for minimizing... except we need at least 3 billboards. That means: for every value that appears 3+ times, just look at all consecutive triplets of its indices and take the smallest span.




## Approach

1. Use a hash map to collect all indices where each number appears.
2. For any number that appears at least 3 times, slide a window of size 3 over its sorted list of indices.
3. For each window `[i, j, k]`, compute `2 * (k - i)`.
4. Track the global minimum




## Example Walkthrough

**Input:** `nums = [1, 2, 1, 1, 3]`

1. Build index map: `{1: [0, 2, 3], 2: [1], 3: [4]}`
2. Only `1` has 3+ indices. Slide over `[0, 2, 3]`:
   - Window `(0, 2, 3)` → distance = `2 * (3 - 0)` = **6**
3. No other valid windows. Answer: **6** ✅





## Code

```python
from collections import defaultdict

def minimumDistance(nums):
    # group each value's indices together
    index_map = defaultdict(list)
    for i, v in enumerate(nums):
        index_map[v].append(i)

    min_dist = float('inf')

    for indices in index_map.values():
        # need at least 3 occurrences to form a good tuple
        if len(indices) < 3:
            continue

        # slide a window of 3 through sorted indices
        for i in range(len(indices) - 2):
            first = indices[i]
            last = indices[i + 2]

            # distance formula simplifies to 2*(last - first)
            dist = 2 * (last - first)
            min_dist = min(min_dist, dist)

    return min_dist if min_dist != float('inf') else -1
```






## Complexity

**Time:** $O(n)$ — one pass to build the map, then we iterate over all indices (total across all values is still $n$).

**Space:** $O(n)$ — storing all indices in the hash map.





## Edge Cases

- **No value appears 3+ times** (e.g., `[1, 2, 3]`) → return `-1`. The `float('inf')` check handles this.
- **All elements the same** (e.g., `[2, 2, 2, 2]`) → many windows, but the minimum will be the first consecutive triplet `(0, 1, 2)` → distance = `2 * 2 = 4`.
- **Array length exactly 1 or 2** → impossible to form any triplet, return `-1` immediately (handled by the `< 3` check).
- **One huge burst then a gap** (e.g., `[5, 5, 5, 1, 1, 1, 1, 5]`) → the window of 3 with the smallest span wins, not just the first one.






## Lessons

The key tricks here are worth internalizing for other problems too:

**1. The distance formula collapses.** `abs(i-j) + abs(j-k) + abs(k-i)` where `i < j < k` always equals `2*(k-i)`. The middle index doesn't matter for the cost, only the span from first to last. Whenever you see absolute difference sums over sorted indices, try simplifying like this.

**2. Grouping by value is a classic pattern.** Anytime a problem asks "find three (or k) indices with the same value," your first instinct should be: build a map from value to list of indices, then operate on those lists.

**3. Sliding window of fixed size.** Once you have sorted index lists, a window of size 3 gliding forward is all you need. This is O(n) total work, much cheaper than brute-force O(n³).
