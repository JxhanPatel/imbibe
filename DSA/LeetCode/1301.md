# 1301. Number of Paths with Max Score 🔴

**Topics:** Dynamic Programming, Matrix

## Intuition

Imagine you are hiking from the bottom-right corner of a mountain to the top-left.

Each cell contains some treasure (digits), while some cells are blocked (`X`). Your goal is not only to reach the destination, but also to collect as much treasure as possible.

Now suppose many different routes exist.

* Some routes collect less treasure.
* Some collect the maximum treasure.

So for every cell, we need to know two things:

1. **What is the maximum score I can collect from here to the end?**
2. **How many different ways achieve that maximum score?**

Instead of exploring every possible path repeatedly, we compute this information once for every cell using dynamic programming.


## Approach

Let

* `score[r][c]` = maximum score obtainable from `(r, c)` to `S`
* `ways[r][c]` = number of paths that achieve `score[r][c]`

Since movement is only:

* down
* right
* down-right

when thinking from `E` to `S`, we can process the board **backwards** from the bottom-right toward the top-left.

### Base case

At `S`:

```text
score = 0
ways = 1
```

No digits are collected at `S`.


### Transition

For every non-obstacle cell, check the three cells we can move to:

* `(r+1, c)`
* `(r, c+1)`
* `(r+1, c+1)`

Among all reachable neighbors:

* Choose the largest score.
* Sum the number of ways from every neighbor that has this largest score.

Finally, if the current cell contains a digit, add it to the score.

Do **not** add anything for `E` or `S`.


## Example Walkthrough

**Input**

```text
board =
[
"E23",
"2X2",
"12S"
]
```

Start from `S`.

### Step 1

```text
S

score = 0
ways = 1
```

### Step 2

Process nearby cells.

Cell `2` above `S`

```text
score = 2
ways = 1
```

Cell `2` left of `S`

```text
score = 2
ways = 1
```

### Step 3

Continue filling the table.

The obstacle is skipped.

Eventually,

```text
score[0][0] = 7
ways[0][0] = 1
```

So the answer is

```text
[7,1]
```


## Code

```python
from typing import List
class Solution:
    def pathsWithMaxScore(self, board: List[str]) -> List[int]:

        MOD = 1000000007
        n = len(board)

        score = [[-1]*n for _ in range(n)]

        ways = [[0]*n for _ in range(n)]

        score[n-1][n-1] = 0
        ways[n-1][n-1] = 1

        for r in range(n-1, -1, -1):
            for c in range(n-1, -1, -1):

                if (r == n-1 and c == n-1) or board[r][c] == "X":
                    continue

                best = -1
                count=0

                for nr, nc in ((r+1, c), (r, c+1), (r + 1, c + 1)):
                    if nr >= n or nc >= n:
                        continue

                    if score[nr][nc] == -1:
                        continue

                    if score[nr][nc] > best:
                        best = score[nr][nc]
                        count = ways[nr][nc]

                    elif score[nr][nc]==best:
                        count = (count + ways[nr][nc]) % MOD

                if best == -1:
                    continue

                value=0
                if board[r][c].isdigit():
                    value = int(board[r][c])

                score[r][c] = best + value
                ways[r][c] = count

        if ways[0][0] == 0:
            return [0, 0]

        return [score[0][0], ways[0][0]]
```


## Complexity

**Time Complexity**

For every cell, we inspect at most three neighbors.

$$
O(n^2)
$$


**Space Complexity**

Two DP tables of size (n \times n).

$$
O(n^2)
$$


## Edge Cases

1. **No valid path**

```text
[
"E11",
"XXX",
"11S"
]
```

Answer:

```text
[0,0]
```


2. **Multiple optimal paths**

```text
[
"E12",
"1X1",
"21S"
]
```

Both optimal paths collect the same maximum score.

Answer:

```text
[4,2]
```


3. **Single available route**

```text
[
"E23",
"2X2",
"12S"
]
```

Only one path achieves the maximum score.

Answer:

```text
[7,1]
```


## Lessons

* Sometimes one DP state is not enough. Here we store both the **best score** and the **number of ways** to achieve it.
* When counting optimal paths, first determine the maximum value, then accumulate paths only from neighbors that achieve that maximum.
* Reverse thinking can simplify movement constraints. Instead of moving from `S` to `E`, process the DP from `S` backward to `E`.
* Initializing unreachable states with `-1` makes it easy to distinguish valid and invalid paths during transitions.
