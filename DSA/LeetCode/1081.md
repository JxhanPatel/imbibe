# 1081. Smallest Subsequence of Distinct Characters 🟡

**Topics:** String, Stack, Greedy

## Intuition

Imagine you're building the smallest possible word using a set of letter tiles.

Each letter is allowed to appear **only once**, but the letters arrive one by one. Whenever a new letter arrives, you have two choices:

* Keep the letters you've already chosen.
* Or, if the new letter is alphabetically smaller, remove some larger letters from the end and replace them, **only if those removed letters will appear again later**.

For example, if you've built `"bc"` and then see `"a"`, it makes sense to remove both `b` and `c` because you'll encounter them again later. This gives a better starting point `"a"` for the final answer.

This naturally suggests using a **stack**, since we only need to modify the end of our current subsequence.


## Approach

We process the string from left to right while maintaining three data structures:

* `last_count`: stores how many occurrences of each character are still left to process.
* `result_stack`: stores the current smallest subsequence.
* `added`: keeps track of characters already present in the stack to avoid duplicates.

For every character:

1. Decrease its remaining count.
2. If it is already in the stack, skip it.
3. Otherwise:

   * While the stack is not empty,
   * the current character is lexicographically smaller than the stack's top,
   * and the top character will appear again later,
     remove the top character from the stack.
4. Add the current character to the stack and mark it as added.

At the end, joining the stack gives the lexicographically smallest subsequence containing every distinct character exactly once.


## Example Walkthrough

**Input**

```text
s = "cbacdcbc"
```

Initial counts:

```text
c:4
b:2
a:1
d:1
```

| Current | Stack | Action                                          |
| ------- | ----- | ----------------------------------------------- |
| c       | c     | Add                                             |
| b       | b     | Remove `c`, then add `b`                        |
| a       | a     | Remove `b`, then add `a`                        |
| c       | ac    | Add                                             |
| d       | acd   | Add                                             |
| c       | acd   | Already added, skip                             |
| b       | acdb  | Cannot remove `d` because it won't appear again |
| c       | acdb  | Already added, skip                             |

Final answer:

```text
"acdb"
```


## Code

```python
from collections import Counter

class Solution:
    def smallestSubsequence(self, s: str) -> str:
        
        last_count = Counter(s)
        result_stack=[]
        added=set()

        for ch in s:
            last_count[ch] -= 1

            if ch in added:
                continue

            while result_stack and result_stack[-1] > ch and last_count[result_stack[-1]]>0:
                removed = result_stack.pop()
                added.discard(removed)

            result_stack.append(ch)
            added.add(ch)

        return "".join(result_stack)
```


## Complexity

**Time Complexity**

$$
O(n)
$$

Each character is pushed onto the stack at most once and popped at most once.

**Space Complexity**

$$
O(n)
$$

The stack, set, and frequency counter together use linear space.


## Edge Cases

### 1. All characters are distinct

```text
s = "abcd"
```

Output:

```text
"abcd"
```

No characters need to be removed.


### 2. All characters are identical

```text
s = "aaaa"
```

Output:

```text
"a"
```

Only one occurrence is kept.


### 3. A larger character cannot be removed

```text
s = "bac"
```

Output:

```text
"bac"
```

Although `b` is larger than `a`, it does not appear again later, so removing it would make it impossible to include every distinct character.


## Lessons

* A monotonic stack is useful for constructing the lexicographically smallest sequence.
* Before removing an element greedily, always verify that it appears again later.
* A frequency counter helps determine whether a character can be safely removed.
* A set allows duplicate checks in constant time.
