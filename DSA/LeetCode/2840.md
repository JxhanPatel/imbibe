# 2840. Check if Strings Can be Made Equal With Operations II 🟡

**Topics:** Hash Table, String, Sorting

## Intuition

Think of the string like a train with two separate tracks: one track carries characters at **even-indexed** positions (0, 2, 4...), and the other carries characters at **odd-indexed** positions (1, 3, 5...).

The rule says you can only swap two characters if the difference in their positions is **even**. That means a character at position 0 can swap with position 2, 4, 6... but never with position 1, 3, 5. In other words, characters are "stuck on their track" and can only shuffle around within it.

So the key insight is: if two strings can become equal, their even-indexed characters must be rearrangeable into each other, and same for odd-indexed characters. If the **multisets** (character counts) match for each parity group separately, we're good!

## Approach

We split both strings into two buckets each: one for even indices, one for odd indices. Then we compare character frequencies in each bucket.

Instead of `Counter`, we use a plain dictionary and build the frequency map ourselves. Same idea, just more hands-on. If both parity groups have matching frequencies between `s1` and `s2`, return `True`.

## Example Walkthrough

Let's trace through: `s1 = "abcdba"`, `s2 = "cabdab"`

```
indices:  0  1  2  3  4  5

s1:       a  b  c  d  b  a
s2:       c  a  b  d  a  b

s1 even positions (0,2,4): a, c, b  ->  {a:1, c:1, b:1}
s2 even positions (0,2,4): c, b, a  ->  {c:1, b:1, a:1}  match

s1 odd positions  (1,3,5): b, d, a  ->  {b:1, d:1, a:1}
s2 odd positions  (1,3,5): a, d, b  ->  {a:1, d:1, b:1}  match

Both groups match -> return True
```

## Code

```python
class Solution:
    def checkStrings(self, s1: str, s2: str) -> bool:

        def freq(chars):
            counts = {}

            for ch in chars:
                counts[ch] = counts.get(ch, 0) + 1

            return counts



        # pull out even and odd indexed chars from each string
        s1_even = freq(s1[i] for i in range(0, len(s1), 2))
        s1_odd  = freq(s1[i] for i in range(1, len(s1), 2))

        s2_even = freq(s2[i] for i in range(0, len(s2), 2))
        s2_odd  = freq(s2[i] for i in range(1, len(s2), 2))


        return s1_even == s2_even and s1_odd == s2_odd
```

The `dict` comparison with `==` in Python already checks that both keys and values match exactly, so no extra logic needed there.

## Complexity

**Time Complexity:** $O(n)$
One pass through each string to build four dictionaries, each with at most 26 keys.

**Space Complexity:** $O(1)$
Dictionaries are bounded by the alphabet size (26 lowercase letters), so space stays constant.

## Edge Cases

1. **All identical characters** (`s1 = "aaaa"`, `s2 = "aaaa"`): Trivially returns `True`. All freq maps are identical.

2. **Same characters but wrong parity** (`s1 = "ab"`, `s2 = "ba"`): `s1_even = {a:1}`, `s2_even = {b:1}`. These don't match, so `False`. The only possible swap would need indices 0 and 1, but their difference is 1 (odd), which is not allowed.

3. **Length 1** (`s1 = "a"`, `s2 = "b"`): No swaps possible at all. Even-track dicts immediately differ. Returns `False`.

4. **Large `n` with many repeated characters**: Works fine since the dict only ever holds 26 keys max regardless of input size.

## Lessons

**Pattern: Invariant-based thinking.** When operations have restrictions, ask "what does this operation preserve?" Here, every allowed swap keeps characters on their parity track. That invariant is the whole key to the solution.

**Pattern: Frequency (multiset) equality.** Any time a problem is about rearranging characters within a group, a frequency dictionary is your best friend. Two groups are interconvertible if and only if their counts match exactly.

**Pattern: Decompose into independent subproblems.** The even and odd tracks don't interact at all. Solve each independently and `and` the results together. This "partition and compare" strategy shows up constantly in string and array problems.
