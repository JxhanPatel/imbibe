# 3020. Find the Maximum Number of Elements in Subset 🟡

**Topics:** Hash Table, Greedy, Math

## Intuition

Think of building a symmetric tower.

Starting from some number `x`, every next level must be the square of the previous one:

```text
x → x² → x⁴ → x⁸ → ...
```

To make the tower perfectly symmetric, every value except the middle must appear **twice**.

For example:

```text
[2, 4, 16, 4, 2]
```

* `2` appears twice.
* `4` appears twice.
* `16` appears once because it is the center.

This means:

* Every intermediate number needs at least **two copies**.
* The chain can end only when we reach a value that appears exactly **once**, which becomes the center.

The only exception is `1`.

Since:

```text
1² = 1
```

the chain never grows. We simply use the largest **odd** number of `1`s because the final sequence must have odd length.


## Approach

We first count the frequency of every number using a hash map.

### Step 1: Handle the number `1`

Since squaring `1` always gives `1`, the best sequence consists only of `1`s.

* If its frequency is odd, use all of them.
* Otherwise, discard one to make the length odd.

Example:

```text
Frequency = 6 → Answer = 5
Frequency = 5 → Answer = 5
```


### Step 2: Try every number greater than `1`

Treat each distinct number as the starting point of a chain.

For every current value:

* If it appears at least twice, we can place one copy on the left and one on the right.
* Increase the answer by `2`.
* Move to its square.

```text
2 → 4 → 16 → 256
```

Continue until the chain cannot be extended.


### Step 3: Decide how the chain ends

There are two possibilities.

**Case 1: One copy exists**

```text
2,2
4,4
16
```

The single `16` becomes the center.

Sequence:

```text
[2,4,16,4,2]
```


**Case 2: No copy exists**

```text
2,2
4,4
16 does not exist
```

The last pair cannot form the center.

So we remove one level from the answer.


We repeat this process for every distinct starting value and keep the maximum length.

## Code

```python
from collections import Counter
from typing import List

class Solution:
    def maximumLength(self, nums: List[int]) -> int:

        freq = Counter(nums)
        ans=1


        if 1 in freq:
            ans = freq[1] if freq[1]%2 else freq[1]-1

        for start in freq:
            if start==1:
                continue


            length=0
            cur=start

            while freq[cur] >= 2:
                length+= 2
                cur *= cur

            if freq[cur] == 1:
                length += 1

            elif length>0:
                length -= 1

            ans = max(ans, length)

        return ans
```


## Complexity

**Time Complexity**

Building the frequency map takes:

$$
O(n)
$$

Each squaring chain is extremely short because values grow exponentially after every square.

Overall:

$$
O(n)
$$


**Space Complexity**

$$
O(n)
$$

for storing the frequency map.


## Edge Cases

### 1. All ones

```text
nums = [1,1,1,1]
```

Only an odd number can be used.

Answer:

```text
3
```


### 2. No valid square chain

```text
nums = [2,3,5]
```

Every number forms only a single-element sequence.

Answer:

```text
1
```


### 3. Chain stops before finding a center

```text
nums = [2,2,4,4]
```

There is no `16`.

The best sequence is:

```text
[2,4,2]
```

Answer:

```text
3
```


### 4. Long chain

```text
nums = [2,2,4,4,16,16,256]
```

Sequence:

```text
[2,4,16,256,16,4,2]
```

Answer:

```text
7
```


## Lessons

* Count frequencies first when duplicate usage matters.
* A hash map lets us verify whether each level of the chain can be formed.
* Repeated squaring makes numbers grow very quickly, so each chain is short.
* Always watch for special values like `1`, where the normal transition behaves differently.
* When a structure must be symmetric, think about which elements need pairs and which can appear only once.
