# 3514. Number of Unique XOR Triplets II 🟡

**Topics:** Bit Manipulation, Dynamic Programming, Array

## Intuition

Imagine you are building combinations using colored blocks.

* At first, you have built nothing, so your XOR value is `0`.
* Next, you choose one number and record every XOR value you can make.
* Then, you attach a second number to every existing combination and record the new XOR values.
* Finally, you attach a third number.

Instead of checking every possible triplet, we only keep track of **which XOR values are reachable after selecting 0, 1, 2, and 3 numbers**.

Another important observation is that the indices do not matter. Since choosing the same index multiple times is allowed (`i <= j <= k`), only the **distinct values** in the array affect the answer.

Since every number is at most `1500`, every XOR result fits within `11` bits. Therefore, there are only `2048` possible XOR values, making dynamic programming feasible.


## Approach

We use Dynamic Programming to track reachable XOR values.

1. Remove duplicate numbers since repeated values do not create new XOR results.
2. Create a DP table where:

   * `reachable[k][x]` is `True` if XOR value `x` can be formed using exactly `k` numbers.
3. Initialize:

   * Using zero numbers, only XOR `0` is possible.
4. For each step from `1` to `3`:

   * Try appending every distinct number to every previously reachable XOR value.
   * Mark the resulting XOR as reachable.
5. Count how many XOR values are reachable after selecting exactly three numbers.

Because there are only `2048` possible XOR values, the DP remains efficient.


## Example Walkthrough

**Input**

```text
nums = [1, 3]
```

### Step 1: Remove duplicates

```text
distinct = [1, 3]
```

### Step 2: Initial state

```text
reachable[0] = {0}
```

### Step 3: Choose one number

```text
0 XOR 1 = 1
0 XOR 3 = 3

reachable[1] = {1, 3}
```

### Step 4: Choose two numbers

From XOR `1`:

```text
1 XOR 1 = 0
1 XOR 3 = 2
```

From XOR `3`:

```text
3 XOR 1 = 2
3 XOR 3 = 0
```

```text
reachable[2] = {0, 2}
```

### Step 5: Choose three numbers

From XOR `0`:

```text
0 XOR 1 = 1
0 XOR 3 = 3
```

From XOR `2`:

```text
2 XOR 1 = 3
2 XOR 3 = 1
```

```text
reachable[3] = {1, 3}
```

There are `2` unique XOR values.

**Output**

```text
2
```


## Code

```python
class Solution:
    def uniqueXorTriplets(self, nums: List[int]) -> int:

        distinct = list(set(nums))
        n = len(distinct)
        LIMIT = 2048

        reachable = [[False]*LIMIT for _ in range(4)]
        reachable[0][0] = True

        for step in range(1, 4):
            prev = reachable[step-1]
            cur = reachable[step]

            for xor_val in range(LIMIT):
                if not prev[xor_val]:
                    continue

                for num in distinct:
                    cur[xor_val ^ num] = True

        count=0
        for ok in reachable[3]:
            if ok:
                count += 1

        return count
```


## Complexity

Let:

* (n) be the number of distinct values.
* (M = 2048) be the number of possible XOR states.

**Time Complexity**

$$
O(3 \times n \times M)
$$

Since (M = 2048) is a constant,

$$
O(n)
$$

for the given constraints.

**Space Complexity**

$$
O(4 \times M)
$$

which simplifies to

$$
O(M)
$$

or simply

$$
O(1)
$$

because `2048` is a fixed constant.


## Edge Cases

1. **Single element**

```text
nums = [5]
```

Only one value exists, so every triplet is:

```text
5 XOR 5 XOR 5 = 5
```

Answer:

```text
1
```


2. **All elements are identical**

```text
nums = [7, 7, 7]
```

Removing duplicates leaves only `{7}`, so only one XOR value is possible.


3. **All values are distinct**

```text
nums = [6, 7, 8, 9]
```

The DP efficiently explores every reachable XOR value without enumerating every triplet.


4. **Maximum input size**

```text
nums.length = 1500
```

Even with many distinct values, the algorithm remains efficient because it only explores `2048` XOR states.


## Lessons

* When the range of possible XOR values is small, it is often better to perform DP over XOR states instead of enumerating combinations.
* Tracking **reachable states** is a common dynamic programming pattern for bitwise problems.
* Removing duplicate values can significantly reduce unnecessary work when only unique outcomes matter.
* A fixed state space can transform an otherwise cubic brute-force solution into an efficient dynamic programming solution.
