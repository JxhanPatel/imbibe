# 3093. Longest Common Suffix Queries 🔴

**Topics:** Trie, Strings, Hashing, Data Structure





## Intuition

Imagine you are organizing words by their endings instead of their beginnings.

Normally, tries help us match prefixes:

* `"apple"` and `"app"` share `"app"`.

But this problem cares about suffixes:

* `"abcd"` and `"xbcd"` share `"bcd"`.

A neat trick is:

> Reverse every word.

Now suffixes become prefixes.

Example:

* `"abcd"` → `"dcba"`
* `"xbcd"` → `"dcbx"`

Now both start with `"dcb"`.

So we can use a normal trie on reversed strings.

At every trie node, we store:

* the index of the best candidate word
* where "best" means:

  1. shortest length
  2. if tie, earlier index

That way, while traversing the query, the deepest node reached automatically gives the answer for the longest matching suffix.







## Approach

We will:

1. Reverse every word in `wordsContainer`
2. Insert them into a trie
3. Each trie node stores the best index according to:

   * smaller word length
   * earlier index if lengths tie
4. Reverse each query word
5. Traverse the trie as far as possible
6. The last valid node contains the answer









## Example Walkthrough

### Input

```python
wordsContainer = ["abcd","bcd","xbcd"]
wordsQuery = ["cd","bcd","xyz"]
```

### Step 1: Reverse container words

```python
"abcd" -> "dcba"
"bcd"  -> "dcb"
"xbcd" -> "dcbx"
```

### Step 2: Build trie

While inserting:

* root stores best overall word
* each node stores best word passing through it

For `"bcd"`:

* length = 3
* smaller than `"abcd"` length 4
* so many nodes update to index 1




### Query `"cd"`

Reverse:

```python
"cd" -> "dc"
```

Traverse:

* `'d'` exists
* `'c'` exists

We stop there.

The node for `"dc"` stores index `1`.

Answer:

```python
1
```



### Query `"xyz"`

Reverse:

```python
"xyz" -> "zyx"
```

`'z'` does not exist from root.

So longest common suffix is `""`.

Root stores the globally shortest word:

```python
index = 1
```

Answer:

```python
1
```







## Code

```python
class TrieNode:
    def __init__(self):
        self.children = {}
        self.best_idx = -1




class Solution:
    def stringIndices(self, wordsContainer: List[str], wordsQuery: List[str]) -> List[int]:

        root= TrieNode()

        def maybe_update(node, idx):
            cur = node.best_idx


            if cur == -1:
                node.best_idx=idx
                return

            cur_len = len(wordsContainer[cur])
            new_len = len(wordsContainer[idx])

            if new_len < cur_len or (new_len==cur_len and idx<cur):
                node.best_idx = idx

        for idx, word in enumerate(wordsContainer):
            node=root
            maybe_update(node, idx)


            
            for ch in reversed(word):
                if ch not in node.children:
                    node.children[ch] = TrieNode()

                node=node.children[ch]
                maybe_update(node, idx)

        results=[]

        for query in wordsQuery:
            node=root

            for ch in reversed(query):
                if ch not in node.children:
                    break

                node= node.children[ch]
            results.append(node.best_idx)
            

        return results
```








## Complexity

Let:

* (N) = total length of all container words
* (Q) = total length of all query words

### Time

$$
O(N + Q)
$$

### Space

$$
O(N)
$$

for the trie.




## Edge Cases

### 1. No matching suffix

```python
wordsContainer = ["abc"]
wordsQuery = ["xyz"]
```

Answer becomes the globally shortest word.



### 2. Multiple equal suffix matches

```python
["abcd", "xbcd"]
```

Need shortest length.



### 3. Same length words

```python
["abc", "xbc"]
```

Choose earlier index.



## Lessons

### Key Trick

Reverse strings to convert:

* suffix matching → prefix matching



### Important Observation

The answer for every suffix path can be precomputed while building the trie.

So query answering becomes very fast.



### Pattern Learned

This is a classic:

> "Reverse + Trie" problem

Useful whenever problems involve:

* suffixes
* endings
* file extensions
* rhyme matching
* domain matching
