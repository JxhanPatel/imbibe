# 2685. Count the Number of Complete Components 🟡

**Topics:** Graph, Depth-First Search (DFS)

## Intuition

Imagine a classroom where students form friend groups.

A group is called **complete** if **every student is friends with every other student** in that group.

For example:

```text
A ----- B
 \     /
   \ /
    C
```

Everyone is connected to everyone else, so this is a complete group.

Now consider:

```text
A ----- B

C
```

Here, `A` and `B` are connected, but `C` is alone. These are actually two different connected components:

* `{A, B}` which is complete.
* `{C}` which is also complete because a single vertex already satisfies the condition.

Instead of checking every possible pair of vertices, we use an important property:

A connected component with `k` vertices is complete **if and only if** it contains exactly

$$
\frac{k(k-1)}{2}
$$

edges.

So our task becomes:

1. Find every connected component.
2. Count its vertices.
3. Count its edges.
4. Compare the actual number of edges with the expected number.


## Approach

We use an **iterative DFS** with a stack.

1. Build an adjacency list from the given edges.
2. Maintain a `seen` array to avoid visiting a node multiple times.
3. For every unvisited node:

   * Perform DFS to collect all nodes belonging to its connected component.
4. Once the component is discovered:

   * Count its vertices.
   * Compute the total degree of all vertices.
   * Since every edge contributes to the degree of two vertices, divide the degree sum by `2` to get the actual number of edges.
5. Compute the number of edges a complete graph should have:

$$
\frac{k(k-1)}{2}
$$

where `k` is the number of vertices.

6. If the actual and expected edge counts match, increment the answer.

## Code

```python
from collections import defaultdict
from typing import List

class Solution:
    def countCompleteComponents(self, n: int, edges: List[List[int]]) -> int:
        
        adj = defaultdict(list)
        for u, v in edges:
            adj[u].append(v)
            adj[v].append(u)

        seen = [False]*n
        result=0


        for start in range(n):
            if seen[start]:
                continue

            stack = [start]
            seen[start] = True
            nodes=[]

            while stack:
                cur = stack.pop()
                nodes.append(cur)
                for nxt in adj[cur]:
                    if not seen[nxt]:
                        seen[nxt] = True
                        stack.append(nxt)

            n_nodes = len(nodes)
            total_deg = sum(len(adj[node]) for node in nodes)
            edge_cnt = total_deg//2
            needed = n_nodes*(n_nodes-1) // 2

            if edge_cnt==needed:
                result += 1

        return result
```


## Complexity

**Time Complexity**

$$
O(n + m)
$$

where:

* $n$ is the number of vertices.
* $m$ is the number of edges.

Each vertex and edge is processed only once.

**Space Complexity**

$$
O(n + m)
$$

For the adjacency list, visited array, and DFS stack.


## Edge Cases

### 1. No edges

```text
n = 4
edges = []
```

Each vertex forms its own complete component.

Answer:

```text
4
```


### 2. One complete graph

```text
n = 4
edges = [
    [0,1],[0,2],[0,3],
    [1,2],[1,3],[2,3]
]
```

Every pair of vertices is connected.

Answer:

```text
1
```


### 3. Connected but incomplete

```text
n = 3
edges = [[0,1],[1,2]]
```

There is no edge between `0` and `2`.

Answer:

```text
0
```


### 4. Mixture of isolated and connected nodes

```text
n = 5
edges = [[0,1]]
```

Components:

* `{0,1}` → Complete
* `{2}` → Complete
* `{3}` → Complete
* `{4}` → Complete

Answer:

```text
4
```


## Lessons

* Finding connected components is one of the most common applications of DFS and BFS.
* An iterative DFS using a stack avoids recursion while producing the same traversal.
* In an undirected graph, the sum of all vertex degrees equals twice the number of edges.
* A connected component with `k` vertices is complete if it contains exactly

$$
\frac{k(k-1)}{2}
$$

edges.

* Rather than checking every pair of vertices, comparing the actual and expected edge counts provides a simple and efficient solution.
