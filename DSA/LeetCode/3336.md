# 3336. Find the Number of Subsequences With Equal GCD 🔴

**Topics:** Dynamic Programming, Math, Number Theory

## Intuition

Imagine you are sorting every number into one of **three boxes**:

* Box 1 belongs to the first subsequence (`seq1`).
* Box 2 belongs to the second subsequence (`seq2`).
* Leave the number out completely.

As you process each number, you only care about one piece of information from each box: **the current GCD**.

Instead of remembering every element inside each subsequence, we only remember:

* Current GCD of `seq1`
* Current GCD of `seq2`

This greatly reduces the amount of information we need to store. Every new number simply updates one of these GCDs or is ignored.

So the DP state becomes:

* `g1` = GCD of the first subsequence
* `g2` = GCD of the second subsequence

After processing all numbers, every state where `g1 == g2` (and both are non-zero) represents a valid pair of subsequences.


## Approach

We use **dynamic programming on GCD states**.

Let:

```text
dp[g1][g2]
```

represent the number of ways to process the numbers seen so far such that:

* `seq1` has GCD `g1`
* `seq2` has GCD `g2`

Initially:

```text
dp[0][0] = 1
```

Here, `0` represents an empty subsequence. This works nicely because:

```python
gcd(0, x) = x
```

which naturally initializes the GCD when the first element is added.

For every number `n`, there are exactly **three choices**:

1. Ignore it.
2. Put it into `seq1`.
3. Put it into `seq2`.

If we add `n` to `seq1`, its new GCD becomes:

```python
gcd(g1, n)
```

Similarly, adding it to `seq2` changes its GCD to:

```python
gcd(g2, n)
```

After processing every element, we simply sum all states where both subsequences have the same non-zero GCD.


## Example Walkthrough

**Input**

```text
nums = [2,4]
```

Initially:

```text
dp[0][0] = 1
```

### Process 2

From `(0,0)` we have three choices:

| Choice      | New State |
| ----------- | --------- |
| Skip        | (0,0)     |
| Put in seq1 | (2,0)     |
| Put in seq2 | (0,2)     |


### Process 4

From each existing state, again make three choices.

For example:

From `(2,0)`:

* Skip → `(2,0)`
* Put in seq1 → `(gcd(2,4),0) = (2,0)`
* Put in seq2 → `(2,4)`

Notice that adding `4` to a subsequence whose GCD is already `2` keeps the GCD equal to `2`.

The DP continues similarly for every state.

At the end, only states where

```text
g1 == g2
```

are counted.


## Code

```python
class Solution:
    def subsequencePairCount(self, nums: List[int]) -> int:
        
        MOD = 1000000007
        hi = max(nums)

        dp = [[0] * (hi+1) for _ in range(hi+1)]
        dp[0][0]=1


        for n in nums:
            new_dp = [[0] * (hi+1) for _ in range(hi+1)]

            for g1 in range(hi + 1):
                g1_new = math.gcd(g1, n)

                for g2 in range(hi+1):
                    cnt = dp[g1][g2]
                    
                    if not cnt:
                        continue

                    g2_new = math.gcd(g2, n)

                    new_dp[g1][g2] = (new_dp[g1][g2] + cnt) % MOD
                    new_dp[g1_new][g2] = (new_dp[g1_new][g2] + cnt) % MOD
                    new_dp[g1][g2_new] = (new_dp[g1][g2_new] + cnt) % MOD

            dp = new_dp

        total=0
        for g in range(1, hi+1):
            total += dp[g][g]
            
        total %= MOD

        return total
```


## Complexity

Let:

* $n$ be the length of `nums`
* $M$ be the maximum value in `nums`

For every element, we iterate over all possible GCD pairs.

**Time Complexity**

$$
O(n \times M^2)
$$

Since `M ≤ 200`, there are at most `201 × 201` DP states.

**Space Complexity**

$$
O(M^2)
$$

We maintain two DP tables of size `(M+1) × (M+1)`.


## Edge Cases

### 1. All numbers are the same

```text
nums = [1,1,1,1]
```

Every non-empty subsequence has GCD `1`, leading to many valid pairs.


### 2. No valid equal GCD except one

```text
nums = [10,20,30]
```

Only pairs with GCD `10` satisfy the condition.


### 3. Single element

```text
nums = [5]
```

It is impossible to form two non-empty disjoint subsequences.

Answer:

```text
0
```


## Lessons

* Sometimes the exact contents of a subsequence are unnecessary. Tracking a summary like the GCD is enough.
* The identity `gcd(0, x) = x` is a useful trick for representing an empty subsequence.
* When every element has a fixed number of choices, dynamic programming over states often avoids exponential enumeration.
* Compressing a subsequence into its GCD transforms an otherwise impossible state space into one that is manageable because the maximum value is only `200`.
