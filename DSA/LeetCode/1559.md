# 1559. Detect Cycles in 2D Grid 🟡


**Topics:** Graphs, DFS, Grid, Union-Find






## Intuition

Think of the grid like a map of colored tiles. You can walk only on tiles of the same color, moving up, down, left, or right.

Now imagine walking around on these tiles. A cycle exists if you can start from a tile, walk through same-colored neighbors, and come back to the starting tile without immediately stepping back to the previous tile.

Real-world analogy:
Imagine walking through a maze where all paths are made of the same colored bricks. If you can loop around and return to your starting point without retracing your immediate last step, you've found a cycle.







## Approach

We use **DFS (Depth First Search)** to explore the grid.

Key idea:

* Start DFS from each unvisited cell.
* While exploring neighbors:

  * Only move to cells with the same character.
  * Keep track of the **previous cell** to avoid going backward.
* If we reach a cell that is already visited and is NOT the parent, we found a cycle.

Steps:

1. Maintain a `visited` matrix.
2. For each cell:

   * If not visited, start DFS.
3. In DFS:

   * Mark current cell as visited.
   * Explore 4 directions.
   * If neighbor is:

     * Same character
     * Already visited
     * Not the parent → cycle found








## Example Walkthrough

Input:

```
grid = [
  ["a","a","a","a"],
  ["a","b","b","a"],
  ["a","b","b","a"],
  ["a","a","a","a"]
]
```

Steps:

1. Start DFS from (0,0)
2. Move through all connected 'a' cells
3. Eventually revisit a previously visited 'a' cell that is not the parent
4. This confirms a loop exists

Output:

```
True
```






## Code

```python
from typing import List

class Solution:
    def containsCycle(self, grid: List[List[str]]) -> bool:

        rows, cols = len(grid), len(grid[0])

        visited = [[False] * cols for _ in range(rows)]

        DIRS=[(0, 1), (0, -1), (1, 0), (-1, 0)]



        def dfs(r, c, from_r, from_c):
            visited[r][c] =True
            ch = grid[r][c]

            for dr, dc in DIRS:
                nr, nc= r + dr, c + dc

                if nr < 0 or nr >= rows or nc < 0 or nc >= cols:
                    continue
                if grid[nr][nc] != ch:
                    continue
                if nr == from_r and nc == from_c:
                    continue

                if visited[nr][nc]:
                    return True
                if dfs(nr, nc, r, c):
                    return True

            return False




        for r in range(rows):

            for c in range(cols):
                if not visited[r][c]:
                    if dfs(r, c, -1, -1):
                        return True

        return False
        
```






## Complexity

* Time Complexity: $O(m \times n)$
  Each cell is visited once

* Space Complexity: $O(m \times n)$
  For the visited matrix and recursion stack





## Edge Cases

1. Single cell grid
   No cycle possible

2. All cells different
   No valid moves, so no cycle

3. Large grid with same characters
   Likely to form cycles, needs efficient traversal






## Lessons

* Detecting cycles in grids is similar to cycle detection in graphs
* Tracking the parent node is crucial to avoid false positives
* DFS is a natural fit for exploring connected components
* This pattern appears often in grid and graph problems involving loops
