# 3312. Sorted GCD Pair Queries 🔴

**Topics:** Math, Number Theory, Binary Search, Prefix Sum, Sieve

## Intuition

Imagine grouping numbers by the divisors they share.

For example, every number divisible by `2` goes into one group, every number divisible by `3` goes into another, and so on.

If a divisor `g` has `k` numbers divisible by it, then every pair among those `k` numbers has a GCD that is **at least** `g`. However, some of those pairs may actually have a larger GCD, such as `2g`, `3g`, or another multiple of `g`.

So instead of directly counting pairs whose GCD is exactly `g`, we first count every pair divisible by `g`, then subtract the pairs already belonging to larger GCDs.

Once we know how many pairs have each exact GCD, we can imagine writing those GCD values into a sorted array. A prefix sum tells us where each GCD appears, allowing every query to be answered with a binary search.


## Approach

We solve the problem in four steps.

### 1. Count each value

Create a frequency array.

```text
cnt[x] = number of times x appears
```

This allows us to quickly determine how many numbers are divisible by any divisor.


### 2. Count divisible pairs

For every possible GCD `g`, count how many numbers are divisible by it.

```text
total = numbers divisible by g
```

If there are `total` such numbers, then the number of possible pairs is

$$
\frac{total(total-1)}{2}
$$

Initially,

```text
exact[g] = all pairs divisible by g
```


### 3. Remove larger GCDs

The count above still includes pairs whose actual GCD is larger than `g`.

Since we process divisors from largest to smallest, every larger GCD has already been computed.

So subtract them:

```python
exact[g] -= exact[multiple]
```

for every multiple of `g`.

After this subtraction,

```text
exact[g]
```

stores exactly the number of pairs whose GCD equals `g`.


### 4. Answer queries

Build a prefix sum where

```text
prefix[g]
```

equals the number of pairs whose GCD is at most `g`.

Now every query asks for the element at index `q` in the sorted GCD array.

Using binary search,

```python
bisect_right(prefix, q)
```

returns the smallest GCD whose range contains index `q`.


## Example Walkthrough

**Input**

```text
nums = [2,3,4]
queries = [0,2]
```

### Step 1

Frequency table

| Value | Count |
| ----: | ----: |
|     2 |     1 |
|     3 |     1 |
|     4 |     1 |


### Step 2

Numbers divisible by each divisor

| Divisor | Numbers | Pair Count |
| ------: | ------: | ---------: |
|       1 |       3 |          3 |
|       2 |       2 |          1 |
|       3 |       1 |          0 |
|       4 |       1 |          0 |


### Step 3

Subtract larger GCDs.

* `exact[2] = 1`
* `exact[1] = 3 - 1 = 2`

So the exact frequencies become

| GCD | Frequency |
| --: | --------: |
|   1 |         2 |
|   2 |         1 |

The sorted GCD array is

```text
[1,1,2]
```


### Step 4

Prefix counts

| GCD | Prefix |
| --: | -----: |
|   1 |      2 |
|   2 |      3 |

Queries:

* `0` belongs to GCD `1`
* `2` belongs to GCD `2`

Output

```text
[1,2]
```


## Code

```python
from bisect import bisect_right
from typing import List

class Solution:
    def gcdValues(self, nums: List[int], queries: List[int]) -> List[int]:
        hi = max(nums)
        cnt = [0] * (hi+1)

        for n in nums:
            cnt[n] += 1

        exact = [0] * (hi+1)

        for g in range(hi, 0, -1):
            total=0
            m=g
            while m <= hi:
                total += cnt[m]
                m += g

            exact[g] = total * (total-1) // 2

            m = g*2
            while m <= hi:
                exact[g] -= exact[m]
                m += g

        prefix = [0]
        running=0

        for g in range(1, hi+1):
            running += exact[g]
            prefix.append(running)

        return [bisect_right(prefix, q) for q in queries]
```


## Complexity

Let

* $n$ = length of `nums`
* $M$ = maximum value in `nums`

**Time Complexity**

Building the frequency array:

$$
O(n)
$$

The nested divisor loops run in harmonic-series time:

$$
O(M \log M)
$$

Each query is answered using binary search:

$$
O(\log M)
$$

Overall:

$$
O(n + M \log M + q \log M)
$$

where $q$ is the number of queries.


**Space Complexity**

$$
O(M)
$$

for the frequency, exact-count, and prefix arrays.


## Edge Cases

1. **All numbers are identical**

```text
nums = [4,4,4]
```

Every pair has GCD `4`.


2. **All numbers are pairwise coprime**

```text
nums = [2,3,5,7]
```

Every pair has GCD `1`.


3. **Only one pair**

```text
nums = [6,12]
```

The answer is simply the GCD of those two numbers.


4. **Maximum values**

```text
nums = [50000,50000,...]
```

The divisor-based counting remains efficient because it iterates over multiples rather than all pairs.


## Lessons

* Counting "exactly divisible by" often requires inclusion-exclusion.
* Processing divisors from largest to smallest lets previously computed larger GCDs be subtracted efficiently.
* A divisor sieve is a common optimization when values are much smaller than the number of possible pairs.
* Prefix sums combined with binary search can answer order-statistic queries without ever constructing the full sorted array.
* When the number of pairs is enormous, counting frequencies is often far more efficient than generating every pair.
