# 2078. Two Furthest Houses With Different Colors 🟢
**Topics:** Array, Greedy








## Intuition

Imagine a street where each house is painted a color. You want to stand at one house and look as far as possible to another house that is painted a *different* color.

The key idea is simple:

* The maximum distance will always involve either the **first house** or the **last house**.
  Why? Because those are the extreme ends, and distance grows as we move toward the ends.

So instead of checking all pairs, we just try:

* Compare from the left end to the right
* Compare from the right end to the left










## Approach

1. Start from the beginning:

   * Find the farthest house from index `0` that has a different color.

2. Start from the end:

   * Find the farthest house from index `n-1` that has a different color.

3. Return the maximum of these two distances.

This works because the optimal pair must include one of the endpoints.









## Example Walkthrough

**Input:**
`colors = [1,1,1,6,1,1,1]`

Step 1: Compare with first house (color = 1)

* Check from the end → index 6 → same
* index 5 → same
* index 4 → same
* index 3 → different
  Distance = `3 - 0 = 3`

Step 2: Compare with last house (color = 1)

* Check from the start → index 0 → same
* index 1 → same
* index 2 → same
* index 3 → different
  Distance = `6 - 3 = 3`

Final answer = `max(3, 3) = 3`











## Code (Python)

```python
class Solution:
    def maxDistance(self, colors: List[int]) -> int:

        n = len(colors)

        best = 0

        for j in range(n - 1, -1, -1):

            if colors[j] != colors[0]:
                best = j
                break

        for i in range(n):

            if colors[i] != colors[n - 1]:
                best = max(best, n - 1 - i)
                break



        return best

```








## Complexity

* Time: $O(n)$
* Space: $O(1)$







## Edge Cases

* Only two houses → always return 1
* All same except one → distance still works using ends
* Minimum size input (`n = 2`)
* Colors differ only near the middle








## Lessons

* When maximizing distance, always think about **boundaries**
* Not all problems need brute force pair checking
* Greedy observation can reduce $O(n^2)$ to $O(n)$
* Try fixing one end and scanning for the best partner
