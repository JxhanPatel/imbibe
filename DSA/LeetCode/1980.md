# 1980. Find Unique Binary String 🟡
**Topics:** Array, String, Backtracking



## Intuition

Imagine you're at a hotel with $n$ rooms, and each room has a unique $n$-digit binary code on its door. You want to find a code that isn't being used by any room. 

Here's the clever observation: there are $2^n$ possible binary strings of length $n$, but you only have $n$ rooms occupied. Since $n < 2^n$ for all $n \geq 1$, there are **guaranteed** to be empty rooms. You just need to find one!

The trick? We can construct a missing string by being a bit "contrarian." For each position $i$, we look at what the $i$-th string has at position $i$, and we pick the opposite. This ensures our new string differs from every existing string in at least one position.



## Approach

We'll use the **Diagonalization** approach (also known as Cantor's diagonal argument):

For each index $i$ from $0$ to $n-1$:
- Look at `nums[i][i]` (the $i$-th character of the $i$-th string)
- Append the opposite bit to our result

This guarantees our result differs from `nums[i]` at position $i$, so it cannot equal any string in the array.

**Why does this work?** 
- Our answer differs from `nums[0]` at index 0
- Our answer differs from `nums[1]` at index 1
- ...and so on
- Therefore, our answer differs from ALL strings in `nums`


## Example Walkthrough

**Input:** `nums = ["01", "10"]` (n = 2)


- `nums[0][0]` = '0', so we choose '1'
- `nums[1][1]` = '0', so we choose '1'

**Correct Result:** "11" ✓

"11" differs from "01" at index 0, and from "10" at index 1.



## Code

```python
class Solution:
    def findDifferentBinaryString(self, nums: List[str]) -> str:

        n = len(nums)
        result = []
        

        for i in range(n):
            current_bit = nums[i][i]
            
            # Flip it:- if '0', use '1'; if '1', use '0'
            if current_bit == '0':
                result.append('1')

            else:
                result.append('0')
        


        return ''.join(result)
```




## Complexity

- **Time Complexity:** $O(n)$ ... We iterate through $n$ strings, examining just one character from each.
- **Space Complexity:** $O(n)$ ... For storing the result string (or $O(1)$ extra space if we don't count the output).



## Edge Cases

| Scenario | Input | Output | Explanation |
|----------|-------|--------|-------------|
| **Minimum size** | `["0"]` | `"1"` | n=1, only two possible strings, pick the other one |
| **All zeros** | `["000", "001", "010"]` | `"111"` (or similar) | Works regardless of input pattern |
| **All ones** | `["111", "110", "101"]` | `"000"` (or similar) | The diagonal will be all '1's, we flip to '0's |
| **Maximum constraint** | n=16 | Any valid 16-bit string | Still $O(n)$ time, very efficient |



## Lessons

1. **Diagonalization is powerful:** This technique comes from Cantor's proof that real numbers are uncountable. It's elegant when you can use the input against itself.

2. **Pigeonhole Principle intuition:** When you have fewer items than possible slots ($n < 2^n$), a solution is guaranteed to exist. This gives you confidence to construct one rather than search exhaustively.

3. **Greedy construction beats brute force:** Instead of generating all $2^n$ possibilities and checking which is missing, we build the answer in $O(n)$ time by being clever about constraints.

4. **The "differ at position i" trick:** A common pattern when you need to differ from multiple strings is ensuring you differ from string $i$ specifically at position $i$.
