# 1594. Maximum Non-Negative Product in a Matrix 🟡

**Topics:** Dynamic Programming, Matrix, Greedy



## Intuition

Imagine you're walking through a hallway of rooms, and each room has a multiplier card. Some cards double your score, some cut it in half, and some flip your score's sign (positive to negative or vice versa). You can only move right or forward, and you want to reach the exit with the biggest non-negative score possible.

The catch? When you multiply by a negative number, your best path suddenly becomes your worst, and your worst becomes your best. So at every step, you need to track **both** the maximum and minimum product you've seen so far, because the "min" can flip into a winner with the next negative number.



## Approach

We'll use DP where at each cell `(i, j)`, we store two values:

- `dp_max[i][j]` = the largest product possible to reach this cell
- `dp_min[i][j]` = the smallest (most negative) product possible to reach this cell

Why both? Because when we hit a negative number, the max becomes the min and vice versa. At the end, we check if the max product at the bottom-right is non-negative. If yes, return it mod $10^9 + 7$. If not, return `-1`.

For transitions, at each cell we can arrive from the top or from the left, so we consider all four combinations (max_from_top, min_from_top, max_from_left, min_from_left) multiplied by the current cell value



## Example Walkthrough

Using `grid = [[1, -2, 1], [1, -2, 1], [3, -4, 1]]`

At each cell we track `(dp_max, dp_min)`. Let's walk through:

- `(0,0)`: value = 1. Both max and min = `1`. Start here.
- `(0,1)`: value = -2. Only path is from left: `1 * -2 = -2`. max = `-2`, min = `-2`.
- `(0,2)`: value = 1. `(-2) * 1 = -2`. max = `-2`, min = `-2`.
- `(1,0)`: value = 1. `1 * 1 = 1`. max = `1`, min = `1`.
- `(1,1)`: value = -2. Can come from `(0,1)` (max=-2, min=-2) or `(1,0)` (max=1, min=1).
  - From top: `-2 * -2 = 4`, `-2 * -2 = 4`
  - From left: `1 * -2 = -2`, `1 * -2 = -2`
  - new max = `4`, new min = `-2`
- `(1,2)`: value = 1. From `(0,2)` max=-2, and `(1,1)` max=4.
  - max = `4`, min = `-2`
- `(2,0)`: value = 3. max = `3`, min = `3`.
- `(2,1)`: value = -4. From `(1,1)` (max=4, min=-2) and `(2,0)` (max=3, min=3).
  - Products: `4*-4=-16`, `-2*-4=8`, `3*-4=-12`, `3*-4=-12`
  - max = `8`, min = `-16`
- `(2,2)`: value = 1. From `(1,2)` (max=4, min=-2) and `(2,1)` (max=8, min=-16).
  - max = `8`, min = `-16`

Final max = `8` which is non-negative, so return `8 % (10^9 + 7) = 8`. Correct!


## Code (Py)

```python
class Solution:
    def maxProductPath(self, grid: List[List[int]]) -> int:


        MOD = 10**9 + 7
        m = len(grid)
        n = len(grid[0])

        dp = [[(0, 0)] * n for _ in range(m)]

        # base case: top-left is just the grid value itself
        dp[0][0] = (grid[0][0], grid[0][0])


        # fill the first row ; only one way to get here (from the left)
        for j in range(1, n):
            val = grid[0][j]
            prev_max, prev_min = dp[0][j-1]
            products = [prev_max * val, prev_min * val]
            dp[0][j] = (max(products), min(products))


        # fill the first column ; only one way to get here (from above)
        for i in range(1, m):
            val = grid[i][0]
            prev_max, prev_min = dp[i-1][0]
            products = [prev_max * val, prev_min * val]
            dp[i][0] = (max(products), min(products))


        # fill the rest of the grid
        for i in range(1, m):
            for j in range(1, n):
                val = grid[i][j]

                # gather the max and min from the cell above and the cell to the left
                top_max, top_min = dp[i-1][j]
                left_max, left_min = dp[i][j-1]


                # consider all 4 combinations — negative * negative can become our new max!
                candidates = [
                    top_max * val,
                    top_min * val,
                    left_max * val,
                    left_min * val,
                ]

                dp[i][j] = (max(candidates), min(candidates))


        # final answer is the max product at the bottom-right corner
        final_max, _ = dp[m-1][n-1]

        if final_max < 0:
            return -1 

        # apply modulo only at the end (imp!)
        return final_max % MOD
```



## Complexity

**Time Complexity:** $O(m \times n)$

We visit every cell exactly once and do constant work per cell.

**Space Complexity:** $O(m \times n)$

We store a `(max, min)` pair for every cell. This can be optimized to $O(n)$ by only keeping the previous row, but the full table is easier to reason about.



## Edge Cases

**1. All negative numbers with an odd-length path**
If every cell is negative and the path length forces an odd number of multiplications, every path will end negative, so the answer is `-1`. Like example 1 in the problem.

**2. Zero in the grid**
A zero kills the product along any path that passes through it. But if all other paths are negative, a path through zero gives a product of `0` which is still non-negative. So zero is sometimes your best friend! Example 3 shows this.

**3. Single cell grid**
If `m = n = 1`, the answer is just `grid[0][0]` if it's non-negative, else `-1`. Your DP handles this naturally since the base case covers it.

**4. Applying mod too early**
If you apply `% MOD` inside the DP as you build up products, you'll lose the sign information and get wrong answers. Always track the raw product, and only take the modulo at the very end on the final result.



## Lessons

The key insight here is the **"track both max and min"** trick, which you'll also see in problems like Leetcode 152 (Maximum Product Subarray). Whenever negative numbers are involved in a product-based DP, your current minimum can become your future maximum with a single multiplication.

Another important takeaway is the **"mod only at the end"** rule. Modular arithmetic plays nice with addition but can be tricky with comparisons. Never compare or use mod values to determine which is bigger — keep the raw numbers for logic, then mod at the final step.
