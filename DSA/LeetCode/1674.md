# 1674. Minimum Moves to Make Array Complementary 🟡


**Topics:** Array, Greedy, Prefix Sum







# Intuition

Think of every mirrored pair in the array as two people trying to agree on a common target sum.

For a pair `(a, b)`:

* If their sum is already the target, we need **0 moves**
* If changing one number can make the target, we need **1 move**
* Otherwise, we need **2 moves**

Instead of checking every target sum separately for every pair, we can efficiently track how many moves each possible target sum needs using a **difference array**.

The important observation:

For a pair `(a, b)`:

* Minimum possible sum after changing one value is:

  $$
  \min(a,b) + 1
  $$

* Maximum possible sum after changing one value is:

$$
\max(a,b) + limit
$$


Inside this range, we can achieve the target in **1 move**.
Outside it, we need **2 moves**.

This lets us process ranges efficiently.









# Approach

We process each mirrored pair:

$$
(nums[i], nums[n-1-i])
$$

For every possible target sum from `2` to `2 * limit`, we maintain how many moves are needed.

For one pair:

### Initially

Assume every target requires `2 moves`.

### Then improve ranges

For pair `(a,b)`:

* `0 moves` for:

$$
a+b
$$

* `1 move` for sums in:


$$
\min(a,b)+1,\ \max(a,b)+limit
$$


We use a difference array to apply these range updates efficiently.

Finally, we compute prefix sums to get the total moves needed for every target sum and return the minimum.

# Code

```python
class Solution:
    def minMoves(self, nums: List[int], limit: int) -> int:

        n = len(nums)

        max_sum = 2*limit
        delta = [0]*(max_sum+2)


        for i in range(n//2):

            a, b = nums[i], nums[n-1-i]

            lo = min(a, b) +1
            hi = max(a, b) +limit

            s = a+b

            delta[2]+= 2
            delta[lo]-= 1
            delta[hi+1]+= 1
            delta[s]-= 1
            delta[s+1]+= 1


        best = float("inf")
        running=0

        for target in range(2, max_sum+1):
            running += delta[target]


            if running < best:
                best=running

        return best
```







# Complexity

### Time Complexity

$$
O(n + limit)
$$

* Processing all pairs takes (O(n))
* Prefix sum traversal takes (O(limit))


### Space Complexity

$$
O(limit)
$$

For the difference array.










# Edge Cases

### 1. Array already complementary

```python
nums = [1,2,1,2]
```

Answer:

```python
0
```


### 2. All pairs require 2 moves

```python
nums = [1,1,1,1]
limit = 100000
```

Some target sums may still require changing both values.


### 3. Smallest possible input

```python
nums = [1,2]
```

Only one pair exists.










# Lessons

* Difference arrays are powerful for range updates.
* Instead of evaluating every target separately, process intervals efficiently.
* Many array optimization problems become easier after converting:

$$
"cost\ per\ value"
$$

into:

$$
"range\ contribution"
$$
