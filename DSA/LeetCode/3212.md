# 3212. Count Submatrices With Equal Frequency of X and Y 🟡
**Topics:** Array, Matrix, Prefix Sum



## Intuition

Imagine you're playing a game where you're laying down a transparent sheet over a grid map, and you always have to start from the **top-left corner**. You slowly expand the sheet downward and rightward, and after every expansion you ask: "Are the number of X's and Y's in my current view equal, and do I have at least one X?" That's exactly what this problem is. Every valid position of your sheet's bottom-right corner counts as one answer.

The key insight is that every submatrix we care about **must include `grid[0][0]`**. So instead of checking all possible submatrices (which would be way too slow), we just ask the question cell by cell as we sweep through the grid. At each cell `(i, j)`, we're asking: "What does the submatrix from `(0,0)` to `(i,j)` look like?"



## Approach

We'll maintain two 2D prefix sum grids, `px[i][j]` and `py[i][j]`, where each entry stores the count of `'X'` (or `'Y'`) in the rectangle from `(0,0)` to `(i,j)`.

The classic 2D prefix sum formula for a cell `(i, j)` is:

$$\text{prefix}[i][j] = \text{val}(i,j) + \text{prefix}[i-1][j] + \text{prefix}[i][j-1] - \text{prefix}[i-1][j-1]$$

The subtraction removes the overlap that gets counted twice (the rectangle that's above AND to the left of `(i,j)`). This lets us compute the count for any top-left anchored rectangle in $O(1)$ time after an $O(m \times n)$ preprocessing step.

Once we have those prefix sums, we just walk every cell `(i, j)` and check:
1. Does `px[i][j] == py[i][j]`? (equal X and Y count)
2. Is `px[i][j] >= 1`? (at least one X)

If both are true, increment our answer.



## Example Walkthrough

```
grid = [["X","Y","."],
        ["Y",".","."]]
```

We build `px` and `py` in one pass. Let's trace a few cells:

**`(0,0)` = 'X':** `px[0][0] = 1`, `py[0][0] = 0`. Check: `1 != 0`. Not valid.

**`(0,1)` = 'Y':** `px[0][1] = 1` (still just the one X), `py[0][1] = 1`. Check: `1 == 1` and `px >= 1`. **Valid! Count = 1.**

**`(0,2)` = '.':** `px[0][2] = 1`, `py[0][2] = 1`. Still equal. **Valid! Count = 2.**

**`(1,0)` = 'Y':** `px[1][0] = 1` (X from row 0), `py[1][0] = 1`. Equal! **Valid! Count = 3.**

**`(1,1)` = '.':** `px[1][1] = 1`, `py[1][1] = 2` (Y at (0,1) and Y at (1,0)). Not equal. Not valid.

**`(1,2)` = '.':** `px = 1`, `py = 2`. Still not equal. Not valid.

Final answer: **3.**



## Code

```python
class Solution:
    def numberOfSubmatrices(self, grid: List[List[str]]) -> int:

        rows, cols = len(grid), len(grid[0])
        count = 0   

    # px[i][j] = count of 'X' in submatrix (0,0) to (i,j)
        px = [[0] * cols for _ in range(rows)]
        py = [[0] * cols for _ in range(rows)]
        

        
        for i in range(rows):
            for j in range(cols):
                
                px[i][j] = 1 if grid[i][j] == 'X' else 0
                py[i][j] = 1 if grid[i][j] == 'Y' else 0
                
                # add the row above (if it exists)
                if i > 0:
                    px[i][j] += px[i-1][j]
                    py[i][j] += py[i-1][j]
                
                # add the column to the left (if it exists)
                if j > 0:
                    px[i][j] += px[i][j-1]
                    py[i][j] += py[i][j-1]
                
                # remove the overlap we double-counted (up and to the left)
                if i > 0 and j > 0:
                    px[i][j] -= px[i-1][j-1]
                    py[i][j] -= py[i-1][j-1]
                

                # check if this submatrix (0,0)->(i,j) qualifies
                if px[i][j] == py[i][j] and px[i][j] >= 1:
                    count += 1
        
        return count
            
```



## Complexity

**Time:** $O(m \times n)$ where $m$ is the number of rows and $n$ is the number of columns. We visit each cell exactly once and do $O(1)$ work per cell.

**Space:** $O(m \times n)$ for the two prefix sum grids.

> We could actually reduce space to $O(n)$ by only keeping the previous row's prefix sums and running a column-wise running sum, but the two-grid version is much easier to reason about and debug, which matters more during a contest.



## Edge Cases

**1. No 'X' at `grid[0][0]`:** If the top-left cell is `'Y'` or `'.'`, no submatrix can ever be valid because every submatrix includes `(0,0)` and we need at least one `'X'`. Your code handles this automatically since `px[i][j]` will always be 0 in the first column/row before encountering any X, and the `>= 1` check gates everything out.

**2. All dots:** Like Example 3, a grid of all `'.'` gives `px[i][j] = 0` everywhere. The `>= 1` check ensures we return 0 immediately without false positives from `0 == 0`.

**3. Equal X and Y counts but X comes after Y in (0,0):** Since `grid[0][0]` must be part of every submatrix, if `grid[0][0] == 'Y'`, then `px` starts at 0 and `py` starts at 1, so `px[0][0] != py[0][0]`. No issue.

**4. Large grids (1000x1000):** With $10^6$ cells and $O(1)$ work per cell, this runs comfortably within time limits. A brute-force $O(m^2 n^2)$ approach would be around $10^{12}$ operations and would time out immediately.



## Lessons

**The core tricks to take away from this problem:**

The **2D prefix sum** pattern is one of the most reusable tools in competitive programming. Memorizing the formula (add top, add left, subtract top-left overlap) will pay dividends across many problems. It turns what looks like an $O(n^4)$ brute force into a clean $O(n^2)$ solution.

The key observation that makes this problem tractable is the **constraint that all submatrices include `(0,0)`**. Without it, you'd need to consider all possible top-left corners too. Whenever you see a constraint like "must include a fixed corner," think prefix sums from that corner.

Finally, tracking **two separate prefix grids** (one for X, one for Y) and comparing them at the end is cleaner and less error-prone than tracking a single net difference. When you have two things to count, count them separately.
