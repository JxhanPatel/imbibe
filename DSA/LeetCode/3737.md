# 3737. Count Subarrays With Majority Element I 🟡

**Topics:** Array, Prefix Sum, Counting


## Intuition

Imagine we give every occurrence of `target` a score of **+1** and every other element a score of **-1**.

For any subarray:

* Let `t` be the number of times `target` appears.
* Let `o` be the number of other elements.

The target is a majority element when:

$$
t > \frac{t+o}{2}
$$

Multiplying by 2:

$$
2t > t + o
$$

Which simplifies to:

$$
t > o
$$

Now notice that if we transform:

```text
target     -> +1
non-target -> -1
```

then the sum of a subarray becomes:

$$
t - o
$$

A positive sum means:

$$
t - o > 0
$$

which is exactly:

$$
t > o
$$

So instead of directly counting occurrences, we only need to count subarrays whose transformed sum is positive.

## Approach

1. Convert the array into a new array:

   * `target` becomes `+1`
   * every other value becomes `-1`

2. Build a prefix sum array so that any subarray sum can be computed in constant time.

3. Iterate through all possible subarrays:

   * Compute the subarray sum using prefix sums.
   * If the sum is positive, then `target` is the majority element in that subarray.
   * Add it to the answer.

Since the constraint is small (`n ≤ 1000`), checking all subarrays is fast enough.

## Example Walkthrough

### Input

```text
nums = [1,2,2,3]
target = 2
```

### Step 1: Transform

```text
[-1, 1, 1, -1]
```

### Step 2: Build Prefix Sum

```text
prefix = [0, -1, 0, 1, 0]
```

### Step 3: Check Subarrays

Subarray `[2]`

```text
sum = 1
```

Positive, so valid.

Subarray `[2,2]`

```text
sum = 2
```

Positive, so valid.

Subarray `[1,2,2]`

```text
sum = 1
```

Positive, so valid.

Subarray `[2,2,3]`

```text
sum = 1
```

Positive, so valid.

After checking all subarrays, the answer is:

```text
5
```


## Code

```python
class Solution:
    def countMajoritySubarrays(self, nums: List[int], target: int) -> int:

        mapped = [1 if x==target else -1 for x in nums]

        n = len(mapped)

        prefix = [0] * (n+1)

        for i in range(n):
            prefix[i+1] = prefix[i] + mapped[i]

        ans=0
        

        for l in range(n):
            for r in range(l, n):
                if prefix[r+1] - prefix[l]>0:
                    ans+= 1


        return ans
```

## Complexity

### Time Complexity

The nested loops examine every subarray:

$$
O(n^2)
$$

### Space Complexity

The prefix sum array stores `n + 1` values:

$$
O(n)
$$

## Edge Cases

1. **Target does not appear**

```text
nums = [1,2,3]
target = 4
```

Every element becomes `-1`, so no subarray can have a positive sum.

Answer:

```text
0
```

2. **All elements are target**

```text
nums = [1,1,1,1]
target = 1
```

Every subarray has target as the majority.

Answer:

```text
10
```

3. **Single element equal to target**

```text
nums = [5]
target = 5
```

Mapped array:

```text
[1]
```

Positive sum, so the answer is:

```text
1
```

4. **Single element not equal to target**

```text
nums = [5]
target = 1
```

Mapped array:

```text
[-1]
```

No valid subarray exists.

Answer:

```text
0
```

## Lessons

* Majority-element conditions can often be transformed into a prefix-sum problem.
* Mapping values to `+1` and `-1` is a useful trick for converting frequency comparisons into sum comparisons.
* Prefix sums allow subarray sums to be computed in constant time.
* When constraints are small, a simple (O(n^2)) solution is often the cleanest and easiest to understand.
* Always look for algebraic simplifications of conditions like "appears more than half the time."
