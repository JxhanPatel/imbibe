# 154. Find Minimum in Rotated Sorted Array II 🔴

Topics: Array, Binary Search



## Intuition

Imagine a circular conveyor belt of sorted boxes. Someone rotated the belt, so the smallest box is no longer at the front.

In the previous version of this problem, every box had a unique label, so comparing two boxes always gave a clear direction.

But now duplicates exist.

That is like seeing several identical boxes and not knowing whether the rotation point is to the left or right. Sometimes we lose information and must carefully shrink the search space.

We still use binary search because most of the time we can eliminate half of the array.






## Approach

We compare the middle element with the rightmost element.

### Case 1: `nums[mid] > nums[right]`

The minimum must be on the right side because the left part is sorted and larger.

Move:

```python
left = mid + 1
```



### Case 2: `nums[mid] < nums[right]`

The minimum could be at `mid` or on the left side.

Move:

```python
right = mid
```



### Case 3: `nums[mid] == nums[right]`

This is the tricky duplicate case.

We cannot determine which side contains the minimum because both values are equal.

So we safely shrink the search space by doing:

```python
right -= 1
```

This may degrade the runtime to linear time in the worst case.







## Example Walkthrough

Input:

```python
nums = [2,2,2,0,1]
```

Initial:

```python
left = 0
right = 4
```

### Step 1

```python
mid = 2
nums[mid] = 2
nums[right] = 1
```

Since:

```python
2 > 1
```

Minimum is on the right side.

Move:

```python
left = mid + 1 = 3
```



### Step 2

```python
left = 3
right = 4
mid = 3
```

```python
nums[mid] = 0
nums[right] = 1
```

Since:

```python
0 < 1
```

Move:

```python
right = mid = 3
```



Now:

```python
left == right == 3
```

Answer:

```python
nums[3] = 0
```











## Code

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:

        lo=0
        hi = len(nums)-1

        while lo < hi:
            mid = (lo+hi) //2

            if nums[mid] > nums[hi]:
                lo = mid+1


            elif nums[mid] < nums[hi]:
                hi=mid


            else:
                hi -=1

        return nums[lo]
```







## Complexity

Best/Average case:

$$
O(\log n)
$$

Worst case with many duplicates:

$$
O(n)
$$

Space complexity:

$$
O(1)
$$



## Edge Cases

1. Array with all duplicates

```python
[2,2,2,2]
```

2. Array not rotated

```python
[1,2,3,4]
```

3. Single element

```python
[10]
```

4. Duplicate values around pivot

```python
[3,3,1,3]
```



## Lessons

* Binary search works even in rotated arrays.
* Comparing with the rightmost element helps identify the sorted side.
* Duplicates reduce certainty in binary search decisions.
* When duplicates prevent direction detection, safely shrink the range by one.
* Worst-case complexity increases because duplicates can force near-linear scanning.
