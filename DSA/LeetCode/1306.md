# 1306. Jump Game III 🟡

**Topics:** Array, Depth-First Search (DFS)




## Intuition

Imagine you are standing on stones in a river.

Each stone tells you exactly how far you can jump, either forward or backward. Your goal is to see if you can eventually land on a stone marked `0`.

The tricky part is that you can easily get stuck jumping in circles forever. So, we need a way to remember which stones we already visited.

This problem is really about exploring reachable positions while avoiding revisiting the same index.








## Approach

We can solve this using **DFS** or **BFS**.

Here, we will use **DFS with a visited set**.

### Key idea

From index `i`:

* Jump forward → `i + arr[i]`
* Jump backward → `i - arr[i]`

We keep exploring until:

* We reach a value `0` → return `True`
* We go out of bounds
* We revisit an index

### Steps

1. Start DFS from `start`
2. If current index is invalid or already visited, stop
3. If `arr[i] == 0`, return `True`
4. Mark index as visited
5. Recursively explore both possible jumps












## Example Walkthrough

### Input

```python
arr = [4,2,3,0,3,1,2]
start = 5
```

### Execution

Start at index `5`

* `arr[5] = 1`
* Jump to:

  * `5 + 1 = 6`
  * `5 - 1 = 4`

Go to index `4`

* `arr[4] = 3`
* Jump to:

  * `4 + 3 = 7` (invalid)
  * `4 - 3 = 1`

Go to index `1`

* `arr[1] = 2`
* Jump to:

  * `1 + 2 = 3`

At index `3`

* `arr[3] = 0`

We found a zero, so return `True`.










## Code

```python
class Solution:
    def canReach(self, arr: List[int], start: int) -> bool:

        n = len(arr)

        seen = set()

        def dfs(i):
            if not (0<=i < n) or i in seen:
                return False

            if arr[i] == 0:
                return True

            seen.add(i)
            step = arr[i]

            return dfs(i+step) or dfs(i-step)

        return dfs(start)
```









## Complexity

* Time Complexity: $O(n)$
* Space Complexity: $O(n)$

Each index is visited at most once.








## Edge Cases

1. **Start index already contains 0**

```python
arr = [0]
start = 0
```

Answer is immediately `True`.




2. **Infinite loop possibility**

```python
arr = [1,1,1,1]
```

Without a visited set, we could keep bouncing forever.




3. **All jumps go out of bounds**

```python
arr = [5,6,7]
start = 0
```

No valid moves exist.









## Lessons

* This is a classic graph traversal problem disguised as an array problem.
* Each index behaves like a node in a graph.
* Always track visited states when cycles are possible.
* DFS and BFS both work well for reachability problems.
