# 3740. Minimum Distance Between Three Equal Elements I 🟢
**Topics:** Array, Hash Table, Sorting



## Intuition
Imagine you are a teacher looking for three students wearing the exact same red shirt in a long line. To keep them close together for a group photo, you want to minimize the total walking distance between them. 

The "distance" formula given, $abs(i - j) + abs(j - k) + abs(k - i)$, looks a bit fancy, but it has a secret. If we assume the indices are sorted such that $i < j < k$, the formula simplifies beautifully:
$$(j - i) + (k - j) + (k - i) = 2 \times (k - i)$$
This means the distance is simply **twice the span** between the first and the last element of the trio. To minimize this, we just need to find three identical numbers that are as close to each other as possible in the array.



## Approach
Since the constraints are very small ($n \le 100$), we have a lot of flexibility. However, the most efficient way to teach this is using a **Hash Map (Dictionary)**.

1.  **Group Indices:** Iterate through the array and store the positions (indices) of every number in a dictionary. For example, if `nums = [1, 2, 1, 1]`, our map would look like `{1: [0, 2, 3], 2: [1]}`.
2.  **Check for Trios:** Iterate through the groups in our map. If a number appears fewer than 3 times, we can't form a "good tuple," so we skip it.
3.  **Minimize the Span:** For numbers that appear 3 or more times, we look at their indices. To get the smallest span for a specific number, we check every consecutive window of 3 indices. For indices $[idx_1, idx_2, idx_3, ...]$, the best distance for that number will be $2 \times \min(idx_{m+2} - idx_m)$.
4.  **Update Global Minimum:** Keep track of the smallest distance found across all numbers.



## Example Walkthrough
**Input:** `nums = [1, 2, 1, 1, 3]`

1.  **Map out positions:**
    * `1` is at indices `[0, 2, 3]`
    * `2` is at index `[1]`
    * `3` is at index `[4]`
2.  **Evaluate `1`:**
    * It has 3 indices. The span is $3 - 0 = 3$.
    * Distance = $2 \times 3 = 6$.
3.  **Evaluate others:**
    * `2` and `3` don't have enough occurrences.
4.  **Result:** `6`.



## Code

```python
def minimumDistance(nums):
    # Group indices of each number
    pos_map = {}
    for index, val in enumerate(nums):
        if val not in pos_map:
            pos_map[val] = []
        pos_map[val].append(index)
    
    min_dist = float('inf')
    found = False
    
    # Check each number that appears at least 3 times
    for val in pos_map:
        indices = pos_map[val]
        if len(indices) < 3:
            continue
            
        found = True
        # Slide a window of size 3 over the indices
        for i in range(len(indices) - 2):
            # Distance formula simplifies to 2 * (k - i) 
            # where k is the 3rd element and i is the 1st
            current_span = indices[i+2] - indices[i]
            min_dist = min(min_dist, 2 * current_span)
            
    return min_dist if found else -1
```


## Complexity
* **Time Complexity:** $O(n)$. We traverse the list once to build the map, and then we iterate through the stored indices. Even though there is a nested loop, every index is visited exactly once more across all iterations.
* **Space Complexity:** $O(n)$ to store the indices of the elements in the hash map.



## Edge Cases
1.  **No Trios:** If the input is `[1, 2, 1, 2]`, no number appears three times. The `found` flag ensures we return `-1`.
2.  **All Same Elements:** If the input is `[5, 5, 5, 5, 5]`, the code will check `(0,1,2)`, `(1,2,3)`, and `(2,3,4)` to find the tightest window.
3.  **Minimum Length:** If the array length is less than 3, the loops naturally won't execute, and we return `-1`.

## Lessons
* **Formula Simplification:** Always try to simplify math formulas involving absolute values by assuming an order. It often reveals that you only need the "boundary" elements.
* **The Power of Grouping:** When a problem asks about relationships between identical values, a Hash Map should be your first instinct to bring those values "together."
* **Sliding Window on Indices:** Instead of checking all combinations of three indices $O(n^3)$ sorting the indices (which happened naturally by how we built the map) lets us use a window to find the closest trio in linear time.
