# 2751. Robot Collisions 🔴

**Topics:** Array · Stack · Sorting · Simulation

## Intuition

Imagine a bunch of bumper cars on a track. Some are heading right, some left. The only cars that can crash are a right-moving car followed by a left-moving car (because they're heading toward each other). Two right-movers? They'll never meet. Two left-movers? Same deal.

Now here's the trick: the last right-moving car is always the one that will meet the next left-moving car first. That "last seen" pattern screams **stack**.

Think of the stack as a waiting room. Right-moving robots sit and wait. The moment a left-moving robot shows up, it starts picking fights with whoever's at the top of the waiting room.



## Approach

Since positions can be unsorted but collisions depend on physical order, we first sort robots by their actual positions. But we need to remember the original index of each robot to return results in the original order.

Here's the plan:
1. Sort robots by position.
2. Use a stack to keep track of right-moving robots.
3. When a left-moving robot arrives, simulate collisions with the stack's top.
4. After all collisions, collect survivors and sort them back by original index.



## Example Walkthrough

Using Example: `positions=[3,5,2,6]`, `healths=[10,10,15,12]`, `directions="RLRL"`

**Sort by position:**
- C (pos=2, hp=15, R), A (pos=3, hp=10, R), B (pos=5, hp=10, L), D (pos=6, hp=12, L)

**Simulate:**
1. C is R → push to stack. Stack: `[C=15]`
2. A is R → push to stack. Stack: `[C=15, A=10]`
3. B is L → fight starts. A(10) vs B(10): equal health, both die. Stack: `[C=15]`
4. D is L → fight starts. C(15) vs D(12): C wins, C becomes 14. Stack: `[C=14]`

**No more robots. Survivor: C with hp=14. Output: `[14]`**

## Code

```python
class Solution:
    def survivedRobotsHealths(self, positions: List[int], healths: List[int], directions: str) -> List[int]:

        n = len(positions)  # total number of robos

        robots = sorted(range(n), key=lambda i: positions[i])

        stack = []  # will store indices of robots moving to the right

        for i in robots:

            if directions[i] == 'R':
                # right-moving robots just wait for potential collision
                stack.append(i)

            else:
                # this one is moving left, so it might collide
                while stack:
                    top = stack[-1]  # last right-moving robot

                    # compare their health values
                    if healths[top] > healths[i]:
                        # right robot survives this clash
                        healths[top] -= 1
                        healths[i] = 0  # current robot dies
                        break


                    elif healths[top] < healths[i]:
                        # left robot wins, but loses 1 health
                        healths[i] -= 1
                        healths[top] = 0
                        stack.pop()  # remove dead robot


                        # continue, maybe more collisions ahead
                    else:
                        # equal health -> both destroyed
                        healths[top] = 0
                        healths[i] = 0
                        stack.pop()
                        break


                # if stack is empty, this robot just keeps going left
                # nothing to do here (no need to store it anywhere)

        # collect surviving robots' healths
        result = []  # doing this manually instead of list comprehension (more readable imo)
        for i in range(n):
            if healths[i] > 0:
                result.append(healths[i])

        return result
```

## Complexity

**Time:** $O(n \log n)$

Sorting takes $O(n \log n)$. Each robot is pushed and popped from the stack at most once, so the simulation itself is $O(n)$.

**Space:** $O(n)$

The stack can hold at most $n$ robots in the worst case (all moving right).

## Edge Cases

**1. All robots moving the same direction**
No collisions ever happen. Every robot survives with original health. The stack either fills up (all R) or stays empty (all L, no one to fight).

**2. Equal health tie**
Both robots die. Make sure your code handles the `break` correctly after both are marked dead — the left-mover should not keep fighting.

**3. One left-mover destroying many right-movers**
A single powerful left-mover can wipe out several right-movers in a row. The `while stack` loop handles this: it keeps fighting until the left-mover dies or the stack is empty.

**4. Positions are unsorted**
The problem explicitly says positions may be unsorted. If you forget to sort first, the collision logic breaks completely since you're simulating physical space.

**5. Single robot**
Nothing to collide with. Returns `[healths[0]]`.

## Lessons

**The "last in, first out" collision insight:** The most recently seen right-mover is always the first one a left-mover hits. That's exactly what a stack gives you, and recognizing this pattern is the whole key.

**Sort first when order matters physically but input is logical:** Whenever a problem has physical positions but gives you data in arbitrary order, sort by position before simulating.

**Mark dead with 0, filter at the end:** Instead of removing robots mid-array (expensive), mark them as health=0 and do one final pass. Clean and efficient.

**Stack pattern for "pending" elements:** Stacks shine when you have a group of elements that are "waiting" for something to come along and interact with them. Right-movers sit and wait; left-movers are the trigger. Classic pattern worth internalizing.
