# 2452. Words Within Two Edits of Dictionary 🟡

**Topics:** Strings, Brute Force






## Intuition

Think of each word like a lock combination, and dictionary words as correct passwords.

You are allowed to tweak at most 2 characters in a query word. The question becomes:
"Can I turn this word into any dictionary word by changing at most 2 letters?"

So for every query word, we try matching it with each dictionary word and count how many positions differ.
If differences ≤ 2, it's a valid match.







## Approach

We will use a simple brute-force approach:

1. Loop through each word in `queries`
2. For each query word, compare it with every word in `dictionary`
3. Count character differences:

   * Iterate through characters
   * Increment a counter when characters differ
4. If differences ≤ 2, add the query word to result and break early
5. Return the result list

Why this works:

* Constraints are small (≤ 100 words, length ≤ 100)
* Brute force is efficient enough








## Example Walkthrough

**Input:**

```
queries = ["word","note","ants","wood"]
dictionary = ["wood","joke","moat"]
```

### Step 1: "word"

* Compare with "wood":

  * w == w
  * o == o
  * r != o → diff = 1
  * d == d
* Total diff = 1 ≤ 2 → valid

### Step 2: "note"

* Compare with "joke":

  * n != j → 1
  * o == o
  * t != k → 2
  * e == e
* Total diff = 2 → valid

### Step 3: "ants"

* Compare with all dictionary words → always diff > 2 → invalid

### Step 4: "wood"

* Matches exactly → diff = 0 → valid

**Output:**

```
["word","note","wood"]
```







## Code

```python
from typing import List

class Solution:
    def twoEditWords(self, queries: List[str], dictionary: List[str]) -> List[str]:

        result = []

        for q in queries:

            for w in dictionary:
                diff = 0

                for i in range(len(q)):
                    if q[i] != w[i]:
                        diff += 1

                        
                    if diff > 2:
                        break

                if diff <= 2:
                    result.append(q)
                    break

        return result
```







## Complexity

* Time:
  $O(Q \times D \times N)$
  where
  $Q$ = number of queries
  $D$ = number of dictionary words
  $N$ = length of each word

* Space:
  $O(1)$ extra space (excluding output)







## Edge Cases

1. Empty queries list
   → Return empty list

2. All words completely different
   → No matches, return empty list

3. Exact matches exist
   → Should count as 0 edits and be included

4. Words of length 1
   → At most 1 difference possible







## Lessons

* This is a classic "compare and count differences" pattern
* Early stopping (when diff > 2) improves efficiency
* Brute force is acceptable when constraints are small
* Always think: can I break early to save time?
