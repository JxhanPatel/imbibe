# 3043. Find the Length of the Longest Common Prefix 🟡

**Topics:** Trie, String, Hash Table


## Intuition

Imagine storing phone numbers digit by digit in a contact directory.

For example, if we store:

* `1234`
* `1299`

Both numbers share the path:

`1 -> 2`

After that, their paths split.

A Trie works exactly like this.
Each digit becomes a path in a tree.

Now instead of comparing every number in `arr1` with every number in `arr2`, we:

1. Store all numbers from `arr1` inside a Trie
2. Traverse the Trie using numbers from `arr2`
3. Count how long the matching path continues

The deepest valid path gives the longest common prefix length.



## Approach

We use a Trie to efficiently store prefixes.

### Step 1: Build the Trie

For every number in `arr1`:

* Convert it into a string
* Insert each digit into the Trie
* Create nodes if they do not exist

Example:

```text
123
```

creates:

```text
1 -> 2 -> 3
```



### Step 2: Search using `arr2`

For every number in `arr2`:

* Start from the Trie root
* Traverse digit by digit
* Stop when a digit is missing
* Count how many digits matched

Track the maximum matched length.



## Example Walkthrough

### Input

```python
arr1 = [123, 456, 12]
arr2 = [1299, 457, 1]
```



### Build Trie from `arr1`

Stored paths:

```text
1 -> 2 -> 3
4 -> 5 -> 6
1 -> 2
```



### Check `1299`

* `1` exists
* `2` exists
* `9` does not exist

Matched length = `2`



### Check `457`

* `4` exists
* `5` exists
* `7` does not exist

Matched length = `2`



### Check `1`

* `1` exists

Matched length = `1`



Maximum prefix length = `2`



## Code

```python
class TrieNode:
    def __init__(self):
        self.children = {}


class Solution:
    def longestCommonPrefix(self, arr1: List[int], arr2: List[int]) -> int:

        root = TrieNode()

        for num in arr1:
            cur=root


            for ch in str(num):
                if ch not in cur.children:
                    cur.children[ch] = TrieNode()

                cur = cur.children[ch]

        best=0

        for num in arr2:
            cur = root
            length=0

            for ch in str(num):
                if ch not in cur.children:
                    break

                cur = cur.children[ch]
                length+= 1

            if length > best:
                best=length

        return best
```



## Complexity

### Time Complexity

Building Trie:

$$
O\left(\sum \text{digits in arr1}\right)
$$

Searching:

$$
O\left(\sum \text{digits in arr2}\right)
$$

Overall:

$$
O(n + m)
$$

where:

- (n) = total digits in `arr1`
- (m) = total digits in `arr2`



### Space Complexity

$$
O(n)
$$

where (n) is the total number of digits stored in the Trie.




## Edge Cases

### 1. No common prefix

```python
arr1 = [123]
arr2 = [999]
```

Answer:

```python
0
```




### 2. Exact same numbers

```python
arr1 = [456]
arr2 = [456]
```

Answer:

```python
3
```




### 3. One number is a prefix of another

```python
arr1 = [12]
arr2 = [12345]
```

Answer:

```python
2
```




## Lessons

* Trie is extremely useful for prefix-based problems.
* Converting integers to strings makes digit traversal simple.
* Instead of comparing every pair, store prefixes once and reuse them.
* Tries reduce repeated work and make prefix searching efficient.
* Prefix problems often become much cleaner with tree-like structures.
