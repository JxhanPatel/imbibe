# 1848. Minimum Distance to the Target Element 🟢

**Topics:** Array, Linear Search





## Intuition

Imagine you're standing in a long hallway, and someone hid a $20 bill somewhere in it. You're standing at spot `start`. There might be multiple copies of the bill (multiple places where `nums[i] == target`). Your job is to figure out which copy is closest to where you're standing right now, and return how many steps away it is.

You don't need any fancy strategy. Just walk through every spot in the hallway, check if it has the bill, and if it does, note how far away you are. Keep updating your "best so far" distance as you go.







## Approach

We loop through the entire array once. Whenever we find `nums[i] == target`, we compute `abs(i - start)` and keep track of the minimum distance seen so far. That minimum is our answer.

No sorting. No binary search. Just one clean pass through the array.

Why does this work? The problem guarantees `target` exists in `nums`, so we'll always find at least one match. We just want the one closest to `start`.






## Example Walkthrough

`nums = [1, 2, 3, 4, 5]`, `target = 5`, `start = 3`

We walk through each index. Indices 0 through 3 don't have a `5`. At index 4, we find `nums[4] == 5`. The distance is `abs(4 - 3) = 1`. That's the only match, so our answer is `1`.






## Code

```python
def getMinDistance(nums, target, start):
    min_dist = float('inf')  # start with "infinitely far"

    for i, val in enumerate(nums):
        if val == target:
            dist = abs(i - start)
            min_dist = min(min_dist, dist)

    return min_dist
```







## Complexity

**Time:** $O(n)$ — we visit every element once.

**Space:** $O(1)$ — we only store the current minimum. No extra data structures.





## Edge Cases

**1. Target is at the start index itself**
`nums = [5, 3, 1]`, `target = 5`, `start = 0` → answer is `0`. Your `abs(0 - 0)` correctly gives zero. No special handling needed.

**2. Only one element in the array**
`nums = [7]`, `target = 7`, `start = 0` → the loop runs once, computes `abs(0 - 0) = 0`. Works fine.

**3. Multiple occurrences of target**
`nums = [1, 1, 1, 1, 1]`, `target = 1`, `start = 0` → every index is a match, but `abs(0 - 0) = 0` wins immediately. The running minimum handles this without any extra logic.

**4. Target only appears at the far end**
`nums = [9, 9, 9, 1]`, `target = 1`, `start = 0` → only one match at index 3, distance is `3`. Works as expected.





## Lessons

**Running minimum pattern** is one of the most useful tools in your coding toolkit. Instead of collecting all results and then finding the min, you update the best answer as you go. This saves memory and is very clean.

**Don't overthink Easy problems.** When constraints are small (array length up to 1000 here), a simple $O(n)$ pass is perfectly fine. Reach for the obvious approach first.

**`float('inf')` as a starting value** is a clean way to initialize a minimum tracker when you don't know what the first valid value will be. Any real number will beat it.
