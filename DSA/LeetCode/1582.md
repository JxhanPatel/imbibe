## 1582. Special Positions in a Binary Matrix 🟢

**Topics:** Array, Matrix

It is a fantastic exercise in optimizing how we search through 2D grids!

### Intuition

Think of this problem like trying to find a "lone wolf" employee in a massive open floor plan of cubicles. A position is special if an employee (a `1`) is sitting there, and they are the absolutely only person in their entire row of desks, as well as the only person in their entire column.

If you were a manager looking for these lone wolves, you could walk up to every single person and look down their row and column. That takes a lot of time. A much smarter way is to ask the building supervisor to give you a quick tally of exactly how many people are in each row and each column. Once you have those cheat sheets, you can just walk up to an employee and check if their row's total count is exactly 1 and their column's total count is exactly 1.

### Approach

We will use a technique called **Precomputation**.

Instead of checking the surrounding row and column every time we spot a `1`, we will split our work into two distinct phases:

1. **Phase One (The Tally):** We create two simple lists. One to track the total number of `1`s in each row, and another to track the total number of `1`s in each column. We loop through the matrix once to fill these lists.
2. **Phase Two (The Check):** We loop through the matrix one more time. Whenever we find a `1`, we look at our precomputed lists. If the tally for that specific row is exactly `1` and the tally for that specific column is exactly `1`, we know we found a special position!

### Example Walkthrough

Let us trace through Example 1 where `mat = [[1,0,0], [0,0,1], [1,0,0]]`.

**Step 1: The Tally**

* Row 0 has one `1`. Row 1 has one `1`. Row 2 has one `1`. Our row totals are `[1, 1, 1]`.
* Column 0 has two `1`s. Column 1 has zero `1`s. Column 2 has one `1`. Our column totals are `[2, 0, 1]`.

**Step 2: The Check**

* We look at `mat[0][0]`. It is a `1`. We check our totals. The row total is 1, but the column total is 2. Not special!
* We look at `mat[1][2]`. It is a `1`. The row total is 1, and the column total is 1. This is a special position! We increment our count.
* We look at `mat[2][0]`. It is a `1`. The row total is 1, but the column total is 2. Not special!

Our final count is exactly 1.

### Code

```python
class Solution:
    def numSpecial(self, mat: List[List[int]]) -> int:

        total_rows = len(mat)
        total_cols = len(mat[0])
        

        # lists to hold our precomputed tallies...
        ones_in_row = [0] * total_rows
        ones_in_col = [0] * total_cols
        


        # 1: Count the ones in every row and column
        for row_index in range(total_rows):

            for col_index in range(total_cols):

                if mat[row_index][col_index] == 1:
                    ones_in_row[row_index] += 1
                    ones_in_col[col_index] += 1
                    
        special_positions_count = 0
        



        # 2: Check each '1' against our tallies
        for row_index in range(total_rows):

            for col_index in range(total_cols):
                
                is_current_cell_one = (mat[row_index][col_index] == 1)
                is_only_one_in_row = (ones_in_row[row_index] == 1)
                is_only_one_in_col = (ones_in_col[col_index] == 1)
                
                if is_current_cell_one and is_only_one_in_row and is_only_one_in_col:
                    special_positions_count += 1
                    
        return special_positions_count

```

### Complexity

* **Time Complexity:** $O(m \times n)$ where $m$ is the number of rows and $n$ is the number of columns. We visit every cell in the matrix exactly twice, which is highly efficient.
* **Space Complexity:** $O(m + n)$ because we created two extra lists to store the sums of the rows and the columns.

### Edge Cases

When writing production code, it is always smart to think about weird inputs. Here are a few to consider:

1. **A matrix entirely filled with zeros:** Our code handles this gracefully. The tallies will remain zero, and the second phase will simply skip over everything.
2. **A matrix entirely filled with ones:** Every tally will be equal to the length of the row or column. No cell will pass the `== 1` check, returning 0 correctly.
3. **A tiny 1x1 matrix:** An input like `[[1]]` will correctly compute row and column tallies of 1, outputting 1.

### Lessons

The biggest takeaway here is the **Precomputation Pattern**. Whenever you find yourself doing repetitive work inside a nested loop, ask yourself if you can do a preliminary sweep to gather information first. We traded a tiny amount of memory to dramatically speed up our execution time. This is a very common and powerful trick in algorithmic problem-solving.
