# 2812. Find the Safest Path in a Grid 🟡

**Topics:** Breadth-First Search (BFS), Binary Search, Matrix, Graph

## Intuition

Imagine a city where several police stations are tracking the distance to the nearest thief.

Instead of asking every house, "How far is the closest thief?", all the thieves start spreading their distance outward at the same time. The first time a house is reached, we know its shortest distance to a thief.

Now every cell has a **safety score**.

The second part of the problem is deciding whether we can travel from the start to the destination while never stepping on a cell whose safety score is below some value.

Suppose we guess that the minimum safety should be `3`.

* If such a path exists, then a path with minimum safety `2` or `1` must also exist.
* If it does not exist for `3`, it cannot exist for `4` or higher.

This monotonic behavior makes the problem perfect for **binary search**.


## Approach

We solve the problem in two phases.

### Phase 1: Compute the safety of every cell

We perform a **multi-source BFS**.

* Add every thief to the queue.
* Their distance is `0`.
* Expand outward one level at a time.
* Every newly visited cell gets its shortest distance from any thief.

After this BFS,

```text
dist[r][c] = shortest distance from (r, c) to any thief
```


### Phase 2: Binary search the answer

Now we binary search the maximum possible safeness factor.

For every candidate value `mid`:

* Ignore every cell whose safety is smaller than `mid`.
* Run another BFS from `(0,0)`.
* If we can reach `(n-1,n-1)`, then `mid` is achievable.
* Otherwise, it is too large.

We keep increasing the answer until no larger safeness is possible.


## Example Walkthrough

**Input**

```text
grid =
[
 [0,0,1],
 [0,0,0],
 [0,0,0]
]
```

### Step 1: Multi-source BFS

The thief starts at `(0,2)`.

The computed distance matrix becomes

```text
[
 [2,1,0],
 [3,2,1],
 [4,3,2]
]
```

Each value represents the distance to the nearest thief.


### Step 2: Binary Search

Try minimum safety = `2`

Allowed cells:

```text
[
 [2,X,X],
 [3,2,X],
 [4,3,2]
]
```

A path exists.


Try minimum safety = `3`

Allowed cells:

```text
[
 [X,X,X],
 [3,X,X],
 [4,3,X]
]
```

The starting cell is not allowed.

No valid path exists.

Therefore,

```text
Answer = 2
```


## Code

```python
from collections import deque

class Solution:
    def maximumSafenessFactor(self, grid: List[List[int]]) -> int:

        n = len(grid)

        dist = [[-1]*n for _ in range(n)]

        q = deque((r, c) for r in range(n) for c in range(n) if grid[r][c])

        for r, c in q:
            dist[r][c] = 0

        dirs = [(1, 0), (-1, 0), (0, 1), (0, -1)]

        while q:
            r, c = q.popleft()

            for dr, dc in dirs:
                nr = r+dr
                nc = c+dc

                if 0 <= nr < n and 0 <= nc < n and dist[nr][nc] == -1:
                    dist[nr][nc] = dist[r][c]+1
                    q.append((nr, nc))

        def can_reach(min_safety):
            if dist[0][0] < min_safety:
                return False

            seen = {(0, 0)}
            stack = deque([(0, 0)])
            target = (n-1, n-1)

            while stack:
                cell = stack.popleft()

                if cell==target:
                    return True

                r, c = cell
                for dr, dc in dirs:
                    nr, nc = r + dr, c + dc
                    if not (0 <= nr < n and 0 <= nc < n):
                        continue

                    if (nr, nc) in seen or dist[nr][nc] < min_safety:
                        continue

                    seen.add((nr, nc))
                    stack.append((nr, nc))
            return False

        lo, hi, best = 0, max(v for row in dist for v in row), 0

        while lo <= hi:
            mid = (lo+hi)//2

            if can_reach(mid):
                best = mid
                lo = mid+1

            else:
                hi = mid-1

        return best
```


## Complexity

### Time Complexity

Computing the distance matrix using multi-source BFS:

$$
O(n^2)
$$

Each binary search iteration performs one BFS:

$$
O(n^2)
$$

Binary search runs over the range of possible distances, which is at most proportional to the grid size.

Overall:

$$
O(n^2 \log n)
$$


### Space Complexity

The distance matrix stores one value per cell:

$$
O(n^2)
$$

The BFS queue and visited set together also require:

$$
O(n^2)
$$

Overall:

$$
O(n^2)
$$


## Edge Cases

### 1. Start cell is a thief

```text
[
 [1,0],
 [0,0]
]
```

The start has safety `0`, so the answer is `0`.


### 2. Destination is a thief

```text
[
 [0,0],
 [0,1]
]
```

Since every path must end on a thief, the maximum safeness factor is `0`.


### 3. Single-cell grid

```text
[
 [1]
]
```

The only cell is both the start and the destination.

Answer:

```text
0
```


### 4. Many thieves

When thieves are spread throughout the grid, the distance matrix is still computed efficiently because all thieves expand simultaneously in a single BFS.


## Lessons

* **Multi-source BFS** is the fastest way to compute the shortest distance from every cell to the nearest source.
* When the answer is monotonic, **binary search on the answer** is often an excellent strategy.
* Separate the problem into two independent parts:

  1. Precompute useful information.
  2. Use that information to efficiently check whether a candidate answer is feasible.
* Combining preprocessing with a feasibility check is a common pattern in graph and optimization problems.
