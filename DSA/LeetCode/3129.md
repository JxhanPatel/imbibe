# 3129. Find All Possible Stable Binary Arrays I 🟡 
**Topics:** Dynamic Programming, Prefix Sum, Combinatorics





## **Intuition:**
Imagine you are organizing seating in a single row for two rival sports teams: Team Zero and Team One. You have exactly `zero` fans from Team Zero and `one` fans from Team One. To prevent arguments, stadium security has placed a strict rule: you cannot seat more than `limit` fans of the same team consecutively.

If we place fans seat by seat, we have a choice at every step: seat a Team Zero fan or a Team One fan. The number of valid seating arrangements depends on three things: how many Team Zero fans we've seated, how many Team One fans we've seated, and which team the *last* seated person belonged to.







## **Approach:**
We will use Dynamic Programming (DP). A naive approach would be to add blocks of 0s or 1s, but that requires an extra loop and slows down the solution. Instead, we can build the array one character at a time using a clever "inclusion-exclusion" trick.

Let's define a 3D array: `dp[i][j][k]`, where:

* `i` is the number of 0s used.
* `j` is the number of 1s used.
* `k` is the last number placed (`0` or `1`).

**The Transition Logic:**
To find `dp[i][j][0]` (arrays ending in 0), we can take *any* valid array of length `i + j - 1` and append a `0`. The number of such arrays is simply `dp[i-1][j][0] + dp[i-1][j][1]`.

However, adding this `0` might break the security rule! It breaks the rule if the array we appended to *already* ended in exactly `limit` consecutive zeros. What do those invalid arrays look like? They must have ended with a `1` followed by exactly `limit` zeros.

The number of these specific invalid arrays is exactly the number of valid arrays before those `limit` zeros were placed, which is `dp[i - limit - 1][j][1]`. We just subtract this invalid count to keep our array stable. The same logic applies symmetrically for `dp[i][j][1]`.






## **Example Walkthrough:**

Let's trace `zero = 1, one = 2, limit = 1`.

* **Base cases:** * `dp[1][0][0] = 1` (The array `[0]`)
* `dp[0][1][1] = 1` (The array `[1]`)
* `dp[0][2][1] = 0` (The array `[1, 1]` is invalid because limit is 1)


* Step 1: `i = 1`, `j = 1**`
* `dp[1][1][0]`: Appending `0` to previous valid states. We take `dp[0][1][0] + dp[0][1][1] = 0 + 1 = 1`. No limit exceeded yet. (Valid array: `[1, 0]`)
* `dp[1][1][1]`: Appending `1`. We take `dp[1][0][0] + dp[1][0][1] = 1 + 0 = 1`. (Valid array: `[0, 1]`)


* Step 2: `i = 1`, `j = 2**`
* `dp[1][2][0]`: Appending `0` to states with 0 zeros and 2 ones. `dp[0][2][0] + dp[0][2][1] = 0 + 0 = 0`.
* `dp[1][2][1]`: Appending `1` to states with 1 zero and 1 one. `dp[1][1][0] + dp[1][1][1] = 1 + 1 = 2`.
* *Wait!* We added a `1` and now `j = 2`, which is greater than `limit = 1`. We must subtract the invalid cases where we formed `[0, 1, 1]`. The invalid cases come from `dp[1][2 - 1 - 1][0] = dp[1][0][0] = 1`.
* Correction: `2 - 1 = 1`. (Valid array: `[1, 0, 1]`)



Total valid arrangements: `dp[1][2][0] + dp[1][2][1] = 0 + 1 = 1`.






## **Code:** 

```python

class Solution:
    def numberOfStableArrays(self, zero: int, one: int, limit: int) -> int:
        MOD = 10**9 + 7


        dp = [[[0, 0] for _ in range(one + 1)] for _ in range(zero + 1)]
        

        for i in range(1, min(zero, limit) + 1):
            dp[i][0][0] = 1
            

        for j in range(1, min(one, limit) + 1):
            dp[0][j][1] = 1
            
            
        # Fill the DP table
        for i in range(1, zero + 1):
            for j in range(1, one + 1):

                dp[i][j][0] = (dp[i-1][j][0] + dp[i-1][j][1]) % MOD

                if i > limit:
                    dp[i][j][0] = (dp[i][j][0] - dp[i - limit - 1][j][1] + MOD) % MOD
                

                dp[i][j][1] = (dp[i][j-1][0] + dp[i][j-1][1]) % MOD

                if j > limit:
                    dp[i][j][1] = (dp[i][j][1] - dp[i][j - limit - 1][0] + MOD) % MOD
                    

        return (dp[zero][one][0] + dp[zero][one][1]) % MOD

```



## **Complexity:** 
**Time Complexity:** $O(\text{zero} \times \text{one})$. We use a nested loop iterating through `zero` and `one`. All state transitions take constant $O(1)$ time. 

**Space Complexity:** $O(\text{zero} \times \text{one})$. We are storing states in a 3D array of size `(zero + 1) * (one + 1) * 2`. 





## **Edge Cases:** 
* `limit` is larger than both `zero` and `one`. The subtraction logic `i > limit` will never trigger, naturally degrading into standard permutations of binary strings.
* `limit == 1`. This forces strictly alternating arrays like `[1, 0, 1, 0]`. The subtraction aggressively strips out any adjacent duplicates.
* Impossible constraints where `zero` is vastly larger than `one` combined with a small `limit` (for example `zero = 10, one = 1, limit = 2`). The DP smoothly resolves to `0` without out-of-bounds errors.




## **Lessons:**
* **Inclusion-Exclusion in DP:** Instead of looping $O(\text{limit})$ times to build blocks of different sizes, you can often "include" everything from the previous step and "exclude" the single specific boundary case that makes it invalid. This mathematically flattens an $O(n^3)$ problem into an $O(n^2)$ problem.
* **Modulo Subtraction Guard:** Notice the `+ MOD` in `(val - sub + MOD) % MOD`. While Python naturally handles negative modulos gracefully, using this pattern is a universal best practice in competitive programming to avoid unexpected negative bugs in other languages like C++ or Java.
