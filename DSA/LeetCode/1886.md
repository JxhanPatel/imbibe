# 1886. Determine Whether Matrix Can Be Obtained By Rotation 🟢
**Topics:** Array, Matrix, Simulation



## Intuition

Imagine you're holding a square photo and comparing it to one pinned on a wall. You want to know if your photo can match the one on the wall just by rotating it. You don't need to flip it or do anything fancy, just spin it in place, 90 degrees at a time.

That's the entire problem. You have at most 4 possible positions (0°, 90°, 180°, 270°). After a full 360°, you're back to the start. So the answer is simple: try all four rotations and check if any of them match the target.



## Approach

We rotate `mat` by 90 degrees clockwise, up to 4 times. After each rotation (including the 0° original), we compare it with `target`. If any version matches, we return `True`.

The key operation here is knowing how to rotate a matrix 90 degrees clockwise. For an `n x n` matrix, the element at position `[row][col]` moves to `[col][n - 1 - row]`. A clean Pythonic way to do this is:

```
rotated = [list(row) for row in zip(*matrix[::-1])]
```

This line first reverses the rows of the matrix, then transposes it (swaps rows and columns using `zip`). That combination gives a 90 degree clockwise rotation.

We repeat this up to 4 times, checking for a match each time.



## Example Walkthrough

Let's use Example 3:
```
mat    = [[0,0,0],[0,1,0],[1,1,1]]
target = [[1,1,1],[0,1,0],[0,0,0]]
```

**Rotation 0 (original):**
```
0 0 0
0 1 0
1 1 1
```
Does not match target.

**Rotation 1 (90° clockwise):**
```
1 0 0
1 1 0
1 0 0
```
Does not match target.

**Rotation 2 (180°):**
```
1 1 1
0 1 0
0 0 0
```
This matches target! Return `True`.



## Code (Py)

```python
class Solution:
    def findRotation(self, mat: List[List[int]], target: List[List[int]]) -> bool:

        def rotate_90(matrix):
            return [list(row) for row in zip(*matrix[::-1])]
        

        # trying all 4 possible rotations (0, 90, 180, 270 degrees)
        for _ in range(4):
            if mat == target:
                return True
                
            mat = rotate_90(mat)
        

        return False
```



## Complexity

**Time Complexity:** $O(n^2)$

We perform at most 4 rotations, and each rotation + comparison touches all $n \times n$ cells. So it is $O(4 \cdot n^2)$ which simplifies to $O(n^2)$.

**Space Complexity:** $O(n^2)$

The `rotate_90` function creates a new matrix each time, so we use $O(n^2)$ extra space for each rotated copy.



## Edge Cases

**1. mat and target are already equal**
No rotation is needed. The check at `_ = 0` (before any rotation) catches this immediately and returns `True`.

**2. mat and target are 1x1 matrices**
With a single cell, any rotation produces the same matrix. If `mat[0][0] == target[0][0]`, return `True`. Otherwise `False`. This works naturally with our loop.

**3. mat cannot match target in any rotation**
For example if `mat` has three 1s and `target` has four 1s. No rotation changes the number of 1s in the matrix. Our 4 rotations all fail to match, and we return `False`. The code handles this cleanly, though it is worth knowing that checking the count of 1s beforehand could be used as an early exit optimization in practice.



## Lessons

**1. Bounded brute force is totally valid.** When the search space is tiny and fixed (here, exactly 4 states), it is completely fine to try every possibility. You do not need a clever trick when exhaustive search is cheap.

**2. Know your rotation formula.** `zip(*matrix[::-1])` is one of those one-liners worth memorizing. Reversing rows then transposing equals 90 degrees clockwise. It comes up in many matrix problems.

**3. Build reusable helpers.** Wrapping the rotation in its own function keeps the main logic clean. This habit pays off in interviews and on the job alike.

**4. Mutation awareness.** Notice we reassign `mat = rotate_90(mat)` rather than modifying in place. Since `rotate_90` creates a new matrix each time, we never accidentally corrupt the original, which avoids subtle bugs.
