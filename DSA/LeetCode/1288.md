# 1288. Remove Covered Intervals 🟡

**Topics:** Array, Sorting

## Intuition

Imagine you have a collection of umbrellas.

Some umbrellas are so large that they completely cover smaller ones placed underneath them. Since those smaller umbrellas do not provide any additional coverage, you can simply remove them.

Intervals work the same way. If one interval completely contains another, the smaller interval does not contribute anything new, so we remove it.

The challenge is efficiently determining which intervals are covered without comparing every pair.


## Approach

We first **sort the intervals** using two rules:

1. Sort by the starting point in ascending order.
2. If two intervals have the same starting point, sort by the ending point in descending order.

Why this sorting?

* Smaller starting points come first, making potential covering intervals appear before the intervals they may cover.
* When two intervals start at the same position, placing the longer interval first ensures it can cover the shorter one.

After sorting, we only need to keep track of the **largest ending point seen so far**.

For each interval:

* If its ending point is **less than or equal to** the maximum end seen so far, it is covered.
* Otherwise, it is not covered, so we count it and update the maximum ending point.

This avoids comparing every interval with every other interval.


## Example Walkthrough

**Input**

```text
intervals = [[1,4],[3,6],[2,8]]
```

### Step 1: Sort

```text
[[1,4],[2,8],[3,6]]
```

Initialize:

```text
max_end = 0
count = 0
```

### Step 2: Traverse

| Interval | max_end | Covered? | count |
| -------- | ------- | -------- | ----- |
| [1,4]    | 0       | No       | 1     |
| [2,8]    | 4       | No       | 2     |
| [3,6]    | 8       | Yes      | 2     |

Only `[3,6]` is covered by `[2,8]`.

**Output**

```text
2
```


## Code

```python
class Solution:
    def removeCoveredIntervals(self, intervals: List[List[int]]) -> int:

        intervals.sort(key=lambda iv: (iv[0], -iv[1]))

        remaining=0
        farthest_end=0

        for lo, hi in intervals:
            
            if hi > farthest_end:
                remaining += 1
                farthest_end = hi

        return remaining
```

**Time Complexity**

* Sorting:

$$
O(n \log n)

O(n)

$$

$$

O(1)
$$

Ignoring the space used by Python's sorting algorithm.


## Edge Cases

1. **Single interval**

```text
[[1,5]]
```

Answer:

```text
1
```


2. **Intervals with the same starting point**

```text
[[1,5],[1,3]]
```

After sorting, `[1,5]` comes first, so `[1,3]` is correctly identified as covered.

Answer:

```text
1
```


3. **No intervals are covered**

```text
[[1,2],[2,3],[3,4]]
```

Every interval extends beyond the previous one.

Answer:

```text
3
```


## Lessons

* Custom sorting can simplify interval problems dramatically.
* Sorting by increasing start and decreasing end guarantees potential covering intervals appear before covered ones.
* Keeping track of the maximum ending point seen so far is enough to detect covered intervals.
* Many interval problems become much easier after choosing the right sorting order.
