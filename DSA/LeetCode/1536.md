# **1536. Minimum Swaps to Arrange a Binary Grid** 🟡 
**Topics:** Greedy, Array  

---

## Intuition

Imagine you're organizing books on a shelf, but with a twist. Each book (row) has a certain "height" of content sticking out to the right. For the shelf to look tidy, each book from top to bottom needs to have less and less content sticking out. Specifically, the top book can have a lot sticking out, but as you go down, each subsequent book must have its content end earlier and earlier.

If a book has too much content sticking out for its position, you need to swap it with books below until you find one that fits. The question is: what's the minimum number of swaps to get everything in order?

---

## Approach

**The Key Insight:** For a valid grid, row `i` (0-indexed) needs all zeros from position `i+1` to the end. This means row `i` can have at most `i` ones at the end, or equivalently, the last `n - 1 - i` positions must be zero.

So we precompute for each row: "how many consecutive zeros are at the end?" Let's call this `trailing_zeros[i]`.

For row `i` to be valid, we need `trailing_zeros[row] >= n - 1 - i`.

**Greedy Strategy:** For each position `i` from top to bottom, find the first row at or below `i` that satisfies the requirement. Swap it up to position `i`. This is optimal because:
- We must place some valid row at position `i`
- Bringing the closest valid row minimizes swaps
- This doesn't hurt future placements (rows we skip will still be available below)

---

## Example Walkthrough

**Input:** `grid = [[0,0,1],[1,1,0],[1,0,0]]`

**Step 1:** Calculate trailing zeros for each row:
- Row 0: `[0,0,1]` → ends with `1`, so `0` trailing zeros
- Row 1: `[1,1,0]` → ends with `0`, so `1` trailing zero  
- Row 2: `[1,0,0]` → ends with `0,0`, so `2` trailing zeros

Trailing zeros: `[0, 1, 2]`

**Step 2:** Requirements for each position (need `trailing_zeros >= n - 1 - i`):
- Position 0 (top): need `>= 2` zeros
- Position 1: need `>= 1` zero
- Position 2: need `>= 0` zeros

**Step 3:** Greedy placement:
- **i=0:** Need `>=2`. Look at rows 0,1,2. Row 2 has 2. Swap row 2 to position 0: 2 swaps. Grid becomes: rows `[2,0,1]`, values `[2,0,1]`
- **i=1:** Need `>=1`. Look at rows 1,2 (originally rows 0,1). Row 1 (original row 0) has 0, row 2 (original row 1) has 1. Swap row 2 to position 1: 1 swap. 
- **i=2:** Done!

Total: 3 swaps. ✓

## Code

```python
class Solution:
    def minSwaps(self, grid: List[List[int]]) -> int:
        
        n = len(grid)
        x = [] # x = Trailing Zero

        # This tells us "how far right can this row extend with zeros"
        for row in grid:
            c = 0


            # Walking from right to left counting 0s
            for j in range(n - 1, -1, -1):
                if row[j] == 0:
                    c += 1
                else:
                    break
            x.append(c)
        

        swapz = 0
        

        # Greedy placement from top to bottom
        for i in range(n):
            needed = n - 1 - i
            

            # Find the first row at or below i that satisfies this
            found = False
            for j in range(i, n):
                if x[j] >= needed:

                    found = True
                    swapz += (j - i)
                
                    val = x.pop(j)
                    x.insert(i, val)
                    break
            
            if not found:
                return -1
        
        return swapz
```

---

##  Complexity

- **Time:** $O(n^2)$  
  - Computing trailing zeros: $O(n^2)$ in worst case (scanning each row)
  - Finding and swapping: $O(n^2)$ (for each of $n$ positions, we might scan $O(n)$ rows, and `insert` is $O(n)$)
  
- **Space:** $O(n)$ for storing the `trailing_zeros` array

---

## Edge Cases

| Case | Handling |
|------|----------|
| **Already valid grid** | Returns 0 (no swaps needed) |
| **n = 1** | Always valid, returns 0 (single cell is on diagonal) |
| **Impossible configuration** | Returns -1 when no row satisfies requirement for some position |
| **All rows identical** | Returns -1 if they don't all satisfy their respective positions |
| **Maximum n = 200** | $O(n^2) = 40,000$ operations, well within limits |

---

## Lessons

1. **Transform the problem:** Instead of thinking about the 2D grid, reduce each row to a single number (trailing zeros). This makes the problem much cleaner.

2. **Greedy is optimal for "bring to front" problems:** When you need to arrange items with minimum adjacent swaps, always pick the closest valid candidate. This is similar to problems where you need minimum swaps to group elements or sort with specific constraints.

3. **The "bubble up" pattern:** Counting swaps as `j - i` when moving an element from position `j` to `i` is a common technique. It works because each adjacent swap moves the element one position closer.

4. **Preprocessing pays off:** Computing trailing zeros once avoids repeated calculations and makes the logic crystal clear.

5. **Early termination:** As soon as we can't find a valid row for a position, we return -1 immediately rather than continuing unnecessarily.
