# 2515. Shortest Distance to Target String in a Circular Array 🟢

**Topics:** Array, String





## Intuition

Think of the array like people sitting around a round table. You are sitting at one position, and your goal is to find the closest person who says a specific word.

Since the table is circular, you can go clockwise or counterclockwise. The trick is to always take the shorter path between two positions on a circle.






## Approach

We will go through the array and check every index where the word equals `target`.

For each such index:

* Calculate distance moving forward (clockwise)
* Calculate distance moving backward (counterclockwise)

Because the array is circular:

* Forward distance = `abs(i - startIndex)`
* Backward distance = `n - abs(i - startIndex)`

Take the minimum of these two.

Finally, return the smallest distance found among all occurrences of `target`.

If the target is never found, return `-1`.






## Example Walkthrough

**Input:**
`words = ["hello","i","am","leetcode","hello"]`
`target = "hello"`
`startIndex = 1`

**Step 1:** Find indices of "hello" → `[0, 4]`

**Step 2:** Calculate distance from index 1:

* For index 0:

  * direct = `|0 - 1| = 1`
  * circular = `5 - 1 = 4`
  * min = `1`

* For index 4:

  * direct = `|4 - 1| = 3`
  * circular = `5 - 3 = 2`
  * min = `2`

**Step 3:** Answer = `min(1, 2) = 1`







## Code

```python
class Solution:
    def closestTarget(self, words: List[str], target: str, startIndex: int) -> int:

        n = len(words)
        ans = float('inf')
        
        for i in range(n):
            
            if words[i] == target:
                ans = min(ans, min(abs(i - startIndex), n - abs(i - startIndex)))
        
        return ans if ans != float('inf') else -1
```






## Complexity

* Time Complexity: $O(n)$
* Space Complexity: $O(1)$






## Edge Cases

1. Target does not exist → return `-1`
2. Only one element in array → check if it matches target
3. Multiple occurrences of target → must pick minimum distance
4. Start index already at target → answer is `0`






## Lessons

* Circular arrays often require comparing direct distance vs wrap-around distance
* Always think in terms of minimum of two paths in circular problems
* Brute force over small constraints like `n <= 100` is perfectly fine
* This pattern appears frequently in rotation and circular traversal problems
