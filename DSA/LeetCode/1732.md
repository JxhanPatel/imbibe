# 1732. Find the Highest Altitude 🟢

**Topics:** Array, Prefix Sum



## Intuition

Imagine you are hiking in the mountains with an altitude tracker.

You start at altitude `0`. At each step, your friend tells you how much your altitude changes:

* `+5` means climb 5 units.
* `-3` means descend 3 units.
* `0` means stay at the same altitude.

You do not need to record every altitude you have visited. Instead, you only need to know:

1. Your **current altitude**.
2. The **highest altitude** reached so far.

As you continue the hike, update your current altitude after each move and compare it with the highest altitude seen so far.





## Approach

We use two variables:

* `alt` stores the biker's current altitude.
* `peak` stores the highest altitude reached.

Algorithm:

1. Initialize both `alt` and `peak` to `0` because the biker starts at altitude `0`.
2. Traverse the `gain` array.
3. Add the current gain to `alt`.
4. If the new altitude is greater than `peak`, update `peak`.
5. After processing all values, return `peak`.

This gives us the answer in a single traversal without storing all altitudes.




## Example Walkthrough

**Input**

```text
gain = [-5,1,5,0,-7]
```

Initial state:

```text
alt = 0
peak = 0
```

| Gain | alt | peak |
| ---- | --: | ---: |
| -5   |  -5 |    0 |
| 1    |  -4 |    0 |
| 5    |   1 |    1 |
| 0    |   1 |    1 |
| -7   |  -6 |    1 |

The highest altitude reached is:

```text
1
```

**Output**

```text
1
```




## Code

```python
class Solution:
    def largestAltitude(self, gain: List[int]) -> int:

        peak=0
        alt=0

        for g in gain:
            alt+= g

            if alt>peak:
                peak=alt
                
        return peak
```




## Complexity

**Time Complexity**

$$
O(n)
$$

We visit each element of `gain` exactly once.




**Space Complexity**

$$
O(1)
$$

Only two variables are used.




## Edge Cases

1. **All gains are negative**

```text
gain = [-4,-3,-2,-1]
```

Altitudes:

```text
[0,-4,-7,-9,-10]
```

The biker never goes above the starting point.

Answer:

```text
0
```




2. **All gains are positive**

```text
gain = [1,2,3]
```

Altitudes:

```text
[0,1,3,6]
```

Answer:

```text
6
```




3. **Altitude never changes**

```text
gain = [0,0,0]
```

Altitudes:

```text
[0,0,0,0]
```

Answer:

```text
0
```




## Lessons

* Prefix sum problems do not always require an extra array. A running sum variable is often enough.
* When tracking the maximum or minimum value during traversal, update the answer on the fly.
* The starting state can itself be the answer. Here, altitude `0` may remain the highest altitude throughout the trip.
* Keeping only the necessary information often leads to simpler and more memory-efficient solutions.
