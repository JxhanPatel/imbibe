# 2161. Partition Array According to Given Pivot 🟡

**Topics:** Array, Two Pointers, Simulation

## Intuition

Imagine you're organizing people into three lines based on their height relative to a reference person:

* Shorter people go into the first line.
* People with the same height go into the second line.
* Taller people go into the third line.

One important rule is that people must keep their original ordering within their respective lines.

After forming the three lines, you simply place them one after another.

This problem works exactly the same way. We separate numbers into:

1. Less than `pivot`
2. Equal to `pivot`
3. Greater than `pivot`

Then concatenate the three groups while preserving their original order.

## Approach

We use three separate arrays:

* `smaller` for elements less than `pivot`
* `equal` for elements equal to `pivot`
* `greater` for elements greater than `pivot`

Traverse the array once:

* If `num < pivot`, add it to `smaller`
* If `num == pivot`, add it to `equal`
* Otherwise, add it to `greater`

Finally, return:

```text
smaller + equal + greater
```

Because we append elements in the order they appear, the relative ordering is automatically preserved.

## Example Walkthrough

**Input**

```text
nums = [9,12,5,10,14,3,10]
pivot = 10
```

### Step 1: Build the groups

| Number | Group   |
| ------ | ------- |
| 9      | smaller |
| 12     | greater |
| 5      | smaller |
| 10     | equal   |
| 14     | greater |
| 3      | smaller |
| 10     | equal   |

Now we have:

```text
smaller = [9,5,3]
equal = [10,10]
greater = [12,14]
```

### Step 2: Combine them

```text
[9,5,3] + [10,10] + [12,14]
```

**Output**

```text
[9,5,3,10,10,12,14]
```

## Code

```python
from typing import List

class Solution:
    def pivotArray(self, nums: List[int], pivot: int) -> List[int]:
        
        less= []
        more= []

        pivot_count=0


        for n in nums:
            if n < pivot:
                less.append(n)

            elif n > pivot:
                more.append(n)

            else:
                pivot_count+= 1

        return less + [pivot]*pivot_count + more
```

## Complexity

Traversal takes:

$$
O(n)
$$

Concatenation also takes:

$$
O(n)
$$

Overall time complexity:

$$
O(n)
$$

Space complexity:

$$
O(n)
$$

for the three auxiliary arrays.

## Edge Cases

1. **All elements equal to pivot**

```text
nums = [5,5,5]
pivot = 5
```

Output:

```text
[5,5,5]
```

2. **All elements smaller than pivot**

```text
nums = [1,2,3]
pivot = 10
```

Output:

```text
[1,2,3]
```

3. **All elements greater than pivot**

```text
nums = [8,9,10]
pivot = 5
```

Output:

```text
[8,9,10]
```

4. **Only one element**

```text
nums = [7]
pivot = 7
```

Output:

```text
[7]
```

## Lessons

* When elements must be grouped while preserving their relative order, separate containers are often the simplest solution.
* Stable partitioning can be achieved by collecting elements into different lists and merging them later.
* Sometimes using extra space leads to a cleaner and more efficient implementation than trying to rearrange elements in place.
* Always pay attention to ordering requirements, as they often rule out standard partition techniques like quicksort partitioning.
