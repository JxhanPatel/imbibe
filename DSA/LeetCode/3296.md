
# 3296. Minimum Number of Seconds to Make Mountain Height Zero 🟡
**Topics:** Binary Search, Greedy, Math



## Intuition

Imagine you're planning a group hiking trip to flatten a sand dune. Each person has different stamina levels (workerTimes). When someone works, they get progressively more tired: the first scoop of sand costs them 1 unit of energy, the second scoop costs 2 units, the third costs 3 units, and so on. 

The twist: everyone works **simultaneously**, and you want the entire dune flattened as fast as possible. This means you need to distribute the work so that no single person becomes the bottleneck. You're essentially asking: "What's the minimum time T such that all workers together can remove at least `mountainHeight` units of sand, with no worker exceeding time T?"



## Approach

We'll use **Binary Search on the Answer**. 

Why binary search? Because if all workers can finish in `T` seconds, they can certainly finish in `T+1` seconds. This monotonic property lets us binary search for the minimum feasible time.

For a given time limit `T`, we need to check: what's the maximum height each worker can reduce?

If worker `i` with time `t = workerTimes[i]` works for at most `T` seconds, what's the maximum `x` they can achieve?
- Time needed for `x` units: `t + 2t + 3t + ... + xt = t * (1 + 2 + ... + x) = t * x * (x+1) / 2`

We need: `t * x * (x+1) / 2 <= T`

Solving for `x` (approximately): `x^2 ≈ 2T/t`, so `x ≈ sqrt(2T/t)`

We can find the exact maximum `x` for each worker using binary search or the quadratic formula, then sum up all workers' contributions. If the total >= mountainHeight, time `T` is feasible.



## Example Walkthrough

**Input:** `mountainHeight = 4`, `workerTimes = [2, 1, 1]`

**Binary Search Range:** Low = 0, High = some large number (say, 10^18)

**Check T = 3:**
- Worker 0 (t=2): max x where `2*x*(x+1)/2 <= 3` → `x*(x+1) <= 3` → x=1 (1*2=2 <= 3, 2*3=6 > 3). Contributes 1.
- Worker 1 (t=1): max x where `1*x*(x+1)/2 <= 3` → `x*(x+1) <= 6` → x=2 (2*3=6 <= 6). Contributes 2.
- Worker 2 (t=1): same as worker 1, contributes 2.

Total = 1 + 2 + 2 = 5 >= 4. So T=3 is feasible. Try lower.

**Check T = 2:**
- Worker 0: x=1 (2*1*2/2=2 <= 2). Contributes 1.
- Worker 1: x=1 (1*1*2/2=1 <= 2, but x=2 gives 3 > 2). Contributes 1.
- Worker 2: contributes 1.

Total = 3 < 4. Not feasible.

**Answer:** 3 seconds.



## Code

```python
class Solution:
    def minNumberOfSeconds(self, mountainHeight: int, workerTimes: List[int]) -> int:

        def canFinishInTime(T: int) -> bool:

            total_height_reduced = 0
            
            for t in workerTimes:

                # Binary search for max x this worker can do
                low, high = 0, mountainHeight  # x can't exceed mountainHeight
                

                while low < high:
                    mid = (low + high + 1) // 2 
                    
                    # Time needed for mid units
                    time_needed = t * mid * (mid + 1) // 2
                    

                    if time_needed <= T:
                        low = mid  # can do mid units, try more
                    else:
                        high = mid - 1  # too much, try less
                
                total_height_reduced += low
                

                # Early exit if we've already achieved enough
                if total_height_reduced >= mountainHeight:

                    return True
            
            return total_height_reduced >= mountainHeight
        

        max_time = max(workerTimes)
        left, right = 0, max_time * mountainHeight * (mountainHeight + 1) // 2
        
        while left < right:
            mid = (left + right) // 2
            if canFinishInTime(mid):
                right = mid  # can finish, try less time
            else:
                left = mid + 1  # cannot finish, need more time
        
        return left
```



## Complexity


- **Time Complexity:** $O(n \cdot \log(\text{answer}) \cdot \log(H))$
  - We binary search on the answer (time)
  - For each time check, we look at all $n$ workers
  - For each worker, we binary search to find how much they can do
  
- **Space Complexity:** $O(1)$ extra space


## Edge Cases

1. **Single Worker:** When `len(workerTimes) = 1`, that worker must do all the work. The answer is simply `t * H * (H+1) / 2` where `H = mountainHeight`.

2. **Many Fast Workers:** When `mountainHeight = 10^5` and we have `10^4` workers each with `workerTimes[i] = 1`, the work distributes easily. Each worker only needs to do ~10 units, taking minimal time.

3. **One Slow Worker Among Fast Ones:** `[1000000, 1, 1, 1, ...]` with large mountain height. The slow worker might be ignored entirely if fast workers can handle it, or might be forced to contribute if height is large enough.

4. **Minimum Values:** `mountainHeight = 1`, `workerTimes = [1]`. Answer should be 1 second.



## Lessons

1. **Binary Search on Answer:** When asked for "minimum maximum" or "minimum time," and you can verify feasibility efficiently, binary search is often the key. The monotonic property (if T works, T+1 works) is the signal.

2. **Mathematical Simplification:** Recognizing the arithmetic series `1 + 2 + ... + x = x(x+1)/2` transforms the problem from simulation to direct calculation.

3. **Greedy Distribution:** For a fixed time limit, each worker should work as much as possible (up to their limit). There's no benefit in having a worker do less than their max if we need to maximize total work done.

4. **Quadratic Growth Awareness:** The time cost grows quadratically with units worked. This means workers quickly hit diminishing returns, favoring distributing work across many workers rather than overworking a few.
