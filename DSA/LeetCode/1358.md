# 1358. Number of Substrings Containing All Three Characters 🟡

**Topics:** String, Sliding Window, Hash Table, Two Pointers


## Intuition

Imagine you are collecting three different colored marbles: **a**, **b**, and **c**.

You keep adding marbles one by one to a box. As soon as the box contains at least one of each color, you've formed a **valid collection**.

Now comes the clever observation.

If the current window already contains all three characters, then extending the window further to the right will never make it invalid because all three characters are still present.

So instead of checking every possible substring individually, we can count all valid extensions at once. This is exactly what the sliding window technique allows us to do.


## Approach

We use two pointers to maintain a sliding window.

* `left` marks the beginning of the current window.
* `right` expands the window one character at a time.

A hash map (`chars`) stores the frequency of each character inside the current window.

### Steps

1. Expand the window by moving `right` and update the character count.
2. Check whether the window contains at least one `'a'`, `'b'`, and `'c'`.
3. While it does:

   * Every substring starting at `left` and ending at `right` or beyond is valid.
   * Add `len(s) - right` to the answer.
   * Remove the leftmost character and move `left` forward to search for the next valid window.
4. Continue until the entire string has been processed.

The condition

```python
all(chars[c] > 0 for c in 'abc')
```

simply checks whether all three required characters are present in the current window.

## Code

```python
from collections import defaultdict

class Solution:
    def numberOfSubstrings(self, s: str) -> int:
        
        chars = defaultdict(int)
        left=0
        result=0
        
        for right in range(len(s)):
            chars[s[right]] += 1

            
            while all(chars[c] > 0 for c in 'abc'):
                result += len(s)-right
                chars[s[left]] -= 1
                left+= 1
        
        return result
```


## Complexity

**Time Complexity**

$$
O(n)
$$

Although there is a nested `while` loop, each character is added to and removed from the sliding window at most once. Therefore, both pointers together move at most `n` times.

**Space Complexity**

$$
O(1)
$$

The hash map stores counts for only three possible characters (`a`, `b`, and `c`), so the extra space remains constant.


## Edge Cases

### 1. Smallest valid string



### 2. Repeated characters before completing the set

```text

```

Output:

```text

```


### 3. Missing one character completely

```text
s = "aaaaa"
```

No substring can contain all three characters.

Output:

```text
0
```


## Lessons

* Sliding window is a powerful technique for solving substring problems with conditions on the current window.
* Once a window satisfies the required condition, every larger window ending later is also valid.
* Instead of checking every substring individually, count all valid extensions in one step using `len(s) - right`.
* A frequency map makes it easy to maintain the current state of the window while moving both pointers efficiently.
