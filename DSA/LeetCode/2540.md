# 2540. Minimum Common Value 🟢

**Topics:** Array, Hash Table, Two Pointers, Binary Search




## Intuition

Imagine two people walking through two sorted lists of numbers while trying to find the first number they both have in common.

Because both lists are already sorted, we do not need to compare every pair.
We can move smartly:

* If one number is smaller, move forward in that list
* If both numbers are equal, we found the smallest common value immediately

This works because sorted arrays always move from smaller to larger values.




## Approach

We use the Two Pointers technique.

* Start one pointer at the beginning of `nums1`
* Start another pointer at the beginning of `nums2`
* Compare the current values:

  * If equal, return that value
  * If `nums1[i] < nums2[j]`, move `i`
  * Otherwise, move `j`

Since arrays are sorted, the first common value we encounter will automatically be the minimum common value.





## Example Walkthrough

Input:

```python
nums1 = [1, 2, 3, 6]
nums2 = [2, 3, 4, 5]
```

Step-by-step:

| i | j | nums1[i] | nums2[j] | Action                 |
| - | - | -------- | -------- | ---------------------- |
| 0 | 0 | 1        | 2        | 1 is smaller, move `i` |
| 1 | 0 | 2        | 2        | Equal, return `2`      |

Answer:

```python
2
```




## Code

```python
class Solution:
    def getCommon(self, nums1: List[int], nums2: List[int]) -> int:

        i=0
        j=0

        while i<len(nums1) and j<len(nums2):

            if nums1[i]==nums2[j]:
                return nums1[i]

            elif nums1[i]<nums2[j]:
                    i+= 1

            else:
                    j+= 1

        return -1
```



## Complexity

Time Complexity:

$$
O(n + m)
$$


where:

* $n = \text{len(nums1)}$
* $m = \text{len(nums2)}$

Space Complexity:

$$
O(1)
$$



## Edge Cases

1. No common element

```python
nums1 = [1, 2, 3]
nums2 = [4, 5, 6]
```

Output:

```python
-1
```

2. Common value at the beginning

```python
nums1 = [1, 2]
nums2 = [1, 3]
```

3. Arrays with repeated values

```python
nums1 = [1, 2, 2, 3]
nums2 = [2, 2, 4]
```

## Lessons

* Sorted arrays often hint toward Two Pointers
* Moving only the smaller pointer avoids unnecessary comparisons
* The first common value found is automatically the minimum because arrays are sorted
* Two Pointers can reduce brute force $O(n \times m)$ solutions to linear time
