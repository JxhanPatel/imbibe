# 657. Robot Return to Origin 🟢

**Topics:** String, Simulation



## Intuition

Think of it like a road trip where every mile you drive north, you need to drive south to get home. And every mile you drive east, you need one going west. For the robot to come back to where it started, every move in one direction needs to be perfectly cancelled out by a move in the opposite direction.

So we just count: did we go up as many times as we went down? Did we go left as many times as we went right? If both pairs balance out, we're home.



## Approach

We track two simple counters: one for the vertical axis (up/down) and one for the horizontal axis (left/right).

- `U` increments the vertical counter, `D` decrements it
- `L` increments the horizontal counter, `R` decrements it

At the end, if both counters are 0, the robot is back at the origin.

That's it. One pass through the string, two counters. No extra data structures, no math tricks.


## Example Walkthrough

Input: `"UDLR"`

| Step | Move | x | y |
|------|------|---|---|
| Start | — | 0 | 0 |
| 1 | U | 0 | +1 |
| 2 | D | 0 | 0 |
| 3 | L | -1 | 0 |
| 4 | R | 0 | 0 |

Both x and y are 0 at the end, so we return `True`.



## Code

```python
class Solution:
    def judgeCircle(self, moves: str) -> bool:
        
        x = 0
        y = 0

        for move in moves:
            if move == 'U':
                y += 1

            elif move == 'D':
                y -= 1

            elif move == 'L':
                x -= 1

            elif move == 'R':
                x += 1

        return x == 0 and y == 0
            
```



## Complexity

**Time:** $O(n)$ where $n$ is the length of `moves`. We look at each character once.

**Space:** $O(1)$ — just two integer counters regardless of input size.



## Edge Cases

**Single move:** `"U"` returns `False` since $y = 1 \neq 0$. No panic here, the logic handles it naturally.

**Already at origin with empty logic:** The constraints say `moves.length >= 1`, so an empty string won't appear. But if it did, both counters stay 0 and we'd correctly return `True`.

**All moves in one direction:** `"LLLL"` returns `False` because $x = -4$. There's nothing to cancel it out.

**Very long string:** Up to $2 \times 10^4$ characters. Since we're $O(n)$ time and $O(1)$ space, this is totally fine. No risk of TLE or MLE.



## Lessons

**The balance principle.** Many problems that look like they need complex tracking really just need you to verify that opposing forces cancel out. Count one direction, subtract the other, check if the net is zero.

**Axes are independent.** x and y don't interfere with each other. When you see a 2D movement problem, your first instinct should be: can I treat horizontal and vertical separately? Almost always, yes.

**Counters over data structures.** You might be tempted to use a set or a dictionary, but a couple of integers is almost always cleaner and faster when you're just tracking net movement.
