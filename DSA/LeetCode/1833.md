# 1833. Maximum Ice Cream Bars 🟡

**Topics:** Array, Greedy, Counting Sort

## Intuition

Imagine you are at an ice cream shop with a fixed budget.

You have two choices:

* Buy expensive ice cream first.
* Buy the cheapest ice cream first.

If your goal is to maximize the **number** of ice cream bars, buying expensive ones early is rarely a good idea because they consume a large part of your budget.

Instead, you should always buy the cheapest available bars first.

The problem asks us to use **counting sort**, which is perfect here because the prices are limited to at most `100000`.

Rather than sorting the entire array, we simply count:

```text
Price 1 -> 5 bars
Price 2 -> 3 bars
Price 3 -> 7 bars
...
```

Then we buy bars starting from the smallest price and keep going until we run out of coins.



## Approach

We combine **Counting Sort** with a **Greedy** strategy.

### Step 1: Count frequencies

Create a frequency array:

```python
freq = [0] * (max(costs) + 1)
```

Here,

```text
freq[i] = number of ice cream bars costing i coins
```

Populate it:

```python
for c in costs:
    freq[c] += 1
```



### Step 2: Buy bars from cheapest to costliest

Traverse every possible price:

```python
for price in range(1, len(freq)):
```

If there are no bars at this price:

```python
if not freq[price]:
    continue
```

If we cannot afford even one bar at this price, we are done:

```python
if coins < price:
    break
```

Otherwise:

* Maximum bars we can afford:

```python
coins // price
```

* Maximum bars available:

```python
freq[price]
```

We take the smaller of the two:

```python
take = min(freq[price], coins // price)
```

Update:

```python
bars += take
coins -= take * price
```

Continue until all prices are processed or coins run out.

## Code

```python
class Solution:
    def maxIceCream(self, costs: List[int], coins: int) -> int:

        freq = [0] * (max(costs)+1)

        for c in costs:
            freq[c] += 1

        bars=0

        for price in range(1, len(freq)):
            if not freq[price]:
                continue

            if coins<price:
                break

            take = min(freq[price], coins//price)
            bars += take
            coins -= take*price

        return bars
```



## Complexity

Let:

* $n$ = number of ice cream bars
* $m$ = maximum value in `costs`

### Time Complexity

Building the frequency array:

$$
O(n)
$$

Traversing all possible prices:

$$
O(m)
$$

Overall:

$$
O(n + m)
$$



### Space Complexity

The frequency array stores counts from `0` to `m`.

$$
O(m)
$$



## Edge Cases

1. **Cannot afford any ice cream**

```text
costs = [10,6,8,7,7,8]
coins = 5
```

We cannot buy even the cheapest bar.

Output:

```text
0
```



2. **Enough coins to buy all bars**

```text
costs = [1,6,3,1,2,5]
coins = 20
```

Total cost:

```text
1 + 6 + 3 + 1 + 2 + 5 = 18
```

Output:

```text
6
```



3. **All bars have the same cost**

```text
costs = [2,2,2,2]
coins = 5
```

We can buy:

```text
5 // 2 = 2
```

Output:

```text
2
```



## Lessons

* Counting Sort is useful when values lie in a limited range.
* Greedy strategies often work when choosing the cheapest option always leaves you with the most flexibility.
* Frequency arrays let us avoid sorting individual elements and work directly with counts.
* A useful pattern is:

```python
take = min(available, budget // cost)
```

This appears frequently in problems involving buying items with limited resources.
