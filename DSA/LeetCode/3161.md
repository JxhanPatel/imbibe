# 3161. Block Placement Queries 🔴

**Topics:** Binary Search, Segment Tree, Ordered Set, Offline Queries

## Intuition

Imagine the number line as a long road.

Obstacles divide the road into free segments. A block of size `sz` can be placed if there exists some continuous free segment of length at least `sz` inside `[0, x]`.

The challenge is that obstacles are added over time, and we must answer many queries efficiently.

A direct simulation would be too slow because there can be up to `150,000` queries.

The key observation:

* Obstacles only get added, never removed.
* The maximum block that fits in `[0, x]` depends on the largest gap between consecutive obstacles inside that range.
* We need fast updates when inserting obstacles and fast queries for the maximum available gap.

This suggests maintaining obstacle positions in sorted order and efficiently tracking segment lengths.





## Approach

### Reverse Processing

Instead of handling insertions directly, process queries **backwards**.

Why?

When reversing:

* Type 1 (add obstacle) becomes **remove obstacle**.
* Removing an obstacle merges two neighboring segments into one larger segment.

Merging intervals is much easier than splitting intervals.




### Data Structures

#### 1. Sorted obstacle positions

Store all obstacle positions plus `0`.

While moving backwards:

* Find the left neighbor.
* Find the right neighbor.
* Remove the obstacle.

#### 2. Segment Tree

For every obstacle position `p`, store:

`gap[p] = p - previous_obstacle`

The segment tree maintains the maximum gap among positions.

This allows:

* Update a gap in `O(log n)`
* Query maximum gap in prefix `[0, x]` in `O(log n)`




### Answering Type 2 Query

Suppose we need:

```text
[2, x, sz]
```

Let:

* `prev` = largest obstacle ≤ `x`

Then there are two possible places for the block:

#### Case 1

Inside a complete segment ending at some obstacle ≤ `x`.

The largest such segment is obtained from the segment tree.

#### Case 2

Inside the final partial segment:

```text
(prev, x)
```

whose length is:

```text
x - prev
```

Therefore:

```text
max_available =
max(
    largest_gap_before_x,
    x - prev
)
```

If:

```text
max_available >= sz
```

answer `true`.

Otherwise `false`.

## Code

```python
from bisect import bisect_left, bisect_right


class SegmentTree:
    def __init__(self, n):
        self.n=n
        self.tree = [0] * (4*n)



    def update(self, node, lo, hi, idx, val):
        if lo==hi:
            self.tree[node] = val
            return

        mid = (lo+hi)//2

        if idx <= mid:
            self.update(node*2, lo, mid, idx, val)

        else:
            self.update(node*2+1, mid+1, hi, idx, val)


        self.tree[node] = max(self.tree[node*2], self.tree[node*2+1])


    def query(self, node, lo, hi, ql, qr):
        if qr<lo or hi<ql:
            return 0

        if ql <= lo and hi <= qr:
            return self.tree[node]

        mid = (lo+hi)//2

        left_max = self.query(node*2, lo, mid, ql, qr)
        right_max = self.query(node*2 + 1, mid+1, hi, ql, qr)

        return max(left_max, right_max)



class Solution:
    def getResults(self, queries: List[List[int]]) -> List[bool]:
        obstacles = {0}
        max_x=0

        for q in queries:
            max_x = max(max_x, q[1])

            if q[0]==1:
                obstacles.add(q[1])


        coords = sorted(obstacles | {max_x})
        coord_idx = {x: i for i, x in enumerate(coords)}
        n=len(coords)

        seg=SegmentTree(n)

        for i in range(1, n):
            seg.update(1, 0, n-1, i, coords[i] - coords[i-1])

        active = coords[:]
        answers=[]

        for q in reversed(queries):

            if q[0]==2:
                x, sz = q[1], q[2]
                pos = bisect_right(active, x)-1
                nearest_left = active[pos]

                limit = bisect_right(coords, nearest_left)-1
                best = seg.query(1, 0, n-1, 0, limit)
                best = max(best, x - nearest_left)

                answers.append(best >= sz)


            else:
                x = q[1]
                pos = bisect_left(active, x)
                left_obs = active[pos-1]
                right_obs = active[pos+1] if pos+1 < len(active) else None

                if right_obs is not None:
                    seg.update(1, 0, n-1, coord_idx[right_obs], right_obs-left_obs)

                active.pop(pos)


        return answers[::-1]
```



## Complexity

Let `n = len(queries)`.

* Building structures: `O(n log n)`
* Each update: `O(log n)`
* Each query: `O(log n)`

Overall:

$$
O(n \log n)
$$

Space:

$$
O(n)
$$



## Edge Cases

1. Only one obstacle exists.

```python
[[1,5],[2,5,5]]
```

Block can exactly touch the obstacle.

2. Block size equals available gap.

```python
gap = 4
sz = 4
```

Valid because touching is allowed.

3. Query range ends before the next obstacle.

```python
obstacles = [0,10]
x = 7
```

Need to consider partial segment length `7`.

4. Obstacles very close together.

```python
0,1,2,3,...
```

Maximum gap may be only `1`.



## Lessons

* Reverse processing can transform difficult insertions into easy deletions.
* When intervals merge but never split, offline processing is often powerful.
* Segment trees are excellent for maintaining maximum values under updates.
* For interval problems, storing gap lengths is often more useful than storing intervals themselves.
* Always check both complete segments and the final partial segment when dealing with bounded ranges.
