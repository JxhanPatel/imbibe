# 3534. Path Existence Queries in a Graph II 🔴

**Topics:** Graph, Sorting, Two Pointers, Binary Lifting

## Intuition

Imagine a row of stepping stones placed in order of their heights.

You can only jump from one stone to another if their height difference is at most `maxDiff`. Since stones are arranged by height, every stone can only directly reach a **continuous block** of stones to its right.

Instead of checking every possible edge in the graph, we only need to know one thing:

> **How far can I reach in one jump?**

Once we know that, answering a query becomes similar to repeatedly making the biggest possible jumps until we reach the destination.

Since there can be up to `10^5` queries, repeatedly jumping one step at a time would be too slow. To speed things up, we precompute jump information using **binary lifting**, allowing us to skip multiple jumps at once.


## Approach

### Step 1: Sort the nodes

The graph depends only on the values in `nums`, not on their original positions.

So first, sort all node indices according to their values.

```text
order = sorted(range(n), key=lambda i: nums[i])
```

We also store each node's position in this sorted order.


### Step 2: Compute the farthest node reachable in one move

Using a **sliding window (two pointers)**:

* `left` marks the current node.
* `right` expands as long as the value difference is within `maxDiff`.

For every position, store:

```text
nxt_reach[i]
```

which represents the farthest sorted position reachable using exactly one edge.

Because both pointers only move forward, this step runs in linear time.


### Step 3: Binary lifting preprocessing

Create a jump table where

```text
jump[k][i]
```

represents the position reached after making `2^k` jumps from position `i`.

The first level is simply:

```text
jump[0][i] = nxt_reach[i]
```

Higher levels are built using the recurrence:

```text
jump[k][i] = jump[k-1][ jump[k-1][i] ]
```

This preprocessing allows us to simulate many jumps in logarithmic time.


### Step 4: Process each query

For every query:

1. Convert both nodes into their sorted positions.
2. Ensure we always move from left to right.
3. If both positions are the same, answer is `0`.
4. Otherwise, use binary lifting to make the largest possible jumps without crossing the destination.
5. If one final jump reaches the destination, return the number of jumps.
6. Otherwise, the nodes belong to different connected components, so return `-1`.


## Example Walkthrough

**Input**

```text
nums = [5,3,1,9,10]
maxDiff = 2

queries = [[0,2]]
```

### Step 1: Sort

Sorted values:

```text
[1,3,5,9,10]
```

Their original indices:

```text
[2,1,0,3,4]
```


### Step 2: Compute one-step reach

```text
nxt_reach = [1,2,2,4,4]
```

Meaning:

* Position `0` reaches `1`
* Position `1` reaches `2`
* Position `2` reaches `2`
* Position `3` reaches `4`
* Position `4` reaches `4`


### Step 3: Query

```text
query = [0,2]
```

Original nodes become sorted positions:

```text
2 → 0
```

We must travel from position `0` to `2`.

Binary lifting performs:

```text
0 → 1 → 2
```

Minimum distance:

```text
2
```


## Code

```python
from typing import List

class Solution:
    def pathExistenceQueries(self, n: int, nums: List[int], maxDiff: int, queries: List[List[int]]) -> List[int]:

        order = sorted(range(n), key=lambda i: nums[i])
        pos = [0]*n
        for i, idx in enumerate(order):
            pos[idx] = i

        vals = [nums[i] for i in order]

        nxt_reach = [0]*n
        right=0
        for left in range(n):
            if right<left:
                right = left
            while right+1 < n and vals[right+1] - vals[left] <= maxDiff:
                right += 1
            nxt_reach[left] = right

        log = max(1, n.bit_length())
        jump = [nxt_reach]
        for k in range(1, log):
            prev = jump[k-1]
            row = [0]*n
            for i in range(n):
                row[i] = prev[prev[i]]
            jump.append(row)

        res=[]
        for u, v in queries:
            i, j = pos[u], pos[v]
            if i>j:
                i, j = j, i

            if i==j:
                res.append(0)
                continue

            if nxt_reach[i] == i:
                res.append(-1)
                continue

            cur, hops = i, 0
            for k in range(log-1, -1, -1):
                nxt = jump[k][cur]
                if nxt < j:
                    cur = nxt
                    hops += 1 << k

            if nxt_reach[cur] >= j:
                res.append(hops+1)
            else:
                res.append(-1)

        return res
```


## Complexity

### Time Complexity

Sorting:

$$
O(n \log n)
$$

Building the one-step reach array:

$$
O(n)
$$

Building the binary lifting table:

$$
O(n \log n)
$$

Each query:

$$
O(\log n)
$$

Overall:

$$
O((n + q)\log n)
$$


### Space Complexity

The binary lifting table stores:

$$
O(n \log n)
$$

extra space.


## Edge Cases

### 1. Querying the same node

```text
nums = [3,6,1]
query = [1,1]
```

Answer:

```text
0
```


### 2. Completely disconnected graph

```text
nums = [1,10,20]
maxDiff = 1
```

Every node is isolated.

Any query between different nodes returns:

```text
-1
```


### 3. Fully connected graph

```text
nums = [1,2,3,4]
maxDiff = 10
```

Every pair of nodes has a direct edge.

Most queries return:

```text
1
```


### 4. Multiple jumps required

```text
nums = [1,3,5,7,9]
maxDiff = 2
```

To travel from the first value to the last:

```text
1 → 3 → 5 → 7 → 9
```

Distance:

```text
4
```


## Lessons

* Sorting can expose hidden structure in graphs defined by value constraints.
* A two-pointer sliding window efficiently computes the farthest node reachable in one step.
* Binary lifting is useful beyond trees whenever repeated jumps follow a deterministic transition.
* Preprocessing jump information transforms repeated graph traversal queries from linear time into logarithmic time.
* Combining sorting, sliding windows, and binary lifting is a common pattern for solving large offline query problems efficiently.
