# 1260. Shift 2D Grid 🟢

**Topics:** Array, Matrix

## Intuition

Imagine taking every element from the grid and placing them in a single line.

For example,

```text
1 2 3
4 5 6
7 8 9
```

becomes

```text
1 → 2 → 3 → 4 → 5 → 6 → 7 → 8 → 9
```

Now, shifting the grid once is exactly the same as rotating this line to the right by one position.

Instead of performing one shift at a time, we can directly calculate where every element should go after `k` shifts. Once the shifted line is ready, we simply split it back into rows to rebuild the grid.

This avoids repeatedly moving elements and makes the solution much more efficient.



## Approach

We solve the problem in three steps.

### Step 1: Normalize the number of shifts

If the grid contains `rows × cols` elements, then shifting it that many times returns it to its original state.

So we only need:

```python
k %= rows * cols
```


### Step 2: Build the shifted 1D array

Create a flattened array of size `rows × cols`.

For every cell `(r, c)`:

1. Convert its position into a 1D index.

```python
index = r * cols + c
```

2. Compute its shifted position.

```python
new_index = (index + k) % n
```

3. Store the value directly into its final location.

This avoids physically flattening the grid first.


### Step 3: Rebuild the grid

The flattened array now contains the shifted order.

Simply take groups of `cols` elements to reconstruct each row.


## Example Walkthrough

**Input**

```text
grid = [
 [1,2,3],
 [4,5,6],
 [7,8,9]
]

k = 1
```

The grid has:

```text
rows = 3
cols = 3
n = 9
```

After placing each element into its shifted position, the flattened array becomes:

```text
[9,1,2,3,4,5,6,7,8]
```

Now split it into rows of length `3`.

```text
[
 [9,1,2],
 [3,4,5],
 [6,7,8]
]
```

This is the final shifted grid.


## Code

```python
from typing import List
class Solution:
    def shiftGrid(self, grid: List[List[int]], k: int) -> List[List[int]]:


        rows = len(grid)
        cols = len(grid[0])
        n = rows*cols
        k %= n

        flat = [0]*n
        for r in range(rows):
            for c in range(cols):
                flat[(r*cols+c+k) % n] = grid[r][c]

        result=[]
        for i in range(0, n, cols):
            result.append(flat[i:i + cols])

        return result
```



## Complexity

**Time Complexity**

$$
O(m \times n)
$$

Each element is processed exactly once.

**Space Complexity**

$$
O(m \times n)
$$

An additional flattened array is used to store the shifted elements.



## Edge Cases

### 1. No shifts

```text
grid = [[1,2],[3,4]]
k = 0
```

Output:

```text
[[1,2],[3,4]]
```



### 2. Shifts equal to the number of elements

```text
grid = [[1,2,3],[4,5,6]]
k = 6
```

Since `6 % 6 = 0`, the grid remains unchanged.



### 3. Very large value of `k`

```text
grid = [[1,2],[3,4]]
k = 100
```

Since:

```text
100 % 4 = 0
```

The result is identical to the original grid.


## Lessons

* Many matrix problems become simpler when viewed as operations on a one-dimensional array.
* Converting between 2D and 1D indices is a useful technique:

  * `index = row * cols + col`
  * `row = index // cols`
  * `col = index % cols`
* The modulo operator (`%`) is a powerful tool for handling circular movement and wrap-around.
* Instead of simulating each shift one by one, compute the final destination of every element directly for an efficient solution.
