# 3225. Maximum Score From Grid Operations 🔴

**Topics:** Dynamic Programming, Prefix Sum, Optimization






## Intuition

Imagine each column as a vertical pipe where you decide how much of it to fill with black color from the top.

Now focus on a **row instead of the whole grid**. For any row:

* Some columns are black (filled above that row)
* Others are white

A white cell gives you points only if it **touches a black cell horizontally**.

So the only places that matter are **boundaries between black and white columns**.

Think of it like this:

> Every time there is a “height difference” between adjacent columns, that difference creates scoring opportunities.








## Approach

We use **Dynamic Programming with prefix sums and optimization**.

### Step 1: Prefix Sum per Column

We precompute:

* `cpfx[c][r]` → sum of column `c` from row `0` to `r-1`

This helps us quickly compute vertical segment sums.

---

### Step 2: DP State

We define:

```text
dp[x][y]
```

* `x` = height of previous column
* `y` = height of current column

This represents the **maximum score so far** with these boundary heights.

---

### Step 3: Transition

For each new column:

* We compute a new DP table `ndp`

To optimize transitions:

* Use **prefix max (`pre_max`)** → handles cases where height increases
* Use **suffix max (`suf_max`)** → handles cases where height decreases

This avoids recomputing all transitions and keeps things efficient.

---

### Step 4: Final Column

Handle the last column separately:

* Add remaining contributions using prefix sums






## Example Walkthrough

Consider a small grid:

```text
[[1,2],
 [3,4]]
```

* Try heights for column 0 and 1
* If column 0 height = 0 and column 1 height = 1:

  * Row 1 becomes boundary → contributes `grid[1][0] = 3`

DP systematically explores all such transitions efficiently.







## Code

```python
class Solution:
    def maximumScore(self, grid: List[List[int]]) -> int:

        n =len(grid)


        if n== 1:
            return 0

        cpfx= [[0] *(n+ 1) for _ in range(n)]


        for c in range(n):
            for r in range(n):
                cpfx[c][r+1] = cpfx[c][r] + grid[r][c]


        dp = [[max(0, cpfx[0][y] - cpfx[0][x]) for y in range(n+1)] for x in range(n+1)]


        INF = float('inf')

        for col in range(1, n - 1):

            P = cpfx[col]
            ndp = [[0] * (n + 1) for _ in range(n + 1)]



            for x in range(n + 1):
                pre_max = [-INF] * (n+1)
                running = -INF


                for z in range(n + 1):
                    running = max(running, dp[z][x])
                    pre_max[z] = running

                suf_max = [-INF] * (n+2)
                running = -INF


                for z in range(n, -1, -1):
                    running = max(running, dp[z][x] + max(0, P[z] - P[x]))
                    suf_max[z] = running


                for y in range(n+1):

                    ndp[x][y] = max(
                        pre_max[y] + max(0, P[y] - P[x]),
                        suf_max[y + 1]
                    )

            dp = ndp


        last = cpfx[n - 1]

        ans = 0
        for x in range(n +1):
            
            for y in range(n+ 1):
                s = dp[x][y] + max(0, last[x] - last[y])


                if s > ans:
                    ans = s

        return ans
```







## Complexity

Time:

$$
O(n^3)
$$

Space:

$$
O(n^2)
$$






## Edge Cases

1. `n = 1`
   No adjacent cells → answer is 0

2. All values are 0
   No contribution possible

3. Strictly increasing or decreasing columns
   Forces optimal boundary placements






## Lessons

* Convert grid problems into **column transitions**
* Always question **when a value should be counted**
* Prefix sums reduce repeated work
* Prefix max and suffix max are powerful DP optimizations
* Avoid double counting by carefully defining transitions
