# 2095. Delete the Middle Node of a Linked List 🟡


**Topics:** Linked List, Two Pointers

## Intuition

Imagine two runners on a track.

* One runner moves **one step at a time**.
* The other runner moves **two steps at a time**.

Since the fast runner moves twice as quickly, by the time they reach the end of the track, the slow runner will be standing right in the middle.

This idea works perfectly for linked lists.

There is one extra challenge though. In a singly linked list, we cannot move backwards. So if we want to delete the middle node, we need to keep track of the node just before it.

That is why we maintain three pointers:

* `slow` to find the middle node.
* `fast` to reach the end quickly.
* `prev` to remember the node before `slow`.

Once `slow` reaches the middle, `prev.next` can skip over it and point directly to the next node.






## Approach

We use the **slow and fast pointer** technique.

1. Handle the edge case where the list has only one node.

   * The only node is also the middle, so return `None`.

2. Initialize:

```text
prev = None
slow = head
fast = head
```

3. Traverse the list:

   * Move `slow` by one node.
   * Move `fast` by two nodes.
   * Before moving `slow`, store it in `prev`.

4. When the loop ends:

   * `slow` points to the middle node.
   * `prev` points to the node just before the middle.

5. Delete the middle node:

```python
prev.next = slow.next
```

6. Return `head`.

This completes the task in a single traversal.



## Example Walkthrough

**Input**

```text
head = [1,3,4,7,1,2,6]
```

Initial state:

```text
prev = None
slow = 1
fast = 1
```

### First iteration

```text
prev = 1
slow = 3
fast = 4
```

### Second iteration

```text
prev = 3
slow = 4
fast = 1
```

### Third iteration

```text
prev = 4
slow = 7
fast = 6
```

The loop stops because `fast.next` is `None`.

Current list:

```text
1 -> 3 -> 4 -> 7 -> 1 -> 2 -> 6
          ↑    ↑
        prev  slow
```

Delete the middle:

```python
prev.next = slow.next
```

Updated list:

```text
1 -> 3 -> 4 -> 1 -> 2 -> 6
```

**Output**

```text
[1,3,4,1,2,6]
```



## Code

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def deleteMiddle(self, head: Optional[ListNode]) -> Optional[ListNode]:

        if head is None or head.next is None:
            return None

        prev=None
        slow=head
        fast=head


        while fast and fast.next:
            prev = slow
            slow = slow.next
            fast = fast.next.next


        if prev:
            prev.next = slow.next
            
        return head
```








## Complexity

**Time Complexity**

$$
O(n)
$$

The list is traversed only once.

**Space Complexity**

$$
O(1)
$$

Only three pointers are used regardless of the size of the list.






## Edge Cases

1. **Single node**

```text
head = [1]
```

The only node is the middle.

Output:

```text
[]
```



2. **Two nodes**

```text
head = [2,1]
```

Middle index:

```text
floor(2/2) = 1
```

Delete node `1`.

Output:

```text
[2]
```



3. **Even number of nodes**

```text
head = [1,2,3,4]
```

Middle index:

```text
floor(4/2) = 2
```

Delete node `3`.

Output:

```text
[1,2,4]
```






## Lessons

* The slow and fast pointer technique is a fundamental linked list pattern.
* When deleting a node in a singly linked list, keep track of the previous node because backward traversal is not possible.
* Moving pointers at different speeds is a clever way to locate the middle in one pass.
* This pattern is also useful for:

  * Finding the middle of a linked list.
  * Detecting cycles.
  * Finding the starting node of a cycle.
  * Splitting a linked list into two halves for merge sort.
