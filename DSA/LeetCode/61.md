# 61. Rotate List 🟡

**Topics:** Linked List, Two Pointers






## Intuition

Think of the linked list like a circular conveyor belt. Rotating right by `k` means taking the last `k` elements and moving them to the front.

Instead of actually shifting elements one by one, it is easier to imagine connecting the tail back to the head, forming a circle. Then you just "break" the circle at the right place to get your rotated list.








## Approach

We will use this idea:

1. First, compute the length of the list.
2. Connect the last node to the head, forming a circular list.
3. Compute effective rotations:

   * `k = k % length` because rotating more than length repeats the pattern.
4. Find the new tail:

   * It will be at position `length - k - 1`.
5. The node after the new tail becomes the new head.
6. Break the circle to form the final list.









## Example Walkthrough

Input: `head = [1,2,3,4,5], k = 2`

1. Length = 5
2. Effective rotation = `2 % 5 = 2`
3. New tail position = `5 - 2 - 1 = 2` (value = 3)
4. New head = node after 3 → 4
5. Break the link after 3

Result: `[4,5,1,2,3]`











## Code

```python
# Definition for singly-linked list.
# class ListNode:
#     def __init__(self, val=0, next=None):
#         self.val = val
#         self.next = next
class Solution:
    def rotateRight(self, head: Optional[ListNode], k: int) -> Optional[ListNode]:


        if not head or not head.next or k==0:
            return head


        tail=head
        length= 1


        while tail.next:
            tail = tail.next
            length +=1



        k = k % length

        if k==0:
            return head


        tail.next = head

        steps = length-k-1
        cur = head

        
        while steps > 0:
            cur = cur.next
            steps -=1

        new_head = cur.next
        cur.next = None

        return new_head
```









## Complexity

* Time: $O(n)$
* Space: $O(1)$








## Edge Cases

* Empty list → return `None`
* Single node → no change
* `k = 0` → no rotation
* `k` very large → handled using modulo








## Lessons

* Converting a linear structure into a circular one can simplify problems.
* Always reduce unnecessary work using modulo for repeated patterns.
* Linked list problems often become easier when you focus on pointer movement instead of values.
