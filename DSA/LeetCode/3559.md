# 3559. Number of Ways to Assign Edge Weights II 🔴

**Topics:** Tree, Binary Lifting, Lowest Common Ancestor (LCA), Math, DFS



## Intuition

Imagine every edge on the path is a light switch that can be set to one of two states:

* Weight `1` (odd)
* Weight `2` (even)

We want the total path cost to be odd.

Instead of focusing on the actual weights, we only care about whether each weight contributes an odd or even value to the sum.

For a path containing `d` edges:

* Each edge has 2 possible assignments.
* Total assignments = `2^d`.
* For every assignment producing an even sum, there is a corresponding assignment producing an odd sum.
* Therefore, exactly half of all assignments produce an odd path cost.

So when the path contains at least one edge:

$$
\text{Answer} = 2^{d-1}
$$

The problem now becomes finding the number of edges on the path between two nodes efficiently.

Since there can be up to `10^5` queries, repeatedly traversing paths would be too slow. This naturally leads us to using Lowest Common Ancestor (LCA) with Binary Lifting.

## Approach

We preprocess the tree using Binary Lifting.

### Step 1: Build the tree

Convert the edge list into an adjacency list.

### Step 2: Compute depths and ancestors

Starting from the root (`1`), perform an iterative DFS.

For every node, store:

* Its depth from the root.
* Its direct parent.
* Its `2^k`-th ancestors for all values of `k`.

The table:

```text
parent_at[k][node]
```

stores the ancestor reached after moving up `2^k` levels.

### Step 3: Find LCA efficiently

To find the Lowest Common Ancestor of two nodes:

1. Lift the deeper node until both nodes are at the same depth.
2. Lift both nodes together from the highest power of two downward.
3. The first common ancestor reached is the LCA.

This takes:

$$
O(\log n)
$$

time per query.

### Step 4: Compute path length

Once the LCA is known:

$$
\text{distance}
$$

$$
depth[u]
+
depth[v]
$$

$$
2 \cdot depth[lca]
$$

This gives the number of edges on the path.

### Step 5: Count valid assignments

If the path contains no edges (`u == v`), the answer is `0`.

Otherwise:

$$
\text{answer}
$$

$$
2^{distance-1}
\pmod{10^9+7}
$$

To answer queries quickly, precompute powers of two.

## Example Walkthrough

### Input

```text
edges = [[1,2],[1,3],[3,4],[3,5]]
queries = [[2,5]]
```

Tree:

```text
      1
     / \
    2   3
       / \
      4   5
```

### Query: [2,5]

Path:

```text
2 -> 1 -> 3 -> 5
```

The LCA of `2` and `5` is `1`.

Depth values:

```text
depth[2] = 1
depth[5] = 2
depth[1] = 0
```

Distance:

$$
1 + 2 - 2(0) = 3
$$

The path contains 3 edges.

Total assignments:

$$
2^3 = 8
$$

Exactly half produce an odd path cost:

$$
2^{3-1} = 4
$$

Answer:

```text
4
```

## Code

```python
class Solution:
    def assignEdgeWeights(self, edges: List[List[int]], queries: List[List[int]]) -> List[int]:

        MOD = 1000000007

        n = len(edges)+1
        LOG = 17

        graph = [[] for _ in range(n+1)]
        for a, b in edges:
            graph[a].append(b)
            graph[b].append(a)

        depth = [0] * (n+1)
        parent_at = [[0] * (n+1) for _ in range(LOG)]

        stack = [(1, 0)]
        while stack:
            node, par = stack.pop()
            parent_at[0][node] = par
            for k in range(1, LOG):
                parent_at[k][node] = parent_at[k - 1][parent_at[k - 1][node]]
            for nxt in graph[node]:
                if nxt != par:
                    depth[nxt] = depth[node] + 1
                    stack.append((nxt, node))

        def find_lca(a, b):
            if depth[a] < depth[b]:
                a, b = b, a
            diff = depth[a] - depth[b]
            for k in range(LOG):
                if diff & (1 << k):
                    a = parent_at[k][a]
            if a == b:
                return a
            for k in range(LOG - 1, -1, -1):
                if parent_at[k][a] != parent_at[k][b]:
                    a = parent_at[k][a]
                    b = parent_at[k][b]
            return parent_at[0][a]

        pow2 = [1] * (2 * n+1)
        for i in range(1, len(pow2)):
            pow2[i] = pow2[i - 1] * 2 % MOD

        res = []
        for u, v in queries:
            if u == v:
                res.append(0)
                continue
            anc = find_lca(u, v)
            dist = depth[u] + depth[v] - 2 * depth[anc]
            res.append(pow2[dist - 1])

        return res
```

## Complexity

### Preprocessing

Building depths and ancestor table:

$$
O(n \log n)
$$

### Per Query

Finding the LCA:

$$
O(\log n)
$$

### Total

For `q` queries:

$$
O(n \log n + q \log n)
$$

### Space Complexity

$$
O(n \log n)
$$

for the binary lifting table.

## Edge Cases

### 1. Same node query

```text
u = v
```

The path contains no edges.

```text
Answer = 0
```

### 2. Single edge tree

```text
1 -- 2
```

Path length is `1`.

```text
Answer = 1
```

### 3. Long chain tree

```text
1 - 2 - 3 - ... - n
```

The path length can be very large, so powers of two should be precomputed modulo `10^9 + 7`.

### 4. Root as LCA

```text
u = 4
v = 5
```

The root or a high ancestor may be the LCA. The distance formula still works correctly.

## Lessons

* Sometimes counting problems become much simpler when viewed through parity instead of actual values.
* A path with `d` edges has `2^d` weight assignments, and exactly half produce an odd sum.
* Binary Lifting is one of the most important techniques for answering ancestor and path queries on trees.
* The distance between two nodes can be computed quickly using their depths and LCA.
* Precomputing powers of two avoids repeated modular exponentiation during queries.
