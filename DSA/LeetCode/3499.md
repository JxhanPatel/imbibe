# 3499. Maximize Active Section with Trade I 🟡

**Topics:** String, Greedy

## Intuition

Imagine a road with stretches of working streetlights (`1`) separated by stretches of broken lights (`0`).

A trade works like this:

1. You temporarily turn off one isolated working section.
2. This connects the two neighboring broken sections into one larger broken section.
3. You then repair that entire merged broken section, turning it into working lights.

The only part that really matters is how many inactive sections can be merged together. Every isolated block of `1`s sits between two zero blocks, so removing it joins those two zero runs into one larger run.

Therefore, for every pair of consecutive zero runs, we calculate how many inactive sections would become active after merging them. The largest such merge gives the best trade.

## Approach

1. Count the number of active sections (`1`s) already present.

2. Traverse the string and record the length of every contiguous block of `0`s.

3. If there are fewer than two zero runs, no valid trade is possible, so return the original count of active sections.

4. Otherwise, examine every pair of adjacent zero runs.

5. The gain from removing the `1` block between them is simply:

   ```
   left_zero_run + right_zero_run
   ```

6. Choose the maximum gain and add it to the original number of active sections.

## Example Walkthrough

**Input**

```text
s = "01010"
```

### Step 1: Count active sections

```text
ones = 2
```

### Step 2: Find zero runs

```text
Zero runs = [1, 1, 1]
```

### Step 3: Compute possible gains

Between first and second run:

```text
1 + 1 = 2
```

Between second and third run:

```text
1 + 1 = 2
```

Best gain:

```text
2
```

### Step 4: Final answer

```text
2 + 2 = 4
```

**Output**

```text
4
```

## Code

```python
class Solution:
    def maxActiveSectionsAfterTrade(self, s: str) -> int:

        ones = s.count('1')

        zero_runs=[]
        i=0
        n=len(s)

        while i < n:
            if s[i] == '0':
                start=i

                while i < n and s[i] == '0':
                    i += 1

                zero_runs.append(i-start)

            else:
                i+= 1

        if len(zero_runs)<2:
            return ones

        best_gain=0

        for j in range(len(zero_runs)-1):
            gain = zero_runs[j] + zero_runs[j+1]

            if gain>best_gain:
                best_gain = gain

        return ones+best_gain
```

## Complexity

**Time Complexity**

Traversing the string takes linear time.

$$
O(n)
$$

**Space Complexity**

The list of zero runs can contain at most (n) elements.

$$
O(n)
$$

## Edge Cases

1. **No zero sections**

```text
s = "11111"
```

No trade is possible.

Answer:

```text
5
```


2. **Only one zero run**

```text
s = "00011"
```

There is no pair of zero runs to merge.

Answer:

```text
2
```


3. **Alternating characters**

```text
s = "01010"
```

Each zero run has length `1`, so merging neighboring runs gives the maximum benefit.

Answer:

```text
4
```


4. **All inactive**

```text
s = "0000"
```

There is no isolated block of `1`s to remove.

Answer:

```text
0
```

## Lessons

* Sometimes the operation can be simplified by focusing on what changes instead of simulating every step.
* Identifying contiguous runs of characters is a common and useful string technique.
* Consecutive run information often lets you solve problems with a single linear scan.
* When only neighboring segments interact, storing run lengths is usually enough to compute the optimal answer efficiently.
