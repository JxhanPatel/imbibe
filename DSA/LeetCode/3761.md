# 3761. Minimum Absolute Distance Between Mirror Pairs 🟡

**Topics:** Hash Map, String Manipulation







## Intuition

Think of each number as a word, and its reversed version as its “mirror twin.”

Now imagine walking through a list of people. Each person is looking for their mirror twin. The moment you find someone whose twin has already appeared before, you measure how far apart they are.

To get the **minimum distance**, you want to catch these mirror matches as early and as close together as possible.







## Approach

We will use a **hash map (dictionary)** to store the last seen index of each number.

Steps:

1. Traverse the array from left to right.
2. For each number:

   * Compute its reversed value.
   * Check if this reversed value has been seen before.
   * If yes, compute the distance between indices.
   * Update the minimum distance.
3. Store/update the current number’s index in the map.
4. Return the minimum distance found, or `-1` if none exists.

Why this works:

* Hash map gives **O(1)** lookup.
* We always compare with the closest previous occurrence.






## Example Walkthrough

**Input:** `nums = [12, 21, 45, 33, 54]`

Step-by-step:

* i = 0, num = 12 → reverse = 21
  Not seen → store {12: 0}

* i = 1, num = 21 → reverse = 12
  Found at index 0 → distance = 1
  min_dist = 1
  store {21: 1}

* i = 2, num = 45 → reverse = 54
  Not seen → store {45: 2}

* i = 3, num = 33 → reverse = 33
  Not seen → store {33: 3}

* i = 4, num = 54 → reverse = 45
  Found at index 2 → distance = 2
  min_dist still = 1

Final answer = 1







## Code (Python)

```python
class Solution:
    def minMirrorPairDistance(self, nums: List[int]) -> int:

        def flip_digits(n):
            s = str(n)
            return int(s[::-1])


        rev_index = {}
        closest = float('inf')


        for idx, val in enumerate(nums):

            if val in rev_index:
                gap = idx - rev_index[val]

                if gap < closest:
                    closest = gap

            flipped = flip_digits(val)
            
            rev_index[flipped] = idx

        if closest == float('inf'):
            return -1

        return closest
        
```




## Complexity

* Time: $O(n \cdot d)$ where ( d ) is number of digits
* Space: $O(n)$




## Edge Cases

* Single element array → no pairs → return `-1`
* Numbers like `10 → reverse = 1` (leading zero removed)
* Same number like `33` → reverse is itself
* Large input size up to (10^5)





## Lessons

* Hash maps are powerful for **tracking previous occurrences efficiently**
* Reversing numbers via string is simple and safe
* Always think in terms of **"have I seen the counterpart before?"** for pair problems
* Greedy idea: closest pair comes from comparing with the **most recent occurrence**
