# 3635. Earliest Finish Time for Land and Water Rides II 🟡

**Topics:** Binary Search, Sorting, Prefix Minimum, Suffix Minimum, Arrays

## Intuition

Imagine you finish your first ride at time `x` and now want to choose the best ride from the other category.

For every possible second ride, one of two things happens:

1. The ride is already open when you arrive.

   * You start immediately.
   * Finish time = `x + duration`.

2. The ride is not open yet.

   * You must wait for it to open.
   * Finish time = `startTime + duration`.

So for a fixed arrival time `x`, every ride contributes:

$$
\max(startTime, x) + duration
$$

Checking every pair of rides would take $O(nm)$, which is far too slow for $5 \times 10^4$ rides.

The key observation is that rides naturally split into two groups:

* rides with `startTime <= x`
* rides with `startTime > x`

This allows us to preprocess useful minimum values and answer each query in (O(\log n)).



## Approach

Suppose we want to know the best water ride after finishing a land ride at time `x`.

We need:

$$
\min(\max(waterStart_j, x) + waterDuration_j)
$$

### Case 1: Water ride already open

If:

$$
waterStart_j \le x
$$

then:

$$
\max(waterStart_j, x) = x
$$


and the finish time becomes:

$$
x + waterDuration_j
$$

Among all such rides, we only need the minimum duration.


### Case 2: Water ride not open yet

If:

$$
waterStart_j > x
$$

then:

$$
\max(waterStart_j, x) = waterStart_j
$$

and the finish time becomes:

$$
waterStart_j + waterDuration_j
$$

Among all such rides, we only need the minimum value of:

$$
waterStart_j + waterDuration_j
$$



### Preprocessing

Sort rides by start time.

Build:

* `prefixMinDuration[i]`

  * minimum duration among rides up to index `i`

* `suffixMinFinish[i]`

  * minimum `(start + duration)` from index `i` onward

For a query time `x`:

1. Binary search to find the last ride with `start <= x`.
2. Use prefix minimum for Case 1.
3. Use suffix minimum for Case 2.
4. Take the smaller result.

We do this twice:

* Land → Water
* Water → Land

and return the overall minimum finish time.



## Example Walkthrough

### Input

```text
landStartTime = [2,8]
landDuration  = [4,1]

waterStartTime = [6]
waterDuration  = [3]
```

### Land → Water

Land ride 0:

```text
Start = 2
Finish = 6
```

Query water rides with `x = 6`.

Water ride opens at 6, so:

```text
Finish = 6 + 3 = 9
```

Current answer = 9.



Land ride 1:

```text
Start = 8
Finish = 9
```

Water ride already opened:

```text
Finish = 9 + 3 = 12
```

Answer remains 9.



### Water → Land

Water ride:

```text
Start = 6
Finish = 9
```

Land ride 1 is already open:

```text
Finish = 9 + 1 = 10
```

Best result here is 10.



Overall:

```text
min(9, 10) = 9
```

### Output

```text
9
```



## Code

```python
from bisect import bisect_right
from typing import List

class Solution:
    def earliestFinishTime(self, landStartTime: List[int], landDuration: List[int], waterStartTime: List[int], waterDuration: List[int]) -> int:

        def build(starts, durations):

            rides = sorted(zip(starts, durations))

            sorted_starts = [s for s, _ in rides]

            n=len(rides)

            pre_min_dur = [0]*n

            pre_min_dur[0] = rides[0][1]

            for i in range(1, n):
                pre_min_dur[i] = min(pre_min_dur[i-1], rides[i][1])

            suf_min_finish = [0]*n

            suf_min_finish[-1] = rides[-1][0]+rides[-1][1]

            for i in range(n - 2, -1, -1):
                suf_min_finish[i] = min(suf_min_finish[i+1], rides[i][0]+rides[i][1])

            return sorted_starts, pre_min_dur, suf_min_finish

        def best_finish_after(t, starts, pre_min_dur, suf_min_finish):
            idx = bisect_right(starts, t)-1
            best = float("inf")

            if idx >= 0:
                best = t+pre_min_dur[idx]

            if idx+1 < len(starts):
                best = min(best, suf_min_finish[idx+1])

            return best


        land = build(landStartTime, landDuration)
        water = build(waterStartTime, waterDuration)

        ans = float("inf")


        for s, d in zip(landStartTime, landDuration):
            ans = min(ans, best_finish_after(s+d, *water))

        for s, d in zip(waterStartTime, waterDuration):
            ans = min(ans, best_finish_after(s+d, *land))

        return ans
```



## Complexity

Sorting:

$$
O(n \log n + m \log m)
$$

Each query uses binary search:

$$
O(\log n)
\quad \text{or} \quad
O(\log m)
$$

Total:

$$
O(n \log m + m \log n)
$$

Space:

$$
O(n + m)
$$



## Edge Cases

1. **Only one land ride and one water ride**

```text
landStartTime = [5]
landDuration = [3]
waterStartTime = [1]
waterDuration = [10]
```

Must correctly consider both possible orders.



2. **Second ride opens much later**

```text
landStartTime = [1]
landDuration = [2]
waterStartTime = [100]
waterDuration = [1]
```

Waiting time dominates the answer.



3. **All rides already open before arrival**

```text
landStartTime = [1, 2]
waterStartTime = [1, 2]
```

The answer should come from the smallest duration ride in the second category.



## Lessons

* Expressions involving `max(start, x)` often split naturally into two cases.
* When queries ask for minimum values over elements before or after a position, prefix and suffix minima are powerful tools.
* Binary search plus preprocessing can reduce an (O(nm)) comparison problem to roughly (O((n+m)\log n)).
* Always look for ways to transform pairwise comparisons into independent queries over a preprocessed structure.
