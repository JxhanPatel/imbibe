# 1331. Rank Transform of an Array 🟢

**Topics:** Array, Sorting, Hash Table

## Intuition

Imagine you're organizing participants in a race based on their finish times.

* The fastest runner gets rank `1`.
* The second fastest gets rank `2`.
* If multiple runners finish with the same time, they all receive the same rank.
* The next distinct finish time gets the next available rank.

This problem works the same way. Instead of race times, we have array values. We first determine the order of all unique values, assign each one the smallest possible rank, and then replace every element with its assigned rank.

## Approach

We use a combination of **sorting** and a **hash map**.

1. Extract all unique values from the array.
2. Sort these unique values in ascending order.
3. Assign ranks starting from `1`.
4. Store each value and its rank in a dictionary.
5. Traverse the original array and replace every value with its corresponding rank.

Using a dictionary allows each lookup to be done in constant time.

## Example Walkthrough

**Input**

```text
arr = [40,10,20,30]
```

### Step 1: Get unique values

```text
{10,20,30,40}
```

### Step 2: Sort them

```text
[10,20,30,40]
```

### Step 3: Assign ranks

| Value | Rank |
| ----- | ---- |
| 10    | 1    |
| 20    | 2    |
| 30    | 3    |
| 40    | 4    |

Dictionary becomes:

```text
{
    10: 1,
    20: 2,
    30: 3,
    40: 4
}
```

### Step 4: Replace each element

| Original | Rank |
| -------- | ---- |
| 40       | 4    |
| 10       | 1    |
| 20       | 2    |
| 30       | 3    |

**Output**

```text
[4,1,2,3]
```

## Code

```python
from typing import List
class Solution:
    def arrayRankTransform(self, arr: List[int]) -> List[int]:

        rank={}

        for i, value in enumerate(sorted(set(arr)), 1):
            rank[value]=i

        return [rank[value] for value in arr]
```

## Complexity

**Time Complexity**

* Creating the set: $O(n)$
* Sorting unique values: $O(k \log k)$, where $k$ is the number of distinct values
* Building the answer: $O(n)$

Overall:

$$
O(n + k \log k)
$$

In the worst case, when all elements are distinct ($k = n$):

$$
O(n \log n)
$$

**Space Complexity**

$$
O(k)
$$

where $k$ is the number of distinct values.

## Edge Cases

1. **Empty array**

```text
arr = []
```

Output:

```text
[]
```

2. **All elements are the same**

```text
arr = [100,100,100]
```

Output:

```text
[1,1,1]
```

3. **Negative numbers**

```text
arr = [-5,-10,-5,0]
```

Sorted unique values:

```text
[-10,-5,0]
```

Output:

```text
[2,1,2,3]
```

## Lessons

* Sorting unique values is an efficient way to assign ordered ranks.
* A hash map is useful for storing the relationship between a value and its assigned rank.
* Removing duplicates before sorting avoids assigning multiple ranks to the same value.
* This pattern of **sort once, map values, then transform the original array** appears frequently in coordinate compression and ranking problems.
