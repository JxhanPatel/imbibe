# 3567. Minimum Absolute Difference in Sliding Submatrix 🟡
**Topics:** Array, Sorting, Sliding Window, Matrix



## Intuition

Think of it like a sliding magnifying glass moving over a newspaper. You're holding a square lens of size `k x k`, and every time you place it somewhere on the page, you want to find which two different numbers inside that lens are the closest to each other in value.

The core insight is: **if you sort the numbers inside the window, then the closest pair is always neighbors in that sorted list.** You don't need to check every pair. If the values are `[-2, 1, 3, 8]`, the smallest gap can only come from two adjacent values after sorting, never from values farther apart.



## Approach

Since the grid is at most 30x30 and `k <= 30`, the total cells in one submatrix is at most 900. We have at most 29x29 = 841 positions. So the brute force approach is completely fine here.

For each possible top-left corner `(i, j)`, we:
1. Collect all values inside the `k x k` submatrix
2. Remove duplicates (since we need *distinct* values only)
3. Sort those unique values
4. Walk through adjacent pairs in the sorted list and track the minimum difference

If only one unique value exists, the answer is 0 (as the problem states).



## Example Walkthrough

**Input:** `grid = [[1, -2, 3], [2, 3, 5]]`, `k = 2`

We have 2 possible windows:

**Window at (0, 0):**
```
1  -2
2   3
```
Values: `{-2, 1, 2, 3}` → sorted: `[-2, 1, 2, 3]`  
Differences: `|1-(-2)|=3`, `|2-1|=1`, `|3-2|=1` → min = **1**

**Window at (0, 1):**
```
-2  3
 3  5
```
Values: `{-2, 3, 5}` (duplicate 3 removed) → sorted: `[-2, 3, 5]`  
Differences: `|3-(-2)|=5`, `|5-3|=2` → min = **2**

Output: `[[1, 2]]` ✓

---

### Code

```python
class Solution:
    def minAbsDiff(self, grid: List[List[int]], k: int) -> List[List[int]]:
        m, n = len(grid), len(grid[0])
        
        rows_out = m - k + 1
        cols_out = n - k + 1
        
        ans = [[0] * cols_out for _ in range(rows_out)]
        

        # slide the top-left corner of our kxk window
        for i in range(rows_out):
            for j in range(cols_out):
                

                # gather all values inside this kxk submatrix
                values = []
                for r in range(i, i + k):
                    for c in range(j, j + k):
                        values.append(grid[r][c])
                

                # get only unique values, then sort em
                unique_vals = sorted(set(values))
                

                # if everything is the same value, answer is 0
                if len(unique_vals) == 1:
                    ans[i][j] = 0
                    continue
                
                min_diff = float('inf')
                for idx in range(len(unique_vals) - 1):
                    diff = unique_vals[idx + 1] - unique_vals[idx]
                    min_diff = min(min_diff, diff)
                
                ans[i][j] = min_diff
        

        return ans
```



## Complexity

**Time Complexity:** $O((m - k + 1)(n - k + 1) \cdot k^2 \log k)$

For each of the $(m-k+1)(n-k+1)$ windows, we collect $k^2$ elements and sort them in $O(k^2 \log k)$. Given the constraints (m, n ≤ 30, k ≤ 30), this is completely fine.

**Space Complexity:** $O(k^2)$

We only store the values from one window at a time.


## Edge Cases

**1. k = 1 (single cell windows)**  
Every window has exactly one value. Since you need two *distinct* values to form a difference, the answer for every cell is 0. Our code handles this cleanly because `set(values)` will have length 1.

**2. All elements in the entire grid are the same**  
Every window will have one unique value, and every answer is 0. No special handling needed.

**3. Grid contains negative numbers**  
Since we sort the values, `-2` naturally comes before `1`, and `|1 - (-2)| = 3` is computed correctly by just doing `unique_vals[idx+1] - unique_vals[idx]` (no need for `abs()` since the list is sorted ascending).

**4. k = min(m, n) (largest possible window)**  
Only one window exists. We just do the full matrix. The output is a single cell `[[answer]]`.


## Lessons


**Sorting reveals adjacency.** Whenever you need the "minimum gap between any two elements," sort first and only compare neighbors. This drops you from $O(n^2)$ comparisons to $O(n \log n)$ sorting + $O(n)$ scan.

**Deduplication matters.** The problem says *distinct values*. Using `set()` before sorting handles this effortlessly.

**Brute force is not always wrong.** When constraints are small (≤30), reaching for the clean, readable solution is often the right call. Don't over-optimize when you don't need to.

**Sliding window on a matrix.** This is the 2D version of sliding window problems you see in 1D arrays. You're nested-looping the top-left corner and collecting a rectangular patch. It is a fundamental pattern worth internalizing.
