# 48. Rotate Image 🟡

**Topics:** Array, Matrix





## Intuition

Think of the matrix like a square photo printed on paper. If you rotate the paper 90 degrees clockwise, what happens?

* The top row becomes the right column
* The left column becomes the top row (but reversed)

Instead of physically rotating, we simulate this by rearranging elements in-place.

A useful trick:

1. Transpose the matrix (flip across diagonal)
2. Reverse each row

This combination gives a 90 degree clockwise rotation.








## Approach

We will use a 2-step approach:

### Step 1: Transpose the matrix

Swap `matrix[i][j]` with `matrix[j][i]`

This converts rows into columns.

### Step 2: Reverse each row

After transpose, reverse every row to complete the rotation.

This avoids using extra space and keeps everything in-place.










## Example Walkthrough

Input:

```
1 2 3
4 5 6
7 8 9
```

### Step 1: Transpose

```
1 4 7
2 5 8
3 6 9
```

### Step 2: Reverse each row

```
7 4 1
8 5 2
9 6 3
```

Final rotated matrix achieved.








## Code

```python
class Solution:
    def rotate(self, matrix: List[List[int]]) -> None:
        """
        Do not return anything, modify matrix in-place instead.
        """
        n = len(matrix)

        for i in range(n):

            for j in range(i+1, n):
                matrix[i][j], matrix[j][i] = matrix[j][i], matrix[i][j]

        for row in matrix:
            left, right = 0, len(row) -1


            while left < right:
                row[left], row[right] = row[right], row[left]

                left+=1
                right-=1
```








## Complexity

Time Complexity:
$$O(n^2)$$
We visit each cell once.

Space Complexity:
$$O(1)$$
In-place rotation, no extra matrix used.








## Edge Cases

* Single element matrix

  ```
  [[1]]
  ```

  No change needed

* Empty matrix (if allowed in variation)
  Should safely return without error

* Negative numbers or mixed values
  Works the same since we only swap positions









## Lessons

* Matrix rotation problems often reduce to patterns like transpose + reverse
* In-place modification is a key constraint in many interview problems
* Avoid brute force copying when constraints demand optimization
* Understanding how indices move is more important than memorizing steps
