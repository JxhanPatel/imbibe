# 1758: Minimum Changes To Make Alternating Binary String 🟢  
**Topics:** String, Greedy  



## Intuition

Imagine you're decorating a hallway with alternating black and white floor tiles. You can start with either a black tile or a white tile - both are valid patterns! 

Now, suppose some tiles are already laid down, but they're all jumbled up. Your job is to flip the minimum number of tiles so that the entire hallway follows a perfect alternating pattern. 

Since you have two choices for the starting color (black or white), you need to check both possibilities and pick whichever requires fewer flips. It's like trying on two different outfits and choosing the one that needs fewer adjustments!

---

## Approach

The key insight is that an alternating binary string can only be one of two patterns:

1. **Pattern A:** Starts with `'0'` → `"010101..."`
2. **Pattern B:** Starts with `'1'` → `"101010..."`

For any given string `s`, we can count how many changes are needed to convert it to Pattern A, and how many for Pattern B. The answer is simply the minimum of these two counts.

Here's the clever part: we don't need to actually build these pattern strings. We can compute the cost on the fly! For position `i`:
- In Pattern A, we expect `'0'` at even indices and `'1'` at odd indices
- In Pattern B is just the opposite

So we iterate through the string once, comparing each character to what it *should* be in both patterns, and keep running tallies.

---

## Example Walkthrough

Let's trace through `s = "0100"`:

| Index | Char | Pattern A expects | Match A? | Pattern B expects | Match B? |
|-------|------|-------------------|----------|-------------------|----------|
| 0 | '0' | '0' (even) | ✓ Yes | '1' (even) | ✗ No |
| 1 | '1' | '1' (odd) | ✓ Yes | '0' (odd) | ✗ No |
| 2 | '0' | '0' (even) | ✓ Yes | '1' (even) | ✗ No |
| 3 | '0' | '1' (odd) | ✗ No | '0' (odd) | ✓ Yes |

- Changes needed for Pattern A: 1 (only index 3 is wrong)
- Changes needed for Pattern B: 3 (indices 0, 1, 2 are wrong)

Minimum is **1**. We change the last character to '1', getting `"0101"`.

---

## Code

```python
class Solution:
    def minOperations(self, s: str) -> int:

        count = 0 
        
        for i, char in enumerate(s):

            expected = '0' if i % 2 == 0 else '1'
            if char != expected:
                count += 1
        
        return min(count, len(s) - count)

```


---

## Complexity

- **Time Complexity:** $O(n)$ where $n$ is the length of the string. We make a single pass through the string.
- **Space Complexity:** $O(1)$. We only use a couple of integer variables regardless of input size.

---

## Edge Cases

| Case | Input | Why It Matters |
|------|-------|--------------|
| **Single character** | `"0"` or `"1"` | Already alternating by definition. Returns 0. |
| **Already perfect** | `"010101"` | Both patterns work, but one costs 0. Returns 0. |
| **All same characters** | `"0000"` or `"1111"` | Must flip exactly half (or half rounded up). For `"1111"`, we can make `"1010"` or `"0101"` with 2 flips. |
| **Maximum length** | String of length $10^4$ | Our $O(n)$ solution handles this easily within time limits. |

---

## Lessons

1. **When you see "alternating", think "two patterns"**: Any alternating sequence has exactly two valid starting points. Always consider both!

2. **Don't build, calculate**: We never constructed the target strings. We just computed what each position *should* be using the index parity. This saves memory and keeps code clean.

3. **The complement trick**: Notice that `count0 + count1 = n` because every position matches exactly one of the two patterns. This lets us compute one count and derive the other, cutting work in half!

4. **Greedy works here**: There's no need for dynamic programming or backtracking. The alternating constraint is so rigid that the solution space collapses to just two possibilities.
