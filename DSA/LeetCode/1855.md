# 1855. Maximum Distance Between a Pair of Values 🟡

**Topics:** Two Pointers, Binary Search, Arrays







## Intuition

Imagine two people walking along two separate downhill paths.
Both paths only go down or stay flat (non-increasing arrays).

* Person A is on `nums1`
* Person B is on `nums2`

You want to find the maximum distance between them such that:

* Person B is always at the same position or ahead (`j >= i`)
* Person B’s height is always greater than or equal to Person A’s height (`nums2[j] >= nums1[i]`)

Since both arrays are sorted in decreasing order, once something becomes invalid, it will remain invalid further ahead. This is a perfect setup for a two-pointer strategy.








## Approach

We use **two pointers**:

* `i` for `nums1`
* `j` for `nums2`

### Key idea:

* Start both at `0`
* If `nums1[i] <= nums2[j]`, it's a valid pair
  → update max distance
  → move `j` forward to try increasing distance
* Else
  → move `i` forward to find a smaller value in `nums1`

This works because both arrays are non-increasing.








## Example Walkthrough

**Input:**
`nums1 = [55,30,5,4,2]`
`nums2 = [100,20,10,10,5]`

### Steps:

* `i=0, j=0` → 55 <= 100 → valid → distance = 0
* `j=1` → 55 > 20 → invalid → move `i`
* `i=1` → 30 > 20 → invalid → move `i`
* `i=2` → 5 <= 20 → valid → distance = 1
* `j=2` → 5 <= 10 → valid → distance = 0
* `j=3` → 5 <= 10 → valid → distance = 1
* `j=4` → 5 <= 5 → valid → distance = 2

Max distance = **2**









## Code

```python
class Solution:
    def maxDistance(self, nums1: List[int], nums2: List[int]) -> int:

        i, j = 0, 0
        best = 0

        while i < len(nums1) and j < len(nums2):

            if nums1[i] <= nums2[j]:
                best = max(best, j - i)
                j += 1
                
            else:
                i += 1

        return best
```







## Complexity

* Time: $O(n + m)$
* Space: $O(1)$






## Edge Cases

1. No valid pairs
   Example: `nums1 = [10]`, `nums2 = [5]` → return `0`

2. All elements valid
   Example: `nums1 = [1,1,1]`, `nums2 = [10,10,10]`

3. Single element arrays
   Example: `nums1 = [5]`, `nums2 = [5]`







## Lessons

* When arrays are **sorted in a specific order**, always think about two pointers
* The condition `i <= j` naturally fits with pointer movement
* Instead of checking all pairs, we **greedily expand `j`** and only move `i` when necessary
* This pattern appears often in problems involving **monotonic arrays**
