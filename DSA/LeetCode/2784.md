# 2784. Check if Array is Good 🟢

**Topics:** Array, Sorting, Hash Table


## Intuition

Think of a "good" array like a classroom attendance sheet.

For a valid class of size `n`:

* Every student from `1` to `n-1` should appear exactly once
* The last student `n` accidentally signed twice

So the expected pattern becomes:

$$
[1, 2, 3, ..., n-1, n, n]
$$

The important observation:

* The largest number in the array must be `n`
* The array length must therefore be `n + 1`

Once we know `n`, we simply check whether the array contains:

* every number from `1` to `n-1` once
* number `n` twice

A simple way to verify this is by sorting.







## Approach

1. Sort the array.
2. Let the maximum element be `n`.
3. The sorted array should exactly match:

$$
[1, 2, 3, ..., n-1, n, n]
$$

4. If it matches, return `True`.
5. Otherwise, return `False`.










## Example Walkthrough

### Input

```python
nums = [1, 3, 3, 2]
```

### Step 1: Sort

```python
[1, 2, 3, 3]
```

### Step 2: Find `n`

Largest element:

```python
n = 3
```

### Step 3: Build expected array

```python
[1, 2, 3, 3]
```

### Step 4: Compare

Both arrays are equal.

So the answer is:

```python
True
```














## Code

```python
class Solution:
    def isGood(self, nums: List[int]) -> bool:

        n= max(nums)
        
        if len(nums) != n+1:
            return False
        
        freq = Counter(nums)

        if freq[n] != 2:
            return False
        
        for i in range(1, n):
            
            if freq[i] != 1:
                return False
        
        return True
```












## Complexity

Sorting dominates the runtime.

* Time Complexity:

$$
O(n \log n)
$$

* Space Complexity:

$$
O(n)
$$

for the expected array.








## Edge Cases

1. Minimum size array

```python
[1, 1]
```

Valid because it matches `base[1]`.

2. Missing number

```python
[1, 2, 2]
```

Invalid because `3` is missing.

3. Extra duplicate

```python
[1, 2, 3, 3, 3]
```

Invalid because `3` appears three times.

4. Wrong length

```python
[2, 1, 3]
```

Invalid because `base[3]` should have 4 elements.







## Lessons

* The maximum element often helps determine the structure of the problem.
* Sorting is useful when checking whether two arrays are permutations of each other.
* Building the expected pattern directly can simplify validation logic.
* Always check both:

  * correct values
  * correct frequency of values
