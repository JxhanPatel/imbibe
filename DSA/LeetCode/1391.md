# 1391. Check if There is a Valid Path in a Grid 🟡

**Topics:** Graph, BFS, DFS, Grid







## Intuition

Think of each cell as a pipe piece in a plumbing system.
Each type of street is like a pipe with openings in certain directions.

A valid path means:

* Water (or you) can flow from the start to the end
* Pipes must connect properly on both sides

For example:

* If you move right from a cell, the next cell must have a pipe that connects from the left

So the key idea:
You can only move between two cells if both agree on the connection.










## Approach

We treat the grid like a graph and use BFS or DFS.

Steps:

1. Define directions for each street type
2. Start from (0, 0)
3. Try moving in all valid directions from the current cell
4. For a move to be valid:

   * Current cell must allow movement in that direction
   * Next cell must allow entry from the opposite direction
5. Use a visited set to avoid cycles
6. If we reach (m-1, n-1), return True

---

### Street Connections

We map each street type to allowed directions:

```
1 -> left, right
2 -> up, down
3 -> left, down
4 -> right, down
5 -> left, up
6 -> right, up
```










## Example Walkthrough

Input:

```
grid = [[2,4,3],
        [6,5,2]]
```

Start at (0,0) → type 2 (up, down)

* Move down → (1,0), type 6 (right, up)

  * Valid because (1,0) allows entry from up

Now from (1,0):

* Move right → (1,1), type 5 (left, up)

  * Valid

From (1,1):

* Move up → (0,1), type 4 (right, down)

  * Valid

Continue and eventually reach (1,2)

So answer = True












## Code

```python
from collections import deque


class Solution:
    def hasValidPath(self, grid: List[List[int]]) -> bool:

        rows, cols = len(grid), len(grid[0])

        pipe_dirs = {
            1: [(0, -1), (0, 1)],
            2: [(-1, 0), (1, 0)],
            3: [(0, -1), (1, 0)],
            4: [(0, 1), (1, 0)],
            5: [(0, -1), (-1, 0)],
            6: [(0, 1), (-1, 0)],
        }


        def connects_back(nx, ny, from_x, from_y):
            return any(nx + ddx==from_x and ny+ddy == from_y

                       for ddx, ddy in pipe_dirs[grid[nx][ny]])


        visited = {(0, 0)}
        q = deque([(0, 0)])


        while q:
            x, y = q.popleft()

            if x ==rows- 1 and y== cols - 1:
                return True


            for dx, dy in pipe_dirs[grid[x][y]]:

                nx, ny = x + dx, y + dy

                if not (0 <= nx < rows and 0 <= ny < cols):
                    continue

                if (nx, ny) in visited:
                    continue

                if not connects_back(nx, ny, x, y):
                    continue
                    

                visited.add((nx, ny))
                q.append((nx, ny))

        return False
        
```








## Complexity

Time Complexity:
$$O(m \times n)$$
Each cell is visited at most once

Space Complexity:
$$O(m \times n)$$
For visited set and queue








## Edge Cases

1. Single cell grid

   * Always True since start == end

2. No valid connections from (0,0)

   * Return False immediately

3. Dead-end paths

   * Path exists initially but breaks midway








## Lessons

* Grid problems often reduce to graph traversal
* Always check bidirectional validity in connection problems
* Mapping rules clearly before coding avoids confusion
* BFS is safer for shortest path or reachability problems
