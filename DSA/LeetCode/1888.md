# 1888. Minimum Number of Flips to Make the Binary String Alternating 🟡
**Topics:** String, Sliding Window, Dynamic Programming


## Intuition

Imagine you have a circular train track with train cars labeled either '0' or '1'. You can rotate the entire train (Type-1 operation) for free, but repainting a car (Type-2 operation) costs you money. Your goal is to arrange the train so that no two adjacent cars have the same label, spending as little as possible on repainting.

The key insight is that an alternating train can only look like two possible patterns: starting with '0' ("010101...") or starting with '1' ("101010..."). Since rotation is free, we want to find the rotation that requires the fewest repaints to match one of these two patterns.

For even-length trains, rotation doesn't change the parity of positions, but for odd-length trains, every rotation flips which positions are even and odd, changing the game completely!



## Approach

We'll use a **sliding window with dynamic updates** to efficiently check all rotations without actually rotating the string:

1. **Count initial mismatches** against the "01" pattern (starting with '0')
2. **Leverage complementarity**: If `cnt` flips are needed for "01...", then `n - cnt` flips are needed for "10..." (since every position matches exactly one pattern)
3. **Simulate rotations**: For each character moved from front to back, update the mismatch count in $O(1)$ time by:
   - Removing its contribution from its old position
   - Adding its contribution at its new position
4. **Track the minimum** across all rotations and both patterns

This avoids the $O(n^2)$ brute force approach of actually rotating and checking each time.



## Example Walkthrough

**Input:** `s = "111000"` (n = 6)

**Step 1:** Count mismatches with "010101..." pattern
- Position 0: '1' vs '0' → mismatch ✗
- Position 1: '1' vs '1' → match ✓
- Position 2: '1' vs '0' → mismatch ✗
- Position 3: '0' vs '1' → mismatch ✗
- Position 4: '0' vs '0' → match ✓
- Position 5: '0' vs '1' → mismatch ✗

`cnt = 4` mismatches for "01..." pattern, so `6 - 4 = 2` for "10..." pattern.
Current best: `min(4, 2) = 2`

**Step 2:** Simulate rotations (conceptually moving first char to end)
- After moving '1' to end: Update `cnt` by removing position 0 contribution (was mismatch) and adding position 6 contribution (position 6 expects '0', '1' is mismatch)
- New `cnt` remains 4, but checking both patterns gives us `min(4, 2) = 2`

Continuing through all rotations, we find the minimum stays **2**, achieved by rotating twice to get "100011", then flipping positions 2 and 5 to get "101010".



## Code

```python
class Solution:
    def minFlips(self, s: str) -> int:

        n = len(s)
        target = "01"



        cnt = sum(c != target[i & 1] for i, c in enumerate(s))
        ans = min(cnt, n - cnt)


        
        for i in range(n):
            cnt -= s[i] != target[i & 1]
            
            cnt += s[i] != target[(i + n) & 1]
            
            ans = min(ans, cnt, n - cnt)
        
        return ans
```



## Complexity

- **Time Complexity:** $O(n)$ - We scan the string once initially, then perform $O(1)$ work for each of the $n$ rotations.
- **Space Complexity:** $O(1)$ - Only using a few integer counters, no additional data structures proportional to input size.



## Edge Cases

1. **Already Alternating:** `s = "010"` or `s = "1010"` → Returns 0, no flips needed
2. **Single Character:** `s = "0"` or `s = "1"` → Returns 0, single char is trivially alternating
3. **All Same Characters:** `s = "0000"` or `s = "1111"` → Returns $n/2$ (ceil), need to flip half to create alternation
4. **Odd Length with Optimal Rotation:** `s = "1110"` → Returns 1; rotating to "1101" or "1011" doesn't help, but flipping position 1 of original gives "1010"



## Lessons

**The "Two Pattern" Trick:** For any alternating binary string problem, remember there are only two valid end states. Instead of exploring exponentially many possibilities, just compare against these two templates.

**Complementary Counting:** If computing cost for pattern A, pattern B cost is simply `n - cost_A` since they are bitwise complements. This halves your computation!

**Virtual Rotation:** When allowed free rotations, don't actually rotate the string (expensive!). Use modular arithmetic to simulate what each character's "new position" would be and update counts incrementally.

**Sliding Window on Circular Arrays:** The technique of doubling the string conceptually (using `(i + n) & 1` for new positions) lets you handle circular operations with linear time while maintaining $O(1)$ space.
