# 2833. Furthest Point From Origin 🟢

**Topics:** Greedy, String







## Intuition

Think of yourself standing at position 0 on a number line.

* Each `'L'` pushes you left
* Each `'R'` pushes you right
* Each `'_'` is like a "wildcard step" where you can choose either direction

Now imagine you want to go as far as possible from the origin.
So the question becomes:

> How do we use these flexible `'_'` moves to maximize distance?

Key idea:

* To go as far as possible in one direction, we should use all `'_'` moves in that same direction.
* Then just combine them with existing `'L'` or `'R'`.








## Approach

1. Count:

   * number of `'L'` → `left`
   * number of `'R'` → `right`
   * number of `'_'` → `blank`

2. Two possibilities:

   * Go **fully left** → distance = `left + blank`
   * Go **fully right** → distance = `right + blank`

3. Return the maximum of the two.

Why this works:

* Mixing directions reduces distance
* Committing fully to one direction maximizes displacement








## Example Walkthrough

**Input:**

```
moves = "L_RL__R"
```

Count:

```
L = 2
R = 2
_ = 3
```

Try both directions:

* Go left:

```
distance = L + _ = 2 + 3 = 5
```

* Go right:

```
distance = R + _ = 2 + 3 = 5
```

But actual max displacement depends on net direction:

* Left movement: -5 → distance = 5
* Right movement: +5 → distance = 5

**Answer:** `5`

But note: sometimes direction signs cancel out, so we just care about absolute max.









## Code

```python
class Solution:
    def furthestDistanceFromOrigin(self, moves: str) -> int:
        L = moves.count('L')
        R = moves.count('R')
        freee = moves.count('_')

        ans = R - L
        
        return abs(ans) + freee

```







## Complexity

* Time: $O(n)$
* Space: $O(1)$







## Edge Cases

1. All blanks

```
moves = "_____"
→ answer = 5
```

2. Already balanced

```
moves = "LRLR"
→ answer = 2
```

3. Only one direction

```
moves = "LLLL"
→ answer = 4
```






## Lessons

* When choices are flexible, push all of them in one direction
* Greedy works when local optimal choices build global optimum
* Always consider extremes in such problems rather than simulating every path
