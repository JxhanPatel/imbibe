# 1727. Largest Submatrix With Rearrangements 🟡
**Topics:** Array, Greedy, Sorting, Matrix


## Intuition

Imagine you're a city planner looking at a skyline of buildings. Each column represents a street, and the consecutive 1s going upward represent how tall the buildings are on that street. 

Now, here's the key insight: **you can rearrange the streets (columns) however you want!** This is like being able to reorder the buildings in the skyline to create the widest possible rectangular plaza at any given height.

For any height `h`, if you have streets with buildings at least `h` tall, you can group them together side by side. The best strategy is: for each possible height, count how many columns can support that height, then multiply height by that count.



## Approach

**The "Histogram + Sorting" Strategy:**

1. **Build Histogram Heights:** For each cell, calculate how many consecutive 1s extend upward (including itself). This transforms our binary matrix into a histogram where each column has a "height" of consecutive 1s.

2. **Sort Each Row:** For each row, sort the heights in descending order. This lets us greedily pick the widest possible base for each possible height.

3. **Calculate Maximum Area:** After sorting a row, for each position `i`, we can form a rectangle of height `heights[i]` with width `(i + 1)`. The area is `heights[i] * (i + 1)`. Take the maximum across all positions and all rows.

**Why sorting works:** By placing taller columns first, when we're at index `i`, we guarantee that all columns from `0` to `i` are at least as tall as `heights[i]`. So we can definitely form a rectangle of that height with width `i+1`.



## Example Walkthrough

**Input:** `matrix = [[0,0,1],[1,1,1],[1,0,1]]`

**Step 1: Build histogram heights row by row**
- Row 0: `[0, 0, 1]` (just copy the row)
- Row 1: `[1, 1, 2]` (add to previous heights, reset on 0)
- Row 2: `[2, 0, 3]` (add to previous heights, reset on 0)

**Step 2: Sort each row and find max area**

Row 0: `[0, 0, 1]` → sorted: `[1, 0, 0]`
- i=0: `1 * 1 = 1`
- i=1: `0 * 2 = 0`
- i=2: `0 * 3 = 0`

Row 1: `[1, 1, 2]` → sorted: `[2, 1, 1]`
- i=0: `2 * 1 = 2`
- i=1: `1 * 2 = 2`
- i=2: `1 * 3 = 3`

Row 2: `[2, 0, 3]` → sorted: `[3, 2, 0]`
- i=0: `3 * 1 = 3`
- i=1: `2 * 2 = 4`  **← Maximum found!**
- i=2: `0 * 3 = 0`

**Result:** 4


## Code

```python
class Solution:
    def largestSubmatrix(self, matrix: List[List[int]]) -> int:

        if not matrix or not matrix[0]:
            return 0
        
        m = len(matrix)
        n = len(matrix[0])
        max_area = 0
        
        # heights[j] = consecutive 1s ending at current row for column j
        heights = [0] * n
        
        for i in range(m):

            for j in range(n):
                if matrix[i][j] == 1:
                    heights[j] += 1  # Extend the streak of 1s
                else:
                    heights[j] = 0   # Reset, broken streak

            
            # Sort heights descending so tallest columns come first
            sorted_heights = sorted(heights, reverse=True)
            
            for k in range(n):
                area = sorted_heights[k] * (k + 1)
                max_area = max(max_area, area)        
        return max_area
        
```



## Complexity

- **Time:** $O(m \cdot n \log n)$ where $m$ is rows and $n$ is columns. We process each of $m$ rows, and for each row we sort $n$ elements.
- **Space:** $O(n)$ for the heights array and sorted copy.



## Edge Cases

1. **Single row matrix:** `[[1,0,1,1]]` → Sorting gives `[1,1,1,0]`, max area is `1 * 3 = 3`. Works because we can group all 1s together.

2. **Single column matrix:** `[[1],[1],[0],[1]]` → Each row has one height value. Max area is just the maximum height (2), since width is always 1.

3. **All zeros:** `[[0,0],[0,0]]` → All heights stay 0, result is 0. Handled naturally.

4. **All ones:** `[[1,1],[1,1]]` → Row 1: `[1,1]`, area max 2. Row 2: `[2,2]` sorted, area `2*2=4`. Correct!



## Lessons

1. **Histogram Transformation:** Converting 2D problems into 1D histograms is a classic pattern (also seen in "Maximal Rectangle" problems).

2. **Greedy Sorting:** When you can rearrange elements freely, sorting often reveals the optimal structure. Here, descending sort lets us know exactly how wide a rectangle we can form at each height.

3. **The "For Each as Minimum" Pattern:** Similar to "Largest Rectangle in Histogram," we consider each bar as the limiting height and calculate what rectangle it could form.

4. **Column Rearrangement = Sorting:** The ability to rearrange columns is powerful because it means we can always group compatible columns together. Recognizing when sorting unlocks the solution is a valuable skill.
