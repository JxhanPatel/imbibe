# 153. Find Minimum in Rotated Sorted Array 🟡

**Topics:** Array, Binary Search

## Intuition

Imagine a perfectly sorted line of books arranged by height.

Normally, the smallest book is at the beginning. But someone takes a chunk from the front and moves it to the back.

Example:

```text
Original:
1 2 3 4 5 6 7

Rotated:
4 5 6 7 1 2 3
```

The array is still sorted in two parts:

* Left part is sorted
* Right part is sorted
* The minimum element is exactly where the "break" happens

Instead of checking every element one by one, we can use Binary Search to quickly locate the break point.

The key observation:

* If `nums[mid] > nums[right]`, the minimum must be on the right side
* Otherwise, the minimum is on the left side including `mid`

This lets us shrink the search space in logarithmic time.








## Approach

We use Binary Search.

### Steps

1. Set two pointers:

   * `left = 0`
   * `right = len(nums) - 1`

2. While `left < right`:

   * Find middle index
   * Compare `nums[mid]` with `nums[right]`

3. Cases:

   * If `nums[mid] > nums[right]`

     * Minimum lies to the right of `mid`
     * Move `left = mid + 1`
   * Else

     * Minimum could be `mid` itself
     * Move `right = mid`

4. When loop ends:

   * `left` points to the minimum element









## Example Walkthrough

### Input

```python
nums = [4,5,6,7,0,1,2]
```

### Step 1

```text
left = 0
right = 6
mid = 3
nums[mid] = 7
nums[right] = 2
```

Since `7 > 2`, minimum is on the right side.

```text
left = mid + 1 = 4
```



### Step 2

```text
left = 4
right = 6
mid = 5
nums[mid] = 1
nums[right] = 2
```

Since `1 < 2`, minimum is at `mid` or left side.

```text
right = mid = 5
```



### Step 3

```text
left = 4
right = 5
mid = 4
nums[mid] = 0
nums[right] = 1
```

Again:

```text
right = mid = 4
```

Now:

```text
left == right == 4
```

Answer:

```python
nums[4] = 0
```










## Code

```python
class Solution:
    def findMin(self, nums: List[int]) -> int:


        lo=0
        hi = len(nums)-1

        while lo < hi:
            mid = (lo+hi) //2

            if nums[mid] > nums[hi]:
                lo = mid+1

            else:
                hi = mid

        return nums[lo]
```









## Complexity

* Time Complexity: $O(\log n)$
* Space Complexity: $O(1)$









## Edge Cases

### 1. Array not rotated

```python
[11,13,15,17]
```

Minimum is simply the first element.



### 2. Array rotated once

```python
[5,1,2,3,4]
```

Minimum appears immediately after the first element.



### 3. Single element array

```python
[7]
```

Return `7`.









## Lessons

* Rotated sorted arrays often hint toward Binary Search.
* Comparing with `nums[right]` helps identify which side is unsorted.
* In Binary Search problems, focus on how to eliminate half the search space safely.
