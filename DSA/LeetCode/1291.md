# 1291. Sequential Digits 🟡

**Topics:** Enumeration, String, Math

## Intuition

Imagine the digits from `1` to `9` are written on a ribbon:

```text
123456789
```

Every sequential number is simply a **continuous piece** cut from this ribbon.

For example:

* `12`
* `123`
* `3456`
* `6789`
* `12345`

Instead of checking every number between `low` and `high`, we can generate every possible sequential number directly from this ribbon. Since there are only nine digits, there are only a handful of candidates to consider.

## Approach

We use the string `"123456789"` as the source of all sequential numbers.

1. Store `"123456789"` in a string.
2. Try every possible number length from `2` to `9`.
3. For each length, slide a window across the string.
4. Convert the current substring into an integer.
5. If the number is smaller than `low`, continue to the next substring.
6. If the number is larger than `high`, break because all remaining numbers of the same length will also be larger.
7. Otherwise, add the number to the answer.
8. Finally, sort the results and return them.

The early `break` works because, for a fixed length, the generated numbers are already in increasing order.

## Example Walkthrough

**Input**

```text
low = 1000
high = 13000
```

Start with:

```text
digits = "123456789"
```

### Length = 4

Generate candidates:

```text
1234
2345
3456
4567
5678
6789
```

All of these lie within the range, so they are added.

### Length = 5

Generate candidates:

```text
12345
23456
34567
...
```

* `12345` is within the range, so add it.
* `23456` is greater than `13000`, so stop checking further 5-digit numbers because they will only get larger.

Final answer:

```text
[1234, 2345, 3456, 4567, 5678, 6789, 12345]
```

## Code

```python
from typing import List

class Solution:
    def sequentialDigits(self, low: int, high: int) -> List[int]:
        
        digits = "123456789"
        results=[]

        for length in range(2,10):
            for i in range(10-length):
                candidate = int(digits[i:i+length])
                
                if candidate<low:
                    continue

                if candidate>high:
                    break

                results.append(candidate)

        results.sort()
        return results
```

## Complexity

**Time Complexity**

There are at most

$$
8 + 7 + 6 + \cdots + 1 = 36
$$

possible sequential numbers.

Therefore,

$$
O(1)
$$

since the total number of generated candidates is constant.

**Space Complexity**

$$
O(1)
$$

because the maximum number of sequential numbers is fixed.

## Edge Cases

1. **Range contains exactly one sequential number**

```text
low = 123
high = 123
```

Output:

```text
[123]
```

2. **No valid sequential numbers**

```text
low = 500
high = 520
```

Output:

```text
[]
```

3. **Largest possible sequential number**

```text
low = 123456789
high = 123456789
```

Output:

```text
[123456789]
```

4. **Smallest valid range**

```text
low = 10
high = 12
```

Output:

```text
[12]
```

## Lessons

* Sometimes generating all valid candidates is much simpler than checking every value in a range.
* A fixed pattern can often be represented as a string and explored using a sliding window.
* When generated values are naturally ordered, an early `break` can avoid unnecessary work.
* Always look for problems where the number of possible answers is small, even if the input range is large.
