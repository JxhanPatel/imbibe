# 3751. Total Waviness of Numbers in Range I 🟡


**Topics:** Math, Simulation, Digit Processing

## Intuition

Think of the digits of a number as hills and valleys on a landscape.

A digit is:

* A **peak** if it is higher than both neighboring digits.
* A **valley** if it is lower than both neighboring digits.

Since the first and last digits only have one neighbor, they can never be peaks or valleys.

The constraints are very small (`num2 <= 100000`), so instead of looking for a complicated mathematical pattern, we can simply examine every number in the range and count its peaks and valleys directly.

## Approach

For every number from `num1` to `num2`:

1. Convert the number into a string of digits.
2. If it has fewer than 3 digits, its waviness is `0`.
3. Check every middle digit:

   * Peak: `digit[i] > digit[i - 1]` and `digit[i] > digit[i + 1]`
   * Valley: `digit[i] < digit[i - 1]` and `digit[i] < digit[i + 1]`
4. Count all peaks and valleys for that number.
5. Add the result to the overall answer.

Since there are at most `100000` numbers and each number has at most `6` digits, this approach is easily fast enough.

## Example Walkthrough

**Input**

```text
num1 = 120
num2 = 130
```

Check each number:

```text
120 -> 2 is a peak      -> 1
121 -> 2 is a peak      -> 1
122 -> none             -> 0
123 -> none             -> 0
124 -> none             -> 0
125 -> none             -> 0
126 -> none             -> 0
127 -> none             -> 0
128 -> none             -> 0
129 -> none             -> 0
130 -> 3 is a peak      -> 1
```

Total:

```text
1 + 1 + 1 = 3
```

**Output**

```text
3
```

## Code

```python
class Solution:
    def totalWaviness(self, num1: int, num2: int) -> int:

        def get_waviness(val: int) -> int:
            digits = str(val)

            if len(digits) <3:
                return 0
            
            peaks_and_valleys=0
            i= 1


            while i < len(digits)-1:

                prev_d, curr_d, next_d = digits[i-1], digits[i], digits[i+1]
                
                is_peak = curr_d>prev_d and curr_d>next_d
                is_valley = curr_d<prev_d and curr_d<next_d
                
                if is_peak or is_valley:
                    peaks_and_valleys+= 1
                i+= 1
                
            return peaks_and_valleys

        total= 0


        for num in range(num1, num2+1):
            total += get_waviness(num)
        return total
```

## Complexity

Let:

* $n = \text{num2} - \text{num1} + 1$
* $d$ = number of digits in a number (at most 6)

Time complexity:

$$
O(n \cdot d)
$$

Since $d \le 6$, this is effectively:

$$
O(n)
$$

Space complexity:

$$
O(d)
$$

which is:

$$
O(1)
$$

because the digit count is bounded.

## Edge Cases

1. **Single-digit numbers**

```text
num1 = 1
num2 = 9
```

Every number has waviness `0`.

2. **Two-digit numbers**

```text
num1 = 10
num2 = 99
```

No peaks or valleys are possible.

3. **Repeated digits**

```text
111
55555
```

Equal neighboring digits cannot form peaks or valleys.

4. **Multiple peaks and valleys**

```text
4848
```

* 8 is a peak
* 4 is a valley

Waviness = `2`.

## Lessons

* Always check the constraints before designing a complex solution.
* When the digit count is very small, direct digit simulation is often the simplest and most reliable approach.
* Peaks and valleys are local properties, so each digit only needs to be compared with its immediate neighbors.
* A straightforward brute-force solution can be optimal when the input size is small enough.
