# 3612. Process String with Special Operations I 🟡

**Topics:** String, Simulation


## Intuition

Imagine you are typing in a text editor that has three unusual buttons:

* `*` works like the **backspace** key and removes the last character.
* `#` works like **copy + paste**, duplicating everything currently on the screen.
* `%` flips the entire text backwards.

We process the string from left to right and maintain the current text in a list called `out`.

Whenever we encounter a special character, we perform its corresponding operation immediately. Otherwise, we simply append the character.

Since the length of `s` is at most `20`, a straightforward simulation is more than enough.



## Approach

We use a list `out` to store the current result.

For every character `ch` in `s`:

1. If `ch == '*'`

   * Remove the last character if `out` is not empty.

2. If `ch == '#'`

   * Duplicate the current contents using:

```python
out = out + out
```

3. If `ch` is an alphabet character

   * Append it to `out`.

4. Otherwise, the character must be `%`

   * Reverse the current result using:

```python
out = out[::-1]
```

Finally, join all characters and return the resulting string.

## Code

```python
class Solution:
    def processStr(self, s: str) -> str:

        out=[]

        for ch in s:

            if ch == '*':
                if out:
                    out.pop()
                continue


            if ch == '#':
                out = out+out
                continue

            if ch.isalpha():
                out.append(ch)
                continue

            out = out[::-1]
        return ''.join(out)
```



## Complexity

Let `m` be the length of the final result.

**Time Complexity**

$$
O(m)
$$

* Appending takes constant time.
* `pop()` takes constant time.
* Duplication and reversal take time proportional to the current size of `out`.
* Since `|s| \le 20`, the total work is bounded by the size of the generated string.

**Space Complexity**

$$
O(m)
$$

We store the resulting characters in the list `out`.



## Edge Cases

1. **Backspace on an empty result**

```text
s = "*"
```

Nothing is removed.

Output:

```text
""
```



2. **Duplicate an empty result**

```text
s = "#"
```

Duplicating an empty string still gives an empty string.

Output:

```text
""
```



3. **Only reverse operations**

```text
s = "abc%%"
```

Process:

```text
"abc"
-> "cba"
-> "abc"
```

Output:

```text
"abc"
```



4. **Multiple backspaces**

```text
s = "ab***"
```

Process:

```text
"ab"
-> "a"
-> ""
-> ""
```

Output:

```text
""
```



## Lessons

* Simulation is often the simplest and cleanest solution when constraints are small.
* Lists are usually a better choice than strings when frequent modifications are required.
* `pop()` is a natural way to implement backspace operations.
* Reversal and duplication can be expressed elegantly using Python list slicing and concatenation.
* Always map each special character to a clear operation and process the input sequentially.
