# 3742. Maximum Path Score in a Grid 🟡

**Topics:** Dynamic Programming, Grid, Knapsack-style DP






## Intuition

Think of this problem like traveling through a city grid where each cell is a checkpoint.

* Some checkpoints give you rewards (score).
* Some checkpoints charge you a fee (cost).
* You have a limited budget `k`.

Your goal is not just to reach the destination, but to **maximize your reward without overspending**.

This is similar to a mix of:

* grid path problems (move right or down)
* and knapsack problems (limited cost budget)

At every step, you must decide:

> "Is taking this cell worth the cost given my remaining budget?"








## Approach

We use **Dynamic Programming**.

### Key Idea

Let:

```
dp[i][j][c] = maximum score to reach cell (i, j) using cost exactly c
```

We:

1. Initialize DP with `-inf` (invalid states)
2. Start from `(0,0)` with cost `0` and score `0`
3. For each cell, try:

   * coming from top
   * coming from left
4. Update cost and score accordingly
5. At the end, check all `dp[m-1][n-1][c]` where `c <= k`








## Example Walkthrough

**Input:**

```
grid = [[0,1],
        [2,0]]
k = 1
```

### Step-by-step:

Start at `(0,0)`:

* score = 0, cost = 0

Move options:

1. Right → `(0,1)`:

   * value = 1 → score +1, cost +1
   * total: score = 1, cost = 1

2. Down → `(1,0)`:

   * value = 2 → score +2, cost +1
   * total: score = 2, cost = 1

From `(1,0)` → `(1,1)`:

* value = 0 → no change
* final: score = 2, cost = 1

Valid since cost ≤ k

**Answer = 2**









## Code (Python)

```python
from typing import List
class Solution:
    def maxPathScore(self, grid: List[List[int]], k: int) -> int:

        m =len(grid)
        n =len(grid[0])

        dp = [[[-1] * (k+1) for _ in range(n)] for _ in range(m)]

        dp[0][0][0] = 0


        for i in range(m):

            for j in range(n):

                for c in range(k+1):
                    cur = dp[i][j][c]

                    if cur == -1:
                        continue

                    for di, dj in ((0, 1), (1, 0)):
                        ni, nj = i+di, j+dj

                        if ni >= m or nj >= n:
                            continue

                        nval = grid[ni][nj]
                        nc = c + (1 if nval > 0 else 0)

                        if nc > k:
                            continue

                        if dp[ni][nj][nc] < cur+nval:
                            dp[ni][nj][nc] = cur+nval


        best = max(dp[m-1][n-1])
        return best if best != -1 else -1
```








## Complexity

* Time Complexity:
  $O(m \cdot n \cdot k)$

* Space Complexity:
  $O(m \cdot n \cdot k)$








## Edge Cases

1. `k = 0`

   * Only paths with zero cost are allowed

2. Single cell grid

   * Always return 0 since start = end

3. No valid path within cost

   * Return `-1`

4. Grid full of `2`s

   * Cost accumulates quickly, many paths invalid









## Lessons

* This is a classic **grid + knapsack hybrid**
* When constraints include "limit on cost", think:

  * add a third DP dimension
* Always track:

  * state (position)
  * resource usage (cost)
* Use `-1` or `-inf` to mark unreachable states
