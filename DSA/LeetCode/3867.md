# 3867. Sum of GCD of Formed Pairs 🟡

**Topics:** Array, Math, Number Theory, Sorting

## Intuition

Imagine you are organizing people by their strengths.

First, each person receives a new score based on the **greatest common divisor (GCD)** between their own strength and the strongest person seen so far. This creates a new list of values.

Next, you sort these scores and repeatedly pair the weakest remaining score with the strongest remaining score. For every pair, you calculate their GCD and add it to the final answer.

The problem naturally breaks into these two phases:

1. Build the `prefixGcd` array.
2. Sort it and pair elements using two pointers.


## Approach

### Step 1: Build `prefixGcd`

Maintain the maximum value encountered while traversing the array.

For every element:

* Update the running maximum.
* Compute the GCD of the current number and the running maximum.
* Store this value in `prefixGcd`.

Since the running maximum is updated in constant time, this step takes only one pass.


### Step 2: Sort the array

Sort `prefixGcd` in non-decreasing order.

Sorting makes it easy to repeatedly pair:

* the smallest element,
* with the largest element.


### Step 3: Form pairs

Use two pointers:

* `lo` starts at the beginning.
* `hi` starts at the end.

While `lo < hi`:

* Compute the GCD of the current pair.
* Add it to the answer.
* Move both pointers inward.

If the array has an odd length, the middle element is left unpaired, exactly as required.


## Example Walkthrough

**Input**

```text
nums = [3, 6, 2, 8]
```

### Step 1: Build `prefixGcd`

| Index | Number | Running Max | GCD |
| ----: | -----: | ----------: | --: |
|     0 |      3 |           3 |   3 |
|     1 |      6 |           6 |   6 |
|     2 |      2 |           6 |   2 |
|     3 |      8 |           8 |   8 |

```text
prefixGcd = [3, 6, 2, 8]
```


### Step 2: Sort

```text
[2, 3, 6, 8]
```


### Step 3: Pair

| Pair   | GCD |
| ------ | --: |
| (2, 8) |   2 |
| (3, 6) |   3 |

Total:

```text
2 + 3 = 5
```

**Output**

```text
5
```


## Code

```python
from math import gcd
from typing import List

class Solution:
    def gcdSum(self, nums: list[int]) -> int:
        prefixes=[]
        running_max=0

        for n in nums:
            running_max = max(running_max, n)
            prefixes.append(gcd(n, running_max))

        prefixes.sort()

        lo, hi = 0, len(prefixes)-1
        total=0

        while lo < hi:
            total += gcd(prefixes[lo], prefixes[hi])
            lo += 1
            hi -= 1

        return total
```


## Complexity

**Time Complexity**

Building the `prefixGcd` array:

$$
O(n)
$$

Sorting:

$$
O(n \log n)
$$

Pairing elements:

$$
O(n)
$$

Overall:

$$
O(n \log n)
$$


**Space Complexity**

$$
O(n)
$$

An additional array is used to store the `prefixGcd` values.


## Edge Cases

### 1. Single element

```text
nums = [7]
```

No pair can be formed.

Answer:

```text
0
```


### 2. All elements are the same

```text
nums = [5, 5, 5, 5]
```

```text
prefixGcd = [5, 5, 5, 5]
```

Every pair contributes `5`.


### 3. Odd length array

```text
nums = [2, 6, 4]
```

After sorting:

```text
[2, 2, 6]
```

Only one pair is formed:

```text
gcd(2, 6) = 2
```

The remaining middle element is ignored.


## Lessons

* A running maximum allows prefix-based computations to be built in a single traversal.
* The GCD function is efficient enough to be used repeatedly since it runs in logarithmic time.
* Sorting followed by the two-pointer technique is a simple and effective pattern for pairing the smallest and largest elements.
* Breaking a problem into independent phases often leads to a clean and easy-to-understand solution.
