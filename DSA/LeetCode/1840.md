# 1840. Maximum Building Height 🔴

**Topics:** Greedy, Sorting, Array, Math

## Intuition

Imagine building a mountain range.

Some checkpoints have strict height limits:

```text
Building 2 ≤ 1
Building 5 ≤ 3
Building 7 ≤ 4
```

Also, neighboring buildings cannot differ by more than `1`.

This means a restriction does not only affect its own building. It also limits how quickly nearby buildings can rise or fall.

Think of every restriction as pulling down the surrounding heights.

Our goal is:

1. Make all restrictions consistent with each other.
2. Find the highest peak possible between every pair of neighboring restrictions.

The important observation is:

> After adjusting all restrictions, the tallest building must lie somewhere between two consecutive restrictions.



## Approach

We use a **greedy + constraint propagation** approach.

### Step 1: Add mandatory restrictions

Building `1` always has height `0`.

```python
restrictions.append([1, 0])
```

If building `n` is not restricted, we still add:

```python
restrictions.append([n, n - 1])
```

Why `n - 1`?

Starting from:

```text
building 1 = 0
```

and increasing by at most `1` each step:

```text
0 -> 1 -> 2 -> ... -> n-1
```

So building `n` can never exceed `n-1`.



### Step 2: Sort restrictions

Sort by building index.

Example:

```text
[[1,0],[2,5],[5,3],[7,4],[10,3]]
```

Now neighboring restrictions are easy to process.



### Step 3: Left to right pass

Suppose:

```text
(x1, h1)
(x2, h2)
```

Distance:

```text
gap = x2 - x1
```

Since adjacent buildings differ by at most `1`:

```text
h2 ≤ h1 + gap
```

So we tighten:

```python
h2 = min(h2, h1 + gap)
```

This ensures the right restriction is reachable from the left.



### Step 4: Right to left pass

Similarly,

```text
h1 ≤ h2 + gap
```

So:

```python
h1 = min(h1, h2 + gap)
```

This ensures the left restriction is reachable from the right.

After both passes, all restrictions are valid simultaneously.



### Step 5: Find the highest peak

Consider:

```text
(x1, h1)
(x2, h2)
```

Distance:

```text
d = x2 - x1
```

The tallest shape possible is:

```text
       peak
      /\
     /  \
    /    \
 h1        h2
```

It rises by `1` each step and then falls by `1`.

The maximum peak becomes:

```text
(h1 + h2 + d) // 2
```

Compute this for every adjacent pair and take the maximum.

## Code

```python
class Solution:
    def maxBuilding(self, n: int, restrictions: List[List[int]]) -> int:

        restrictions.append([1, 0])
        restrictions.append([n, n-1])
        restrictions.sort()


        m = len(restrictions)


        for i in range(1, m):
            gap = restrictions[i][0] - restrictions[i-1][0]
            restrictions[i][1] = min(restrictions[i][1], restrictions[i-1][1]+gap)


        for i in range(m-2, -1, -1):
            gap = restrictions[i+1][0] - restrictions[i][0]
            restrictions[i][1] = min(restrictions[i][1], restrictions[i+1][1] + gap)

        best=0

        for i in range(1, m):
            x1, h1 = restrictions[i-1]
            x2, h2 = restrictions[i]
            best = max(best, (h1+h2 + x2-x1) // 2)

        return best
```



## Complexity

**Time Complexity**

Sorting dominates the runtime.

$$
O(m \log m)
$$

where `m` is the number of restrictions.



**Space Complexity**

$$
O(1)
$$

Ignoring the space used internally by sorting.



## Edge Cases

### 1. No restrictions

```text
n = 6

restrictions = []
```

Buildings:

```text
0 1 2 3 4 5
```

Answer:

```text
5
```



### 2. Restriction too large

```text
n = 5

restrictions = [[2,100]]
```

Building `2` cannot actually become `100`.

Because:

```text
building 1 = 0

|h2 - h1| ≤ 1
```

So:

```text
h2 ≤ 1
```

The left-to-right pass automatically fixes this.



### 3. Conflicting restrictions

```text
n = 5

restrictions =

[
[2,5],
[3,0]
]
```

Since:

```text
|h2 - h3| ≤ 1
```

and:

```text
h3 = 0
```

we must have:

```text
h2 ≤ 1
```

The right-to-left pass adjusts this automatically.



## Lessons

* When constraints propagate to neighboring positions, consider performing sweeps from both directions.
* Adding artificial boundaries like `[1,0]` and `[n,n-1]` can simplify interval problems significantly.
* For adjacent difference constraints, the optimal shape between two fixed points is often a mountain that rises and falls at the maximum allowed rate.
* Constraint propagation combined with greedy interval optimization is a powerful pattern that appears frequently in hard array problems.
