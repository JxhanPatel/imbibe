# 1545. Find Kth Bit in Nth Binary String 🟡  
**Topics:** String, Recursion, Bit Manipulation


## Intuition

Imagine you're building a **fractal mirror**. You start with a simple shape (just "0"), then each time you create a bigger version by taking what you have, placing a mirror in the middle ("1"), and then reflecting and inverting your original shape on the other side.

The key insight is: **you don't need to build the entire string to find one bit**. Just like you wouldn't build an entire skyscraper to check what's on the 50th floor, you can use the recursive structure to "teleport" directly to where your answer lives.


## Approach

We'll use **recursive descent with position tracking**. Here's the beautiful pattern hidden in the construction:

For any string $S_n$ with length $2^n - 1$:
- The **middle bit** is always "1" (that's the `+ "1"` part)
- The **left half** is exactly $S_{n-1}$
- The **right half** is the reverse-invert of $S_{n-1}$

So for position $k$ in $S_n$:
1. Find the middle: `mid = length // 2 + 1`
2. If `k == mid`: return "1" (always the middle)
3. If `k < mid`: the answer is in the left half, so recurse into $S_{n-1}$ at position $k$
4. If `k > mid`: the answer is in the right half. The right half is reversed, so position $k$ corresponds to position `length - k + 1` in the inverted left half. Inversion means we flip the bit we find.


## Example Walkthrough

Let's trace `n = 3, k = 5`:

- $S_3$ has length $2^3 - 1 = 7$, middle is at position 4
- $k = 5 > 4$, so we're in the **right half**
- The right half mirrors the left, so position 5 maps to position `7 - 5 + 1 = 3` in the inverted left side
- We need the bit at position 3 of $S_2$ (inverted)

Now for $S_2$ (length 3, middle at 2):
- Position 3 > 2, so right half again
- Maps to position `3 - 3 + 1 = 1` in inverted $S_1$

For $S_1$ (length 1):
- Position 1 is the middle, return "0"

But remember we had **two inversions** (one from each recursive step), so: `0 → 1 → 0`

## Code

```python
class Solution:
    def findKthBit(self, n: int, k: int) -> str:
        
        # base case: S1 is just 0
        if n == 1:
            return "0"
        

        # working with half length to avoid big numbers
        half_length = (1 << (n - 1)) - 1  
        middle_position = half_length + 1 
        
        if k == middle_position:
            return "1"
        

        elif k < middle_position:
            return self.findKthBit(n - 1, k)
        

        else:
            mirror_position = 2 * half_length + 1 - k + 1
            
            mirror_position = 2 * middle_position - k
            
            bit_from_left = self.findKthBit(n - 1, mirror_position)
            
            return "1" if bit_from_left == "0" else "0"
```



## Complexity

- **Time Complexity:** $O(n)$ — We recurse at most $n$ times (once per level), and each level does $O(1)$ work.
- **Space Complexity:** $O(n)$ — Recursion stack depth, or $O(1)$ if we use an iterative approach.



## Edge Cases

1. **k is exactly the middle** (`k = 2^(n-1)`): Should return "1" immediately without recursion
2. **n = 1**: The base case, only valid k is 1, returns "0"
3. **k is at the very end** (`k = 2^n - 1`): Requires maximum recursion depth with multiple inversions
4. **Large n with small k** (e.g., `n=20, k=1`): Quickly resolves to base case since we're always in left half



## Lessons

- **Don't materialize what you can compute**: The string grows exponentially ($2^{20}$ is about 1 million), but we solve it in 20 steps.
- **Mirror symmetry is your friend**: When a structure is built as `X + middle + reverse(X)`, you can always map positions in the right half back to the left half.
- **Track your flips**: Each time you cross into an "inverted" region, think of it as toggling a flip flag. Here, the recursion naturally handles it by inverting the result.
- **Bit manipulation intuition**: `(1 << (n-1))` is a clean way to compute $2^{n-1}$ without importing `math`.
