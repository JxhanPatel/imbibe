# 3300. Minimum Element After Replacement With Digit Sum 🟢

**Topics:** Array, Math






## Intuition

Think of each number as a bag of digits.
Instead of caring about the whole number, we break the bag open and add all the digits together.

For example:

* `14 → 1 + 4 = 5`
* `999 → 9 + 9 + 9 = 27`

After converting every number into its digit sum, we simply return the smallest value.

It is like compressing every number into a simpler score and then finding the minimum score.






## Approach

We will:

1. Traverse every number in `nums`
2. Compute its digit sum
3. Keep track of the minimum digit sum found

To calculate the digit sum:

* Repeatedly take the last digit using `% 10`
* Add it to a running total
* Remove the last digit using `// 10`




## Example Walkthrough

### Input

```python
nums = [10, 12, 13, 14]
```

### Step-by-step

* `10 → 1 + 0 = 1`
* `12 → 1 + 2 = 3`
* `13 → 1 + 3 = 4`
* `14 → 1 + 4 = 5`

New array becomes:

```python
[1, 3, 4, 5]
```

Minimum element is:

```python
1
```






## Code

```python
class Solution:
    def minElement(self, nums: List[int]) -> int:

        def digit_sum(n):
            s=0

            while n:
                s+= n%10
                n//= 10
            return s

        return min(digit_sum(n) for n in nums)
```







## Complexity

* Let `n` be the length of `nums`
* Let `d` be the number of digits in a number

Time Complexity:

$$
O(n \cdot d)
$$

Space Complexity:

$$
O(1)
$$






## Edge Cases

1. Single element array

```python
nums = [5]
```

Answer is `5`.

2. Numbers containing zeros

```python
nums = [1000]
```

Digit sum becomes `1`.

3. Large values within constraints

```python
nums = [9999]
```

Digit sum becomes `36`.






## Lessons

* Digit manipulation problems commonly use:

  * `% 10` to extract the last digit
  * `// 10` to remove the last digit

* Instead of modifying the entire array, we can compute values on the fly and directly track the minimum.

* Simple helper functions make the solution cleaner and easier to understand.
