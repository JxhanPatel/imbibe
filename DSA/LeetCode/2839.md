# 2839. Check if Strings Can be Made Equal With Operations I 🟢
Topics: String, Simulation

## Intuition

Imagine you have two shelves, each with 4 labeled slots. The rule is: you can only swap items that are 2 slots apart. So slot 0 can trade with slot 2, and slot 1 can trade with slot 3. But slot 0 can never reach slot 1.

This means the even positions (0 and 2) form their own little world, and the odd positions (1 and 3) form another. Items inside each world can be rearranged freely, but they can never cross over to the other world.

So the question becomes: do both strings have the same "even world" characters and the same "odd world" characters? If yes, you can always rearrange one to match the other.

## Approach

Instead of simulating the swaps, we realize something smarter: since we can always swap freely within even and odd positions (because swapping twice gets you back, and any order is reachable), we just need to check if the multiset of characters at even indices is the same for both strings, and same for odd indices.

Since the string is always length 4, each group has exactly 2 characters. Sorting those 2 characters and comparing is the easiest path.

## Example Walkthrough

Input: `s1 = "abcd"`, `s2 = "cdab"`

1. Even indices (0, 2): s1 gives `['a', 'c']`, s2 gives `['c', 'a']`
   - Sorted: `['a', 'c']` vs `['a', 'c']` -- match!
2. Odd indices (1, 3): s1 gives `['b', 'd']`, s2 gives `['d', 'b']`
   - Sorted: `['b', 'd']` vs `['b', 'd']` -- match!
3. Both match, so return `True`.

## Code

```python
class Solution:
    def canBeEqual(self, s1: str, s2: str) -> bool:

        even_s1 = sorted(s1[0] + s1[2])
        even_s2 = sorted(s2[0] + s2[2])


        odd_s1 = sorted(s1[1] + s1[3])
        odd_s2 = sorted(s2[1] + s2[3])


        return even_s1 == even_s2 and odd_s1 == odd_s2
```


## Complexity

Since the string is always length 4, technically everything here is $O(1)$. But thinking about the general pattern:

- **Time:** $O(n \log n)$ where $n$ is the string length, because of the sort. For $n = 4$, that's constant.
- **Space:** $O(n)$ for the sorted lists we create.

## Edge Cases

- **Both strings are already equal** (`s1 = "abcd"`, `s2 = "abcd"`): works fine, the sorted groups match immediately.
- **Same characters but wrong lane** (`s1 = "abcd"`, `s2 = "bacd"`): even groups are `{a,c}` vs `{a,c}` (match), but odd groups are `{b,d}` vs `{a,c}` (no match) -- correctly returns `False`. You cannot move `b` from index 1 to index 0.
- **All same characters** (`s1 = "aaaa"`, `s2 = "aaaa"`): trivially `True`, nothing to do.
- **Single character difference in wrong lane** (`s1 = "abcd"`, `s2 = "axcd"`): odd groups are `{b,d}` vs `{x,d}` -- catches the mismatch and returns `False`.

## Lessons

- **Invariant thinking**: instead of simulating what's possible, ask "what property is preserved no matter what operations we do?" Here, the set of characters in each lane is the invariant. Swaps can never change it.
- **Even/odd index splitting** with `s[0::2]` and `s[1::2]` is a clean Python trick worth having in your toolkit.
- **Sorting as a canonical form**: when you need to compare two groups without caring about order, sort both and compare. It's a super common pattern in string and array problems.
- **Read the constraint**: the fixed length of 4 is a hint that the problem doesn't need anything fancy. When constraints are tiny, simple and readable beats clever.
