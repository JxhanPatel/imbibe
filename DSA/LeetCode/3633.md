## 3633. Earliest Finish Time for Land and Water Rides I 🟢

**Topics:** Array, Brute Force, Simulation




## Intuition

Think of the tourist as planning two activities: one land ride and one water ride.

For every possible land-water pair, there are only two schedules:

1. Land ride first, then water ride.
2. Water ride first, then land ride.

After finishing the first ride, the tourist can either immediately start the second ride if it is already open or wait until it opens.

Since there are at most 100 rides in each category, we can simply try every possible pair and both possible orders, then keep track of the earliest finishing time.





## Approach

First, pair each ride's start time and duration using `zip()` for easier iteration.

For every land ride:

* Compute when the land ride finishes.

For every water ride:

* Compute when the water ride finishes.
* Calculate the finish time if we do:

  * Land → Water
  * Water → Land
* Take the smaller of the two.
* Update the global best answer if this schedule finishes earlier.

The key formula is:

```python
max(first_ride_finish, second_ride_start_time)
```

This ensures we either start immediately after finishing the first ride or wait until the second ride opens.






## Example Walkthrough

### Input

```text
landStartTime = [2, 8]
landDuration = [4, 1]

waterStartTime = [6]
waterDuration = [3]
```

### Pair: Land(2,4) and Water(6,3)

Land → Water:

```text
Land starts at 2
Land ends at 6

Water opens at 6
Start water at 6

Finish = 6 + 3 = 9
```

Water → Land:

```text
Water starts at 6
Water ends at 9

Land already opened at 2
Start land at 9

Finish = 9 + 4 = 13
```

Best for this pair:

```text
min(9, 13) = 9
```

### Pair: Land(8,1) and Water(6,3)

Land → Water:

```text
Land ends at 9
Water starts at 9

Finish = 12
```

Water → Land:

```text
Water ends at 9
Land starts at 9

Finish = 10
```

Best for this pair:

```text
min(12, 10) = 10
```

Overall answer:

```text
9
```

## Code

```python
class Solution:
    def earliestFinishTime(self, landStartTime: List[int], landDuration: List[int], waterStartTime: List[int], waterDuration: List[int]) -> int:

        best_time = 999999999999
        
        land_tasks = list(zip(landStartTime, landDuration))
        water_tasks = list(zip(waterStartTime, waterDuration))

        for l_start, l_dur in land_tasks:
            l_end = l_start+l_dur
            
            for w_start, w_dur in water_tasks:
                w_end = w_start+w_dur

                t1 = max(l_end, w_start)+ w_dur
                t2 = max(w_end, l_start)+ l_dur
                
                shortest = t1 if t1<t2 else t2
                
                if shortest < best_time:
                    best_time=shortest


        return best_time
```

## Complexity

Let:

* (n) = number of land rides
* (m) = number of water rides

Time Complexity:

$$
O(n \times m)
$$

Space Complexity:

$$
O(n + m)
$$

The extra space comes from creating:

```python
land_tasks = list(zip(...))
water_tasks = list(zip(...))
```

If we iterate directly using indices, the space can be reduced to:

$$
O(1)
$$

## Edge Cases

### 1. Only one ride in each category

```text
landStartTime = [5]
landDuration = [3]

waterStartTime = [1]
waterDuration = [10]
```

Must compare both possible orders.

### 2. Large waiting time

```text
landStartTime = [1]
landDuration = [2]

waterStartTime = [100]
waterDuration = [5]
```

The tourist waits for the water ride to open.

### 3. Both rides already open

```text
landStartTime = [5]
landDuration = [2]

waterStartTime = [4]
waterDuration = [3]
```

No waiting is required after finishing the first ride.

## Lessons

* Small constraints often allow a straightforward brute-force solution.
* Scheduling problems frequently use `max(current_time, opening_time)` to handle waiting.
* When there are only a few possible execution orders, evaluate all of them directly.
* Pairing related values with `zip()` can make iteration cleaner and easier to understand.
* Always compute finish times first, then determine when the next activity can begin.
