# 3691. Maximum Total Subarray Value II 🔴

**Topics:** Sparse Table, Heap (Priority Queue), Range Query

## Intuition

Imagine every starting index in the array owns a collection of subarrays.

For a fixed starting index `i`, the largest possible subarray value usually comes from extending the subarray as far as possible because it gives more opportunities to encounter larger maximums and smaller minimums.

This solution treats each starting position as a candidate generator:

* Start with the largest subarray beginning at each index.
* Compute its value:

$$
\text{value} = \max(subarray) - \min(subarray)
$$

* Store all these candidates in a max-heap.
* Repeatedly take the best available subarray.
* After using a subarray `(left, right)`, generate the next candidate from the same starting position by shrinking the right boundary.

To make range minimum and maximum queries extremely fast, we build a **Sparse Table**.

The Sparse Table allows us to find:

$$
\min(nums[l..r])
$$

and

$$
\max(nums[l..r])
$$

in constant time after preprocessing.





## Approach

### Step 1: Build a Sparse Table

The Sparse Table stores:

* `min_table[k][i]` = minimum value in interval of length (2^k) starting at `i`
* `max_table[k][i]` = maximum value in interval of length (2^k) starting at `i`

Using these tables, any range query can be answered in:

$$
O(1)
$$

time.


### Step 2: Create Initial Candidates

For every index `i`, create the largest possible subarray:

```python
(i, n - 1)
```

Compute:

```python
max(nums[i:n]) - min(nums[i:n])
```

and push it into a max-heap.

Since Python only provides a min-heap, negative values are stored.


### Step 3: Repeatedly Extract the Best Subarray

While we still need more subarrays:

1. Pop the subarray with the largest value.
2. Add its contribution to the answer.
3. Decrease `k`.
4. Generate the next candidate from the same starting position by reducing the right endpoint.

For example:

```text
(0, 5) → (0, 4) → (0, 3) → ...
```

The heap always keeps track of the current best candidate from every starting position.


### Sparse Table Construction

For every power of two:

$$
2^k
$$

combine two intervals of length:

$$
2^{k-1}
$$

For minimums:

$$
\text{minTable}[k][i]
$$

$$
\min(
\text{minTable}[k-1][i],
\text{minTable}[k-1][i+2^{k-1}]
)
$$

For maximums:

$$
\text{maxTable}[k][i]
$$

$$
\max(
\text{maxTable}[k-1][i],
\text{maxTable}[k-1][i+2^{k-1}]
)
$$

Preprocessing cost:

$$
O(n \log n)
$$

## Code

```python
import heapq
import math

class SparseTable:
    def __init__(self, arr: list[int]):
        self.n = len(arr)

        if not self.n:
            return

        self.max_power = int(math.log2(self.n))+1
        self.min_table = [list(arr)]+[[0]*self.n for _ in range(self.max_power-1)]
        self.max_table = [list(arr)]+[[0]*self.n for _ in range(self.max_power-1)]


        for k in range(1, self.max_power):
            half = 1 << (k-1)


            for i in range(self.n-(1 << k)+1):
                j = i+half
                self.min_table[k][i] = min(self.min_table[k-1][i], self.min_table[k - 1][j])
                self.max_table[k][i] = max(self.max_table[k-1][i], self.max_table[k- 1][j])


    def query_range(self, lo: int, hi: int) -> tuple[int, int]:
        if lo > hi:
            return 0, 0

        k = int(math.log2(hi-lo+1))
        r = hi - (1 << k)+1
        mn = min(self.min_table[k][lo], self.min_table[k][r])
        mx = max(self.max_table[k][lo], self.max_table[k][r])
        return mn, mx


class Solution:
    def maxTotalValue(self, nums: List[int], k: int) -> int:
        if not nums or k <= 0:
            return 0

        n = len(nums)
        table = SparseTable(nums)

        def make_entry(left, right):
            mn, mx = table.query_range(left, right)
            return (-(mx - mn), left, right)

        pq = [make_entry(i, n-1) for i in range(n)]
        heapq.heapify(pq)

        ans=0

        while k > 0 and pq:
            neg_diff, left, right = heapq.heappop(pq)
            diff = -neg_diff

            if diff <= 0:
                break

            ans+= diff
            k -= 1

            if left < right:
                heapq.heappush(pq, make_entry(left, right-1))

        return ans


      
```







## Complexity

### Sparse Table Construction

$$
O(n \log n)
$$

### Range Query

$$
O(1)
$$

### Heap Operations

There are at most:

$$
k
$$

extractions and insertions.

Each heap operation costs:

$$
O(\log n)
$$

Therefore:

$$
O(k \log n)
$$

### Total Complexity

$$
O(n \log n + k \log n)
$$

### Space Complexity

Sparse tables:

$$
O(n \log n)
$$

Heap:

$$
O(n)
$$

Overall:

$$
O(n \log n)
$$

## Lessons

* Sparse Tables are excellent for static range minimum and maximum queries.
* Preprocessing can reduce expensive repeated range operations from (O(n)) to (O(1)).
* Heaps are useful when repeatedly selecting the current best candidate among many choices.
* Combining a Sparse Table with a Priority Queue allows efficient exploration of high-value subarrays.
* This solution greedily explores subarrays by shrinking the right boundary of each starting position while always choosing the best available candidate next.
