# 3464. Maximize the Distance Between Points on a Square 🔴

**Topics:** Binary Search, Greedy, Geometry, Sorting







## Intuition

Imagine walking along the boundary of a square. Instead of thinking in 2D, we “unwrap” the square into a straight line of length `4 * side`.

Now the problem becomes:

> Pick `k` points on a circular track such that the minimum distance between any two selected points is maximized.

This is very similar to the classic “maximize minimum distance” problems like placing cows in stalls.

The key trick is:

* Use the perimeter only for **ordering**
* Use greedy + binary search to **maximize spacing**








## Approach

### Step 1: Convert 2D → 1D (Perimeter Mapping)

We map each boundary point to a linear position:

* Left edge `(0, y)` → `y`
* Top edge `(x, side)` → `side + x`
* Right edge `(side, y)` → `3*side - y`
* Bottom edge `(x, 0)` → `4*side - x`

Now all points lie on `[0, 4 * side)`.

---

### Step 2: Sort Positions

Sort all mapped positions.

---

### Step 3: Binary Search on Answer

We binary search the answer `d` (minimum distance).

For each `d`, we check:

> Can we pick `k` points such that each consecutive point is at least `d` apart?

---

### Step 4: Greedy + Binary Search (Feasibility Check)

For each starting point:

* Try to greedily place the next point at least `d` away
* Use `bisect_left` to jump directly instead of scanning
* Stop if we place `k` points

Also ensure we don’t exceed the circular boundary:

* Limit placements within `start + perimeter - d`












## Example Walkthrough

**Input:**

```
side = 2
points = [[0,0],[1,2],[2,0],[2,2],[2,1]]
k = 4
```

### Step 1: Mapping

```
(0,0) → 0
(1,2) → 3
(2,0) → 6
(2,2) → 4
(2,1) → 5
```

Sorted:

```
[0, 3, 4, 5, 6]
```

---

### Step 2: Try `d = 2`

* Start at 0 → next ≥ 2 → 3
* 3 → next ≥ 5 → 5
* 5 → next ≥ 7 → not possible

Fails → try smaller

---

### Step 3: Try `d = 1`

We can place 4 points → valid

Answer = 1








## Code

```python
class Solution:
    def maxDistance(self, side: int, points: List[List[int]], k: int) -> int:
        perimeter = side * 4

        def to_linear(x, y):
            if x ==0:
                return y

            elif y== side:
                return side+ x

            elif x ==side:
                return side * 3-y

            else:
                return perimeter - x



        positions = sorted(to_linear(x, y) for x, y in points)

        n = len(positions)


        def can_place(min_dist):

            for i in range(n):

                start = positions[i]
                cur = start
                count = 1
                ok = True

                while count < k:
                    idx = bisect_left(positions, cur + min_dist)

                    wrapped = bisect_left(positions, cur + min_dist - perimeter)

                    if idx < n and positions[idx] <= start + perimeter - min_dist:
                        cur = positions[idx]


                    elif wrapped < n and positions[wrapped] + perimeter <= start + perimeter - min_dist + perimeter:
                        cur = positions[wrapped] + perimeter


                    else:
                        ok = False
                        break
                    count += 1

                if ok:
                    return True


            return False

        def can_fit(min_dist):
            for i in range(n):
                anchor = positions[i]

                cutoff = anchor + perimeter - min_dist

                cur = anchor

                placed = 1

                for _ in range(k - 1):
                    idx = bisect_left(positions, cur + min_dist)

                    if idx == n or positions[idx] > cutoff:
                        break

                    cur = positions[idx]

                    placed += 1

                if placed == k:
                    return True
            return False

        lo, hi = 1, side
        best = 0


        while lo <= hi:
            mid = (lo + hi) // 2
            if can_fit(mid):
                best = mid
                lo = mid+ 1
            else:
                hi = mid -1

        return best
```







## Complexity

* Mapping + sorting: $O(n \log n)$
* Binary search: $O(\log side)$
* Each check: $O(n \log n)$

Overall:

$$
O(n \log n \log side)
$$







## Edge Cases

1. All points clustered on one edge
   → minimum distance becomes small

2. `k = number of points`
   → must select all points

3. Very large `side` (up to $10^9$)
   → mapping avoids large coordinate issues







## Lessons

* Convert geometry problems into linear order when possible
* Binary search on answer is powerful for maximizing minimum values
* Greedy works when local choices lead to global optimum
* Use binary search (`bisect`) to optimize greedy jumps
* Circular problems can often be handled with careful bounds instead of duplication
