# 3614. Process String with Special Operations II 🔴

**Topics:** String, Simulation, Reverse Traversal


## Intuition

The most straightforward solution is to simulate every operation and build the final string.

But there is a problem.

```text
k <= 10^15
final string length <= 10^15
```

The final string can become extremely large because of the `#` operation, which duplicates the whole string.

So instead of constructing the string, we ask a different question:

> Which original character eventually ends up at position `k`?

This is much easier.

We first record **only the length** of the string after each operation.

Then we start from the end of the operations and walk backwards.

At each step, we update `k` to represent the position it had **before** the current operation.

Think of it like rewinding a movie.

* If `%` reversed the string, reverse the index too.
* If `#` duplicated the string, map `k` back into the first copy.
* If we reach a letter and its position matches `k`, we found our answer.

This allows us to find the answer without ever building the actual string.





## Approach

We solve the problem in two passes.

### Pass 1: Store sizes

Create an array:

```text
sizes[i]
```

which stores the length of the result after processing the first `i` characters.

For every operation:

* Letter

```text
size += 1
```

* `*`

```text
size = max(0, size - 1)
```

* `#`

```text
size *= 2
```

* `%`

```text
size stays the same
```



### Pass 2: Walk backwards

If

```text
k >= final_size
```

the answer is immediately:

```text
'.'
```

Otherwise, process the operations in reverse order.

#### Letter

A letter is always appended at the end.

If

```text
k == previous_size
```

then this letter is exactly the character we want.

Return it.



#### `#`

Suppose the string before duplication is:

```text
abc
```

After duplication:

```text
abcabc
```

If `k` lies in the second half, it corresponds to the same position in the first half.

So:

```text
k %= previous_size
```



#### `%`

Suppose:

```text
abcd
```

becomes:

```text
dcba
```

Index mapping:

```text
0 -> 3
1 -> 2
2 -> 1
3 -> 0
```

So:

```text
k = previous_size - 1 - k
```



#### `*`

Deleting the last character does not affect any earlier positions.

Since we already know `k` is valid in the final string, nothing needs to be done.



## Example Walkthrough

### Input

```text
s = "a#b%*"
k = 1
```

### Forward pass

| Character | Operation    | Size |
| --------- | ------------ | ---: |
| Start     | Empty string |    0 |
| a         | Append       |    1 |
| #         | Duplicate    |    2 |
| b         | Append       |    3 |
| %         | Reverse      |    3 |
| *         | Delete last  |    2 |

Final size:

```text
2
```

Since:

```text
k = 1 < 2
```

continue.



### Backward pass

Start:

```text
k = 1
```



Process `*`

Deletion only removes the last character.

```text
k remains 1
```



Process `%`

Length before reversal:

```text
3
```

Update:

```text
k = 3 - 1 - 1
  = 1
```



Process `b`

Before appending:

```text
length = 2
```

Since:

```text
k != 2
```

continue.



Process `#`

Original length:

```text
1
```

Update:

```text
k %= 1
k = 0
```



Process `a`

Before appending:

```text
length = 0
```

Since:

```text
k == 0
```

Answer:

```text
'a'
```



## Code

```python
class Solution:
    def processStr(self, s: str, k: int) -> str:

        sizes=[0]

        for char in s:
            curr = sizes[-1]

            if char.isalpha():
                sizes.append(curr+1)

            elif char=='*':
                sizes.append(max(0, curr-1))

            elif char=='#':
                sizes.append(curr*2)

            else:
                sizes.append(curr)


        if k >= sizes[-1]:
            return '.'



        for idx in reversed(range(len(s))):
            char = s[idx]
            prev_size = sizes[idx]


            if char.isalpha():
                if k==prev_size:
                    return char


            elif char=='#':
                if prev_size > 0:
                    k %= prev_size


            elif char=='%':
                if prev_size > 0:
                    k = prev_size-1-k

        return '.'
```



## Complexity

### Time Complexity

First pass:

$$
O(n)
$$

Second pass:

$$
O(n)
$$

Overall:

$$
O(n)
$$



### Space Complexity

We store the size after each operation.

$$
O(n)
$$

where `n` is the length of `s`.



## Edge Cases

1. **Result becomes empty**

```text
s = "z*#"
k = 0
```

Process:

```text
z  -> "z"
*  -> ""
#  -> ""
```

Answer:

```text
'.'
```



2. **Duplicate an empty string**

```text
s = "#"
```

Process:

```text
"" -> ""
```

Length remains zero.



3. **Multiple reversals**

```text
s = "abc%%"
```

Process:

```text
abc
cba
abc
```

Two reversals cancel each other.



4. **Very large k**

```text
k = 10^15
```

The algorithm never builds the final string, so it still runs efficiently in linear time.



## Lessons

* When the final data structure can become enormous, store only the information you actually need.
* Reverse traversal is a powerful technique when operations are easier to undo than to simulate.
* Duplication operations often reduce to modular arithmetic.
* Reversal operations usually translate into simple index transformations.
* Always ask whether you need the entire result or only a small piece of it. Often, tracking positions is enough.
