# 1722. Minimize Hamming Distance After Swap Operations 🟡

**Topics:** Union Find, Graph, Hash Map





## Intuition

Think of the indices as people standing in groups.

If two indices can be swapped, they are connected. And if A can swap with B, and B with C, then A, B, and C are all in the same group. Inside a group, you can rearrange elements however you want.

So instead of worrying about individual swaps, focus on **groups of indices**.

Now imagine each group as a bag. You can shuffle values inside the bag freely. The goal is to match the `target` as much as possible using the values available in each group.






## Approach

We use **Union Find (Disjoint Set)** to form connected components.

Steps:

1. Build groups using Union Find:

   * For each `[a, b]` in `allowedSwaps`, connect `a` and `b`.

2. Group indices by their root parent:

   * Each root represents a connected component.

3. For each group:

   * Count frequencies of elements in `source`.
   * Try to match with `target`.
   * If a target value exists in the group’s source frequency, reduce count.
   * Otherwise, it contributes to Hamming distance.

4. Sum all mismatches.








## Example Walkthrough

**Input:**
source = [1,2,3,4]
target = [2,1,4,5]
allowedSwaps = [[0,1],[2,3]]

Step 1: Build groups

* Group 1: {0,1}
* Group 2: {2,3}

Step 2: Process Group {0,1}

* source values: [1,2]
* target values: [2,1]
* Both can match perfectly → 0 mismatch

Step 3: Process Group {2,3}

* source values: [3,4]
* target values: [4,5]
* 4 matches
* 5 not present → 1 mismatch

Final answer = 1









## Code

```python
from collections import defaultdict
from collections import Counter

class Solution:
    def minimumHammingDistance(self, source: List[int], target: List[int], allowedSwaps: List[List[int]]) -> int:

        parent = list(range(len(source)))

        def find(x):

            while parent[x] != x:
                parent[x] = parent[parent[x]]
                x = parent[x]

            return x

        for a, b in allowedSwaps:

            pa, pb = find(a), find(b)

            if pa != pb:
                parent[pb] = pa



        buckets = defaultdict(list)

        for i in range(len(source)):
            buckets[find(i)].append(i)

        mismatches = 0

        for group in buckets.values():
            available = Counter(source[i] for i in group)

            for i in group:
                if available[target[i]] > 0:
                    available[target[i]] -= 1

                else:
                    mismatches += 1


        return mismatches
```




## Complexity

* Union Find operations: $O(n \alpha(n))$
* Group processing: $O(n)$

Overall:
$$O(n \alpha(n))$$





## Edge Cases

* No allowed swaps → just compare arrays directly
* All indices connected → full freedom to rearrange
* All values same → distance is always 0
* Completely different arrays with no overlap → max distance





## Lessons

* When swaps are allowed, think in terms of **connected components**, not individual swaps
* Inside a component, order does not matter, only frequency matters
* Union Find is powerful for grouping problems
* Matching with frequency maps helps minimize mismatches efficiently
