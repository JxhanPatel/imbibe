# 2463. Minimum Total Distance Traveled 🔴
**Topics:** Array, Dynamic Programming, Sorting




## Intuition

Think of it like a triage system at a hospital. You have broken-down robots scattered along a road, and repair factories spaced out along the same road. Each factory can only repair a limited number of robots. The smartest assignment is obvious once everything is sorted: just assign nearby robots to nearby factories from left to right, like a hospital routing patients to the nearest available bed. You would never fly a patient across the country if there is an open bed next door.

The key insight: **after sorting, optimal assignments never "cross"**. Robot at position 3 should not go to factory at position 10 while robot at position 8 goes to factory at position 2. Sorted order + no crossing = a classic DP setup.




## Approach

The trick here is to simplify the problem by "flattening" each factory into individual slots. A factory at position 5 with limit 3 becomes three separate slots: [5, 5, 5]. This turns a 2D assignment problem into a cleaner 1D matching.

After flattening, you have a sorted list of robots and a sorted list of slots. Then you run a standard 1D DP:

`dp[i][j]` = minimum total distance to assign the first `i` robots to the first `j` slots.

For each state, you have two choices:
1. **Skip this slot** for robot `i` (maybe a closer slot exists later) → `dp[i][j] = dp[i][j-1]`
2. **Use this slot** for robot `i` (and possibly robots before it too, since slots come from the same factory) → `dp[i][j] = dp[i-1][j-1] + |robot[i] - slot[j]|`

You take the minimum of these two options at every cell.







## Example Walkthrough

Input: `robot = [0, 4, 6]`, `factory = [[2, 2], [6, 2]]`

**Step 1 - Sort both:** robots = [0, 4, 6], factories sorted = [[2,2],[6,2]]

**Step 2 - Flatten factory slots:** slots = [2, 2, 6, 6]

**Step 3 - Build DP table** (see diagram above):

- `dp[1][1]`: R0 at position 0 goes to slot at 2. Cost = |2-0| = **2**
- `dp[2][2]`: R1 at position 4 goes to slot at 2. Cost = dp[1][1] + |2-4| = 2 + 2 = **4**
- `dp[3][3]`: R2 at position 6 goes to slot at 6. Cost = dp[2][2] + |6-6| = 4 + 0 = **4**

**Answer: 4**






## Code

```python
class Solution:
    def minimumTotalDistance(self, robot: List[int], factory: List[List[int]]) -> int:

        robot.sort()
        factory.sort()
        
        positions = []

        for f_pos, capacity in factory:
            positions.extend([f_pos] * capacity)
        
        n, m = len(robot), len(positions)

        dp = [float('inf')] * (n + 1)
        dp[0] = 0
        
        for j in range(m):

            for i in range(n, 0, -1):
                
                if dp[i-1] != float('inf'):
                    dp[i] = min(dp[i], dp[i-1] + abs(robot[i-1] - positions[j]))
                    
        return dp[n]
```








## Complexity

**Time:** $O(r \cdot s)$ where $r$ is the number of robots and $s = \sum \text{limits}$ is total factory capacity. In the worst case $s$ can be up to $r \cdot |\text{factory}|$, so roughly $O(r \cdot r \cdot f)$.

**Space:** $O(r \cdot s)$ for the DP table. This can be optimized to $O(s)$ by keeping only the previous row, since each cell only looks at `dp[i-1][j-1]` and `dp[i][j-1]`.








## Edge Cases

**1. Robot already at a factory position**
If `robot[i] == factory[j]`, the cost for that assignment is 0. The code handles this naturally via `abs(robot[i-1] - slots[j-1])`. No special treatment needed.

**2. Factory with limit 0**
A factory with `limit = 0` contributes zero slots to the flattened list, so it simply gets ignored. This is correct because that factory repairs nobody.

**3. More slots than robots**
The extra slots just never get used. The `dp[i][j] = dp[i][j-1]` skip transition lets us slide past unused slots with no penalty.

**4. All robots on the same side of all factories**
Say robots are all at position 100 and the only factory is at position 1. Every robot has to travel 99 units. The DP still works fine since sorting ensures all robots land on the same side and assignments are made left to right.

**5. Single robot, single factory with limit 1**
Simplest case. Flattened slots = [factory_pos]. Answer = |robot[0] - factory_pos|.







## Lessons

**1. Sorting eliminates crossing:** In assignment problems along a line, optimal solutions never have two robots "swap" their factory destinations. Sorting + DP exploits this.

**2. Flatten to simplify:** Turning `(position, limit)` pairs into repeated positions is a classic compression step. It trades a harder multi-dimensional constraint for a clean 1D sequence.

**3. The "skip" transition is the heart of DP:** The `dp[i][j] = dp[i][j-1]` line is what gives you flexibility. It says "I don't have to use every slot, I can be picky." This comes up in interval scheduling, knapsack, and string alignment problems too.

**4. Infinity as "impossible":** Initializing to `float('inf')` and using `if dp[...] < INF` before adding to it is a clean way to propagate "this state is unreachable" without branching everywhere.
